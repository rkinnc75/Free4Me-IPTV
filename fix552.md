## Bug

The TV header (top-left, `tv_top_tab_bar.dart`) renders `assets/icon.png`. That
asset was still the **old green/blue "Free4Me-IPTV" circle logo**, not the
current neon play-triangle. (The fix529 comment "the real app icon, not the
placeholder green TV icon" only meant it stopped using a Material `Icons.tv`
glyph — the asset it pointed at was itself outdated.)

Notably, the **Android launcher icon** (`mipmap-*/ic_launcher.webp`) was ALREADY
the correct neon icon, so the home-screen icon looked right while the in-app TV
header showed the old logo — the mismatch Rich spotted.

## Fix

Replace `assets/icon.png` (1024×1024) with the current neon play-triangle icon
(circuit-board background, cyan/magenta neon triangle) — the same art as the
launcher icon. Only file changed; the single `Image.asset('assets/icon.png')`
reference in `tv_top_tab_bar.dart` now shows the correct icon. No code change.
