## 1. Why

Releases currently carry v1+v2 signatures only — AGP does NOT enable v3 by default (the fix548 comment claiming otherwise was wrong). v3 (Android 9+) adds key **rotation** capability: if the release keystore is ever compromised or lost, we can rotate to a new key without breaking updates for the installed base. No rotation happens now — the key, cert, and cert SHA-256 are completely unchanged; updates install over prior releases exactly as before. v4 (`.idsig`, incremental adb installs) is intentionally NOT added — irrelevant to our distribution.
