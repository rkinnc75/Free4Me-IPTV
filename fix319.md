# fix319 — Fix slow/crashing TV with huge catalogues: skip in-memory cache on low-RAM + index the browse query → v1.26.35+319

## Root cause (from the user's log + 2GB onn 4K Plus)
3 sources, **737,038 channels**. Two problems:
- `ChannelSearchCache: rebuilt 737038 entries in 37090ms (~63MB)` — building a 63MB in-memory cache on a ~1.9GB device causes an OOM restart (the log shows two "App started" cycles = crash + relaunch).
- `SLOW SQL 22-27s` on the no-query/History browse — full-table scans over 737k rows.

## What this does
1. **Skip the in-memory search cache on low-RAM devices.** New `ChannelSearchCache.cacheSkipped` (true when `DeviceMemory.totalMb < 2300`). On those devices `ensureBuilt()`/`rebuild()` early-return (no 63MB alloc, no 37s build), and search transparently uses the direct SQL/FTS path. The cache's mutators already no-op safely when not built, so favorites/history/scan updates are harmless. High-RAM devices (Shield 3GB, phones) are unchanged — they still build the cache.
2. **Index the big-catalogue browse.** Migration 26 adds `idx_channel_src_media_url(source_id, media_type, url)` and a partial `idx_channel_lastwatched_media(last_watched, media_type) WHERE last_watched IS NOT NULL`, so the no-query and History queries seek instead of scanning all rows.

## Why this is safe
- Default search method is already SQL/FTS; the in-memory method was opt-in. The only behaviour change on low-RAM is "don't build the cache," with search routed to SQL — which the codebase already supports as the primary path.
- `cacheSkipped` is a pure function of detected RAM; high-RAM devices keep the fast in-memory cache.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Built against live main (1.26.34+318); patch applies cleanly to a fresh clone.
- Migration 26 tested on a seeded 100k-row sqlite: `EXPLAIN QUERY PLAN` for the History query now shows `SEARCH ... USING INDEX idx_channel_src_media_url` + PK lookup for the hide_dividers subquery — no full SCAN.
- Threshold tested: 1925 & 2048 → skip; 2300/3072/4096 → build (boundary 2299 skip, 2300 build).
