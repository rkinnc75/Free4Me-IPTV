# fix309 — Unify long-press menu for movies/series with live channels (All view consistency) → v1.26.25+309

> ## ⚠️ BUILD MACHINE — APPLY THE PATCH, then RELEASE (EXACTLY). No questions, no DB to query.
> **FIRST:** `find . -name 'fix296-DIAGNOSE*.md' -delete 2>/dev/null || true`

## Root cause
`_onLongPress` only built the bottom-sheet menu (favorite toggle + category link) when `mediaType == MediaType.livestream`. For movie/serie tiles it fell through to a bare `await favorite()` with no menu. In Live TV every tile is a livestream so the menu always appeared; in the All view (mixed types) long-pressing a movie or series silently toggled favorite with no menu — the inconsistency reported.

## The fix
Open the same menu for any non-group tile (`mediaType != MediaType.group`), so livestream, movie, and serie all get favorite + the tappable category link. Mini-player stays gated to `livestream` (and Remove-from-history to the history view) inside the sheet. The dead movie/serie `favorite()` fallthrough is removed.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Built against live main (1.26.24+308); patch applies cleanly to a fresh clone.
- Group tiles still hit their fix308 favorite-category branch first (unchanged); only the livestream-only gate broadened to all non-group types. Mini-player conditional preserved.
