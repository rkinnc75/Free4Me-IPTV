# fix138.md — Channel sort: full 6-tier multi-key sort (Favorites → History → All, Validated-first, Alphabetic) for the picker AND every Home media-type view (Live/Movies/Series/All)

> **Version:** Free4Me-IPTV 1.22.11 build (fix130/132).
> **Supersedes fix134.** Implements the complete intended ordering and applies
> it to BOTH code paths.
>
> **Intended order (6 tiers), favorite wins over history:**
> 1. Favorite + Validated + Alphabetic
> 2. Favorite + Alphabetic            (favorite, not validated)
> 3. History  + Validated + Alphabetic   (watched, NOT favorite)
> 4. History  + Alphabetic            (watched, not validated, not favorite)
> 5. All      + Validated + Alphabetic   (neither favorite nor watched)
> 6. All      + Alphabetic
>
> **Two clarifications baked in from review:**
> - **NO "Validated" section header.** Validation is shown ONLY by the green
>   circle badge on each tile. Validated rows sort to the TOP of their group,
>   but there is no separate "Validated" header. Section headers are just the
>   three PRIMARY groups: **Favourites / History / All channels**.
> - **Applies to ALL media-type views, not just "All".** In Home, Live TV /
>   Movies / Series are NOT separate `ViewType`s — they are `media_type` filters
>   on the SAME query (confirmed: `ViewType` enum is `all/categories/favorites/
>   history`; the tabs change `filters.mediaTypes`). So ONE `ORDER BY` change
>   covers Live, Movies, Series, AND All. The picker is livestream-only by
>   design and sorts in Dart.

## Tier table (single source of truth for both implementations)

| Tier | favorite | watched (lastWatched≠null) | validated | Header group |
|------|----------|----------------------------|-----------|--------------|
| 0 | ✓ | (any) | ✓ | Favourites |
| 1 | ✓ | (any) | ✗ | Favourites |
| 2 | ✗ | ✓ | ✓ | History |
| 3 | ✗ | ✓ | ✗ | History |
| 4 | ✗ | ✗ | ✓ | All channels |
| 5 | ✗ | ✗ | ✗ | All channels |

Within each tier: **alphabetical by name, case-insensitive.** Favorite is
checked FIRST (a favorite is never in a History tier, regardless of watched
state). Validation is a sub-order within each header group and is surfaced by
the green-circle badge — never its own header.

"Validated" = persisted `stream_validated` flag, OR (picker only) this-session
`StreamScanner.results[id] == true`.

# Fix 138.1 — Picker comparator (`channel_picker_screen.dart`)

**File:** `lib/channel_picker_screen.dart`

**Replace `_pickSort` (current 4-tier, ~lines 14–25):**

```dart
/// Sort key for the picker: favorites plus validated streams first, then
/// favorites, validated streams, and alphabetical fallback.
int _pickSort(Channel a, Channel b) {
  int tier(Channel ch) {
    final validated = ch.streamValidated == true ||
        (ch.id != null && StreamScanner.results[ch.id] == true);
    if (ch.favorite && validated) return 0;
    if (ch.favorite) return 1;
    if (validated) return 2;
    return 3;
  }
```

**With the full 6-tier version (note: the inner `tier()` and its closing lines
are fully replaced — see step removal note below):**

```dart
/// fix138: validated = persisted flag OR this-session scan result. Shared by
/// the sort tiers AND the section-header grouping so they can never desync.
bool _isValidated(Channel ch) =>
    ch.streamValidated == true ||
    (ch.id != null && StreamScanner.results[ch.id] == true);

/// fix138: 6-tier multi-key sort.
/// Favorite → History → All; Validated-first within each; then Alphabetic.
/// Favorite wins over history (a favorite is never in a History tier).
int _channelTier(Channel ch) {
  final validated = _isValidated(ch);
  final watched = ch.lastWatched != null;
  if (ch.favorite) return validated ? 0 : 1;   // favorite (watched-state irrelevant)
  if (watched) return validated ? 2 : 3;        // history (not favorite)
  return validated ? 4 : 5;                      // all (neither)
}

int _pickSort(Channel a, Channel b) {
  final ta = _channelTier(a);
  final tb = _channelTier(b);
  if (ta != tb) return ta.compareTo(tb);
  return a.name.toLowerCase().compareTo(b.name.toLowerCase()); // alphabetic within tier
}
```

> Make sure the ENTIRE old `_pickSort` (including its inner `tier()` body and
> the `return` statements that followed) is replaced — don't leave the old
> closing lines dangling. The new `_channelTier` is top-level (not nested).

# Fix 138.3 — Home ORDER BY: covers Live / Movies / Series / All in one change

**File:** `lib/backend/sql.dart`

Live TV, Movies, Series, and All all flow through this single `ORDER BY` (they
differ only by the `media_type IN (...)` filter, not by `ViewType`). So this one
change applies the 6-tier order to every media-type browse view. The current
ORDER BY has no distinct History group, so it does not produce the tiers.

**Current (~lines 346–354):**

```dart
    if (filters.viewType == ViewType.history) {
      sqlQuery += "\nORDER BY c.last_watched DESC";
    } else {
      sqlQuery += "\nORDER BY"
          " COALESCE(c.favorite, 0) DESC,"
          " COALESCE(c.stream_validated, 0) DESC,"
          " COALESCE(c.last_watched, 0) DESC,"
          " c.name ASC";
    }
```

**Replace with:**

```dart
    if (filters.viewType == ViewType.history) {
      // History TAB is its own view: most-recently-watched first (unchanged).
      sqlQuery += "\nORDER BY c.last_watched DESC";
    } else {
      // fix138: 6-tier multi-key sort matching the picker (_channelTier).
      // Applies to ALL media-type browse views (Live/Movies/Series/All) — they
      // share this query and differ only by the media_type filter.
      //   0: favorite+validated   1: favorite
      //   2: history +validated   3: history       (history = watched, not fav)
      //   4: validated            5: none
      // Favorite wins over history. Validated = persisted stream_validated=1.
      sqlQuery += "\nORDER BY"
          " CASE"
          "   WHEN COALESCE(c.favorite,0)=1 AND COALESCE(c.stream_validated,0)=1 THEN 0"
          "   WHEN COALESCE(c.favorite,0)=1 THEN 1"
          "   WHEN c.last_watched IS NOT NULL AND COALESCE(c.stream_validated,0)=1 THEN 2"
          "   WHEN c.last_watched IS NOT NULL THEN 3"
          "   WHEN COALESCE(c.stream_validated,0)=1 THEN 4"
          "   ELSE 5"
          " END ASC,"
          " c.name COLLATE NOCASE ASC";
    }
```

### Key changes
1. **Explicit `CASE` tier** — exactly the 6 tiers in the table, same precedence
   as the picker's `_channelTier`.
2. **`c.name COLLATE NOCASE ASC`** — case-insensitive alphabetical within each
   tier (matches the picker's `name.toLowerCase()`).
3. **Covers Live/Movies/Series/All** — the `else` branch is every non-History-
   tab view; the media-type tabs just vary the `media_type IN (...)` filter.
4. **History TAB unchanged** — still `last_watched DESC`.

> Validation source note: Home SQL uses the persisted `stream_validated` column
> only (no access to in-memory `StreamScanner.results`). The picker also counts
> session scans. They converge once a scan persists. Existing behaviour;
> documented, not a bug.
