# fix622 — Add buffer-seconds as a 3rd line to the multi-view stats overlay

**Target:** v2.2.33+622 (diff against live tag **v2.2.32**, verified `git ls-remote --tags`)
**Apply check:** `git apply --check` PASSED on a fresh v2.2.32 clone.
**Analyze:** run on the build machine — gate not asserted here. (Manual: braces + parens balanced.)

## What this adds
The compact stats overlay used in multi-view / PiP cells currently shows two lines (resolution+codec, fps+drops). fix588 dropped the buffer line there to fit small cells; the full single-view panel already shows it. This adds it back to the compact panel as a **third line**: `buf 12.3s`.

- Reads `demuxer-cache-duration` (mpv's seconds of buffered content) — already polled at 1 Hz and already in `_stats`; no new plumbing or mpv property.
- Formatted with the existing `_secs()` helper.
- Turns **red** (`0xFFFF5252`) when `paused-for-cache == yes` (an imminent/active rebuffer), mirroring the full panel's warn behavior.

## Live behavior (as requested)
Because the overlay refreshes every second, the number updates live: it hovers near the configured cache target ("Livestream cache (seconds)" for live, "VOD/Movie cache (seconds)" for VOD) when healthy, and visibly **counts down** (10 → 7 → 3 …) when the network can't keep up, going red when it stalls. That's the early-warning "is my buffer draining?" indicator, useful on the onn.
