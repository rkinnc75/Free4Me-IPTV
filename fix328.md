# fix328 — TV export OOM on large catalogues: stream files from disk → v1.26.44+328

> ## WARNING BUILD MACHINE — APPLY THE PATCH, then RELEASE (EXACTLY). No questions, no DB to query.
> **FIRST:** `find . -name 'fix296-DIAGNOSE*.md' -delete 2>/dev/null || true`

## Why (confirmed by the fix327 progress dialog)
The export reached "Gathering source data" then hit an **OOM** on the TV box. Root cause: the whole bundle was built in memory at once — the concatenated raw source dumps (tens of MB; the live-streams dump alone was 12MB+), a byte copy of each file in an `ExportItem`, a SECOND copy of every file inside the `Archive`, the full zip output, and then the HTTP server kept ALL of it resident for 10 minutes. On a 2GB box that blows the heap.

## What this does
1. **Disk-backed export (settings_view.dart):** `_buildExportBundle` writes each file (source dump, log, settings) to a temp dir and zips with `ZipFileEncoder` straight to a file on disk. The source dump is streamed file-to-file (`_streamSourceDumpToFile`) — never concatenated into one giant string.
2. **Streamed serving (export_server.dart):** `ExportItem` now holds a `filePath` + `sizeBytes` (was `List<int> bytes`); `/file/` responses stream from disk via `response.addStream(file.openRead())`. Peak heap is ~one buffered chunk, not the whole catalogue.
3. **TV source-dump path de-memoried:** `_exportSourceDumps` no longer builds a `StringBuffer` of every dump for the TV branch; it routes through the streaming builder. The phone branch (SAF, smaller catalogues) is unchanged.

+// fix328: verify the disk-backed export zip (ZipFileEncoder streams files from
+// disk and produces a valid, re-readable archive). Guards the OOM fix's core
+// mechanism.
+import 'dart:io';
+import 'package:archive/archive_io.dart';
+import 'package:flutter_test/flutter_test.dart';
+
+void main() {
+  test('ZipFileEncoder streams files to a valid zip on disk', () async {
+    final tmp = await Directory.systemTemp.createTemp('export_zip_test');
+    final a = File('${tmp.path}/free4me-source-dump.txt')
+      ..writeAsStringSync('===== FILE: x.json =====\n${'big' * 5000}\n');
+    final b = File('${tmp.path}/free4me-settings.json')
+      ..writeAsStringSync('{"schema":3}');
+    final zipPath = '${tmp.path}/out.zip';
+
+    final enc = ZipFileEncoder()..create(zipPath);
+    await enc.addFile(a, 'free4me-source-dump.txt');
+    await enc.addFile(b, 'free4me-settings.json');
+    await enc.close();
+
+    final zf = File(zipPath);
+    expect(await zf.exists(), isTrue);
+    expect(await zf.length(), greaterThan(0));
+
+    final arch = ZipDecoder().decodeBytes(await zf.readAsBytes());
+    final names = arch.files.map((f) => f.name).toSet();
+    expect(names, {'free4me-source-dump.txt', 'free4me-settings.json'});
+    final settings =
+        arch.files.firstWhere((f) => f.name == 'free4me-settings.json');
+    expect(
+        String.fromCharCodes(settings.content as List<int>), '{"schema":3}');
+
+    await tmp.delete(recursive: true);
+  });
