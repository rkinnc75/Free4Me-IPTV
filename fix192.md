# fix192 — Show the detected provider connection limit (read-only) on the source list

> **Version:** build `+192`; patch = next available at ship time (read live pubspec — patch and fix numbers drift; do NOT assume). Tag = next available `v1.23.X` (immutable; never reuse/delete/recreate).
> **Evidence:** The app auto-detects each Xtream provider's connection limit (`max_connections`, fix184/186) and uses it internally to gate multi-view layouts, but the user has no way to see it.
> **Bug:** N/A — this is a small display addition, not a defect fix.
> **Root cause:** N/A.

## Scope (decided with the user)

- The connection limit is **provider-supplied and auto-detected**. The user does **not** enter it. There is **no setting, no editable field, no override**.
- The only change is to **display** the detected value (read-only) on the source list tile, next to the source type.
- When the value is unknown (`null`, e.g. not yet detected or provider didn't report it), show nothing extra — just the source type as today.

> Note: an earlier idea (a manual override field in the add/edit UI) is **dropped** — not deferred, dropped. fix192 is display-only.

## Fix 192.1 — extend the source-tile subtitle to include the connection count

**File:** `lib/settings_view.dart`

Make exactly one edit. Locate the line by the **verbatim** "current code" below (it occurs exactly once). Replace it with the "replacement" block.

### Current code (verbatim — match exactly)

```dart
          subtitle: Text(source.sourceType.label),
```

### Replacement code (verbatim)

```dart
          // fix192: show the auto-detected provider connection limit (read-only)
          // alongside the source type. Hidden when unknown (null).
          subtitle: Text(
            source.maxConnections == null
                ? source.sourceType.label
                : '${source.sourceType.label} · '
                    '${source.maxConnections} '
                    '${source.maxConnections == 1 ? "connection" : "connections"}',
          ),
```

### Key changes
- When `source.maxConnections` is non-null, the subtitle reads e.g. `Xtream · 2 connections` (or `Xtream · 1 connection` for a single connection).
- When `source.maxConnections` is null, the subtitle is unchanged from today (`source.sourceType.label` only).
- The separator is a **literal middot `·`** (copy the replacement block verbatim — do not retype it as `\u00b7`).
- No new widget, no new symbol, no import, no model/SQL change.
