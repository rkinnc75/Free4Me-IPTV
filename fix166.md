# fix166 — TV export: human-readable snapshot timestamp on downloaded filenames (+ deferred D-pad sweep items)

> **Version:** 1.23.6+166 (phone/even build; patch bump from 1.23.5+164)
> **Evidence:** uploaded `free4me-backup__6_.json` (browser de-duped to `(6)` because the name is static) and `free4me_log_1780155704975.txt` (epoch-ms name is unreadable).
> **Tweak:** export filenames don't tell you *when* the snapshot was taken — the backup is a constant `free4me-backup.json` (collides → browser `(6)` suffixes), and the log uses unreadable `millisecondsSinceEpoch`. Stamp **all four** export paths (TV backup + log, phone backup + log) with a sortable, human-readable local datetime via one shared helper.
> **Scope:** a shared `SettingsIo.exportStamp` helper + four filename sites across `settings_io.dart` and `settings_view.dart`; the "Snapshot taken" line on the `:9479` index page (`export_server.dart`). Plus the D-pad reachability items deferred from fix164 (verify-then-fix; see end).

---

## Why

- `free4me-backup.json` has **no** timestamp → every export collides, and the browser disambiguates with `(1)…(6)`, which is meaningless and order-dependent.
- `free4me_log_1780155704975.txt` has a timestamp but it's epoch milliseconds — you can't read it. You have to convert it to know it was 2026-05-30 00:30-ish.

Goal: both files named like `free4me-backup-20260530-154223.json` / `free4me_log-20260530-154223.txt` — sortable, glanceable, and the backup + log from one export share the **same** stamp so they're obviously a matched pair.

# Fix 166.1 — shared timestamp helper + all four filenames

## (a) Shared helper — `lib/backend/settings_io.dart`

`intl` is a dependency but `DateFormat` isn't imported in either export file. Keep it import-free with a tiny core-Dart helper, placed **once** as a static on `SettingsIo` (which both `settings_io.dart` and `settings_view.dart` already use), so the four sites share one definition. Add near the top of the `SettingsIo` class body:

```dart
  /// fix166: local snapshot stamp `yyyymmdd-HHMMSS` for export filenames.
  /// Sortable and human-readable; local time so it matches the wall clock
  /// the user sees when they export. Shared by TV + phone export paths.
  static String exportStamp(DateTime now) {
    String p2(int n) => n.toString().padLeft(2, '0');
    return '${now.year}${p2(now.month)}${p2(now.day)}'
        '-${p2(now.hour)}${p2(now.minute)}${p2(now.second)}';
  }
```

## (b) Phone backup — `lib/backend/settings_io.dart`

In the backup export method (the one with `getTemporaryDirectory()` + `FilePicker.platform.saveFile`), the tmp file and the suggested name are both the static `free4me-backup.json`. Compute one stamp and use it for both. Current:

```dart
    final dir = await getTemporaryDirectory();
    final tmpFile = File('${dir.path}/free4me-backup.json');
    await tmpFile.writeAsString(payload);

    final result = await FilePicker.platform.saveFile(
      dialogTitle: 'Save Free4Me-IPTV backup',
      fileName: 'free4me-backup.json',
      bytes: tmpFile.readAsBytesSync(),
    );
```

Replace with:

```dart
    final stamp = exportStamp(DateTime.now()); // fix166
    final dir = await getTemporaryDirectory();
    final tmpFile = File('${dir.path}/free4me-backup-$stamp.json');
    await tmpFile.writeAsString(payload);

    final result = await FilePicker.platform.saveFile(
      dialogTitle: 'Save Free4Me-IPTV backup',
      fileName: 'free4me-backup-$stamp.json',
      bytes: tmpFile.readAsBytesSync(),
    );
```

> `exportStamp` is a static on the same class, so it's callable unqualified here.

## (c) Phone log — `lib/settings_view.dart`

In the "Export debug log" `ListTile`'s `onTap`, the non-TV branch calls `SettingsIo.exportStringToFile` with `suggestedName: 'free4me_log_${DateTime.now().millisecondsSinceEpoch}.txt'`. Replace that one `suggestedName` value:

```dart
                                suggestedName:
                                    'free4me_log-${SettingsIo.exportStamp(DateTime.now())}.txt',
```

## (d) TV backup + log — `lib/settings_view.dart`

In `_exportEverythingViaServer`, generate one stamp and use it for both items (so the TV backup + log are a matched pair). Current:

```dart
    final items = <ExportItem>[];
    final backup = await SettingsIo.buildBackupPayload(
        includeCredentials: includeCredentials);
    items.add(ExportItem(
      key: 'backup',
      filename: 'free4me-backup.json',
      label: 'Settings backup',
      bytes: utf8.encode(backup),
      contentType: 'application/json',
    ));
    if (settings.debugLogging) {
      final log = await AppLog.readLog();
      if (log.isNotEmpty) {
        items.add(ExportItem(
          key: 'log',
          filename:
              'free4me_log_${DateTime.now().millisecondsSinceEpoch}.txt',
          label: 'Debug log',
          bytes: utf8.encode(log),
          contentType: 'text/plain; charset=utf-8',
        ));
      }
    }
    if (!mounted) return;
    await _showExportServerDialog(items);
```

Replace with:

```dart
    final items = <ExportItem>[];
    // fix166: one instant shared by backup + log filenames AND the page label.
    final captured = DateTime.now();
    final stamp = SettingsIo.exportStamp(captured);
    final backup = await SettingsIo.buildBackupPayload(
        includeCredentials: includeCredentials);
    items.add(ExportItem(
      key: 'backup',
      filename: 'free4me-backup-$stamp.json',
      label: 'Settings backup',
      bytes: utf8.encode(backup),
      contentType: 'application/json',
    ));
    if (settings.debugLogging) {
      final log = await AppLog.readLog();
      if (log.isNotEmpty) {
        items.add(ExportItem(
          key: 'log',
          filename: 'free4me_log-$stamp.txt',
          label: 'Debug log',
          bytes: utf8.encode(log),
          contentType: 'text/plain; charset=utf-8',
        ));
      }
    }
    if (!mounted) return;
    // fix166: readable form for the index page (see 166.2).
    final capturedLabel = captured.toString().split('.').first;
    await _showExportServerDialog(items, capturedAt: capturedLabel);
```

**Key changes (166.1)**
- One helper, four sites: phone backup, phone log, TV backup, TV log all gain `…-yyyymmdd-HHMMSS` names.
- Per export action the backup + log share one stamp → obvious matched pair.
- Static names eliminated, so the browser stops appending `(1)…(6)`.
- No new imports; `exportStamp` is pure core Dart.
- `captured.toString().split('.').first` yields `2026-05-30 15:42:23` (drops microseconds) for the page label in 166.2.
