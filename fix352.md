## Why (from the 2026-06-11/12 S24 log series + provider API dumps)
1. **Oversubscribed multi-view grids are unwatchable and silent about why.** Dino's own API reports max_connections=1; a 4-cell 2×2 on it round-robins forever (each cell connect kills another cell's stream — 97 drops in 13.5 min). The provider limit has been captured into `sources.max_connections` on every Xtream refresh since fix184, but nothing ever used it. (Note: an earlier read of the 08:37 backup suggested it was never captured — wrong; that export simply predated the day's refreshes by one minute.)
2. **~25% of solo VideoController creations on the S24 register their platform texture late (>4s)** while decode + audio run fine. The fix338 4s hard fail then interrupted a healthy stream with a reconnect that accomplished nothing — the Z2U 09:14 occurrence proved it: the reconnect reused the SAME engine and controller, which attached on its own moments later. (Distinct from the fix337 concurrent-create race.)

## What this does
- **`lib/multi_view_screen.dart` — connection-limit warning.** After cell restore and after every explicit cell assignment, counts cells per source and compares against `sources.max_connections`. Oversubscribed → WARN log + 8s SnackBar, e.g. "Dino allows 1 connection — 4 cells will fight over it". De-duped per layout (`_lastConnWarning`); unknown/null limits warn nothing; single cells never warn.
- **`lib/player/mpv_engine.dart` — texture late-attach grace.** Replaces the one-shot 4s check: an id-notifier listener logs `texture attached latency=NNNms` whenever the texture lands (instrumentation directly comparable to the open Shield black-screen case); at 4s a WARN logs that grace is extending; only at **8s** still-null does it emit the restart error (message now says "8s after open"). Re-opens on an already-attached controller skip all of it; fix338's `_emittedError` gate retained; listener removal is dispose-safe.
- `lib/whats_new_modal.dart` — 1.29.0 changelog (2 entries). `pubspec.yaml` — 1.29.0+352.
- No DB schema or statement changes (the limit is already persisted by fix184) — Rule 8 N/A. Migration ceiling stays 27.
