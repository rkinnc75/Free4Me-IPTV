# fix132.md — Build break from fix130: stray @override on a non-interface method (override_on_non_overriding_member)

> **Version:** Free4Me-IPTV 1.22.11 (the fix130 build).
> **Evidence:** `flutter analyze --no-fatal-infos` on CI, exit code 1:
> ```
> warning • The method doesn't override an inherited method. Try updating this
> class to match the superclass, or removing the override annotation •
> lib/player/mpv_engine.dart:242:8 • override_on_non_overriding_member
> ```
>
> **Cause:** one of the methods fix130 added/edited in `mpv_engine.dart` carries
> an `@override` annotation but is NOT declared in the `PlayerEngine` interface
> (`lib/player/player_engine.dart`). WARNINGS are fatal to the pre-tag gate, so
> the build fails. The two `use_build_context_synchronously` INFOs (~1860/2420)
> are the pre-existing tolerated ones — leave them.

---

## Which method

`PlayerEngine` (the abstract interface) declares ONLY these overridable members:
`open`, `dispose`, `bufferingStream`, `completedStream`, `errorStream`,
`positionStream`, `position`, `supportsTrackSelection`, `subtitleTracks`,
`audioTracks`, `setSubtitleTrack`, `setAudioTrack`, `setVolume`,
`handlesOwnFullscreen`, `enterFullscreen`, `exitFullscreen`, `isFullscreen`.

The mpv-only helpers are NOT in the interface and must have **no** `@override`:
`onAdopt()` (fix130.2), `logSurface()` (fix130.4), `videoWidth`, `videoHeight`,
`reapplyOptions()`.

Line 242 is in the block fix130 rewrote. The offending method is almost
certainly **`logSurface()`** (newly added in fix130.4) or **`onAdopt()`**
(rewritten in fix130.2) — whichever ended up with an `@override` directly above
it at line ~242.

---

## fix132.1 — Remove the stray @override

**File:** `lib/player/mpv_engine.dart`

1. Open the file to **line 242** and look at the method immediately below the
   flagged `@override` (column 8 = the `@override` line itself).

2. **If the method is `logSurface`** — remove the `@override` above it:

   **Current:**
   ```dart
     @override
     void logSurface(String where) {
   ```
   **Replace with:**
   ```dart
     void logSurface(String where) {
   ```

3. **If the method is `onAdopt`** — remove the `@override` above it:

   **Current:**
   ```dart
     @override
     void onAdopt() {
   ```
   **Replace with:**
   ```dart
     void onAdopt() {
   ```

4. **If it is some other method at 242** (e.g. `videoWidth`/`videoHeight` picked
   up an `@override` during editing) — same action: delete the `@override` line
   directly above the method the analyzer points to.

### Verify

```
flutter analyze --no-fatal-infos
```

`override_on_non_overriding_member` must be GONE. Expected remaining output:
only the two tolerated `use_build_context_synchronously` INFOs at
settings_view.dart ~1860 / ~2420. Exit-code-wise the WARNING is cleared, so the
gate passes.

**Sanity grep** — every mpv-only helper should have NO `@override`:
```
grep -n -B1 "void onAdopt\|void logSurface\|int? get videoWidth\|int? get videoHeight\|Future<void> reapplyOptions" lib/player/mpv_engine.dart
```
None of those five should show an `@override` on the line above. The 17
interface methods (open, dispose, …, isFullscreen) SHOULD keep theirs.
