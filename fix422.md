# fix422 — remember the full-screen video fit (fit / stretch / crop)

**Version:** 1.35.20+421 → **1.35.21+422**
**Type:** Feature (small). Persisted setting.
**Tag:** v1.35.21
**Base:** fix421 (1.35.20+421). Apply fix419 → fix420 → fix421 → fix422 in order.

## What
The single-cell full-screen zoom mode (`ZoomMode` fit / stretch / crop, the fix404 tap-cycle) is now **persisted** and **restored**. Previously `_zoomMode` reset to `fit` on every player open; now it opens in the user's last choice.

- **Settings model** (`settings.dart`): new `ZoomMode playerZoomMode` (default `fit`).
- **Persistence** (`settings_service.dart`): new `playerZoomModeProp` saved/loaded via the existing key-value settings map, exactly like `devControlsHideSecs` — stored as the enum `.name`, parsed back with a safe `firstWhere(... orElse: fit)`.
- **Player** (`player.dart`):
  - `initState` restores `_zoomMode = widget.settings.playerZoomMode` and pushes it to the engine (`setZoomMode(boxFit)`) so the **first frame** renders in the saved fit (both the freshly-created and the adopted-engine swap paths).
  - `toggleZoom` writes the new mode back (`widget.settings.playerZoomMode = next` + `SettingsService.updateSettings`, fire-and-forget) so cycling persists immediately — restored within the same session and across restarts.

No migration needed (it's a key-value settings row, created on first save). `flutter analyze` clean, 166 tests pass.
