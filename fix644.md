# fix644 — TV guide: fast category->channels, Back returns to categories, 10k category cap

**Target tag:** `v2.2.43` (commit `b7116f4b8736f73170d7c5e6a74986d116c2fc97`)
**New version:** `2.2.44+633`
**Migration:** none

## Summary

Three changes from the onn `2.2.43+632` log (2026-07-03) + Rich's report:

1. **Category select took 15-26s to show channels (guide).** The channel query
   itself was 12ms. The stall was the EPG database open, two compounding bugs:
   - `EpgDbFactory.db` used `_db ??= await _createDB()` — an async race. Every
     caller arriving while the first open was in flight started ANOTHER full
     open (log: two "EpgDb: opened", three programme-count pairs). Fixed by
     memoizing the open FUTURE (`_dbFuture ??= _createDB()`); the now-unused
     `_db` field is removed (0-INFO analyze gate).
   - fix593's FTS self-heal ran two `COUNT(*)`s over ~2.8M programme rows plus
     the trigram FTS INSIDE the open — ~7-15s each on the onn — so the guide's
     first `getGridPrograms` sat behind them. The check is now fire-and-forget
     (`unawaited(_selfHealProgrammesFts(db))`) after the open returns. Worst
     case during the check window: EPG title search shelves briefly stale; the
     guide's NOW/NEXT grid reads the plain programmes table and is unaffected.

2. **Back from the channels rail tried to exit the app.** The guide's channel
   list is a rail-MODE swap (fix584), not a pushed route, so Back bubbled to
   TvShell's exit confirm. The guide build is now wrapped in a `PopScope`:
   in channels mode Back swaps to the categories rail and refocuses the
   selected category (identical to D-pad LEFT — the fix584 handler body is
   extracted to a shared `_swapToCategories()`); in categories mode Back
   bubbles normally so the shell's "Press Back again to exit" still works.

3. **Category caps raised 1,000 -> 10,000** in BOTH the guide rail
   (`_railCap`) and the Categories tab (`TvCategoriesView._cap`) — providers
   with huge category counts were silently truncated. Both lists are lazy
   (builder-based), so the raise costs nothing until categories exist.

1. Cold start -> Live TV -> pick a category: channels should appear in ~1-2s
   (log should show ONE "EpgDb: opened"; the programme-count line arrives
   later, after the grid is already up).
2. In the channels rail, press Back: categories rail returns, selected
   category focused (same as LEFT). Back again from categories: exit toast.
3. Categories tab + guide rail: providers with >1,000 categories now list all
   (up to 10,000).
