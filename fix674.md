# fix674 — Scheduled Recording alarm never registered (manifest components missing)

**Build:** `3.0.16+674` · **Base tag:** `v3.0.15` (fix671) · **Ship order:** after 671
**Files touched:** `android/app/src/main/AndroidManifest.xml`, `lib/whats_new_modal.dart`, `pubspec.yaml`
**Native/Kotlin/Dart logic:** none — manifest declaration only.

---

## Problem

On-device (SM-S938U, 3.0.15+671), a "Record now" created the `recordings` row (status `scheduled`, visible in the Recordings list) but **nothing ever recorded** — no foreground-service notification, no `Movies/Free4Me` directory, no file.

Report log (`free4me_log-sms938u-20260708-225120.txt`) pinpoints it:

```
[2026-07-08 22:46:15] [WARN] RecordingScheduler: alarm registration failed —
PlatformException(error, Component class
dev.fluttercommunity.plus.androidalarmmanager.RebootBroadcastReceiver does not
exist in me.free4me.iptv, ...
  at ...AlarmService.scheduleAlarm(...)
  at ...AndroidAlarmManagerPlugin.onMethodCall(...)
```

**Root cause:** fix667 added the `android_alarm_manager_plus` (^4.0.0) dependency **and its `<uses-permission>`s**, but never declared the plugin's three required `<application>` components. When `RecordingScheduler` calls `oneShotAt(..., rescheduleOnReboot: true)`, the plugin tries to `setComponentEnabledSetting` on its `RebootBroadcastReceiver` — which doesn't exist in the merged manifest — and throws. The alarm is never registered, so the callback never fires, so `RecordingCaptureService` (fix668) never starts. Every SR row is dead on arrival at `scheduled`.

This is a fix667 miss, not a capture-engine bug. The fix668+ capture path has never executed.

## Fix

Add the exact components the plugin's own README/manifest require (verified against the `android_alarm_manager_plus-v4.0.x` source, which matches the `^4.0.0` constraint): `AlarmService` (job service), `AlarmBroadcastReceiver`, and `RebootBroadcastReceiver` (ships `enabled="false"`; the plugin enables it at runtime only when `rescheduleOnReboot` is set — which is the exact call that threw). Names are fixed by the plugin; do not rename.

Also bumps the version and adds a changelog entry.
