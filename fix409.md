# fix409 — Developer setting: control-bar auto-hide timeout

**Target version:** `1.35.8+409` (from `1.35.7+408`)
**Branch:** `main` · **Base:** `1ebf8ad` (v1.35.7+408)
**Toolchain (verified):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.

## What & why

The player's top bar + bottom control row auto-hide after media_kit's default `controlsHoverDuration` (3 s); the app never overrode it. This adds a developer setting to tune that timeout, following the existing `dev*` settings pattern exactly.

- New `Settings.devControlsHideSecs` (int, default **3** = unchanged behaviour).
- Slider in the Developer section: **0–30 s**. **0 = keep until dismissed** (wired as a far-future `Duration` so media_kit never auto-hides).
- Wired into `_mpvThemeData` as `controlsHoverDuration`.
- Governs **both** bars together (they share one timer — media_kit gives no separate top/bottom control, by design).
