# fix331 — SelectDialog TV focus fix + black-screen surface diagnostic → v1.26.47+331

> ## WARNING BUILD MACHINE — APPLY THE PATCH, then RELEASE (EXACTLY). No questions, no DB to query.

## Why
Two Shield issues from the 22:59 log + user report:
1. **Selection dialogs broken on TV.** User: cycling player engines, the first option (Libmpv → ExoPlayer) "won't let me select it, doesn't even highlight." Cause in `select_dialog.dart`: only the FIRST item was wrapped in a bare `Focus(autofocus: true, …)`; on a D-pad TV that captured focus but the inner `ListTile` showed no highlight and couldn't be activated. Affects ALL 6 `SelectDialog` uses.
2. **Black screen, audio plays, every engine.** The SURFACE traces showed `textureId=null rect=null playerWH=nullxnull` at rotate-init and a real texture/rect ONLY at exit — so the surface state DURING playback was never captured. Can't fix blind; need the mid-playback reading first.

## What this does
**Part 1 — SelectDialog (real fix):**
- Every row is now an independently focusable, autofocus-aware `ListTile` (highlights + selects with the remote).
- New optional `selectedId`: autofocuses the CURRENT choice (so the dialog opens on it, not always row 0) and shows a check + highlight. Null = focus first row (unchanged for callers that don't pass it).
- Engine picker passes the active preference as `selectedId`.

**Part 2 — surface diagnostic (not a fix yet):**
- `mpv_engine.dart` probes the surface ~2s AFTER buffering clears (i.e. while playing), one-shot per engine, logging `SURFACE[mid-playback+2s] textureId=… rect=… playerWH=…`. This distinguishes: texture never attached (null) vs attached-but-zero-size (mpv reported no video dims) vs fine-surface-black-elsewhere. That reading determines the fix332 direction.
