# fix554 — Instrument the full TV-search wall-clock (find the Go→results lag)

**Version:** 2.0.54 (`2.0.54+554`)
**Base:** v2.0.53. Single file (`tv_search_view.dart`). Verified clean on a fresh
v2.0.53 clone.
**Branch from:** v2.0.53.

---

## Why

`Sql.search` (name matches) logs ~50ms, but the on-device "Go → results" wait is
several seconds. The TV search runs **three serial awaits per query** and only
the first was instrumented:
1. `Sql.search` (name matches) — logged.
2. `Sql.searchPrograms` (EPG "what's on", 234k programmes) — **invisible.**
3. `Sql.getLiveChannelsByEpg` — **invisible.**

Steps 2–3 are the prime suspects. This fix times all three + total wall-clock so
the field log pinpoints the cost before any optimization.

## What

Diagnostic-only. Wraps each phase in a `Stopwatch` and emits one line per query:

```
TvSearch._run: query="Yankees" total=NNNNms (name=NNms/36 epgProg=NNNNms/N epgCh=NNms/N)
```

No behavior change. (Also surfaces that search fires only via 250ms `onChanged`
debounce — no `onSubmitted` for the Go key — to confirm/deny in the next pass.)

## Verification

- `flutter analyze` → **No issues found!**
- `flutter test` → **259 passed**
- `git apply --check` → clean on fresh v2.0.53

## On-device

Run a few searches, then send the log. Read the `total=` and which phase
dominates (`epgProg` vs `epgCh` vs `name`).
