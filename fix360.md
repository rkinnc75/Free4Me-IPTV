## What this does
Builds on fix357's confirmed live DVR buffer (S25: pause/play works). Adds full transport when DVR is active.
1. **`player_engine.dart` / `mpv_engine.dart`** — expose `dvrActive` (default false on the interface; mpv returns `_dvrActive`). Live seeking is only offered when a DVR buffer truly exists.
2. **`player.dart`** — when `mediaType == livestream && _engine.dvrActive`, the centre button bar becomes: **rewind 10s · play/pause · forward 10s · back-to-live**. Without DVR, the live bar is unchanged (play/pause only, fix348). VOD unchanged.
   - `_dvrSeekBy(±10s)`: clamps to >= 0; never seeks past the live edge.
   - `_dvrGoLive()`: seeks far-forward (mpv clamps to the live edge of the disk cache) and resumes if paused.
   - **D-pad:** the four controls are real `IconButton`s ordered left->right, so remote LEFT/RIGHT traversal moves rewind -> play -> forward -> live naturally; CENTER activates. No custom focus handling needed.
