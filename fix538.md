# fix538 — TV: denser grids (~8/row) + categories cap 500→1000

**Base tag:** `v2.0.37`
**Version:** `2.0.37+537` → `2.0.38+538`
**Scope:** TV grid layout constants + poster-tile sizing. No schema/query change. Full suite green.

---

## Changes

1. **Categories screen cap 500 → 1000** (`tv_categories_view.dart` `_cap`). This is the Categories *management* screen's list cap — distinct from the Live/guide category *rail* cap (`_railCap`, already 1000 since fix532).
2. **~8 tiles per row (was ~4)** on both the Movies/Series poster grid (`tv_browse_view.dart`, `maxCrossAxisExtent` 180→90) and the Categories grid (`tv_categories_view.dart`, 220→110). `SliverGridDelegateWithMaxCrossAxisExtent` derives the column count from width ÷ extent, so halving the extent roughly doubles the columns on a 4K TV.
3. **Poster-tile elements scaled for the smaller tiles** (`channel_tile.dart`, poster mode — used only by those two TV grids): fallback glyph 40→28, favorite star 20→16, category checkbox 24→18, title 2 lines→1 line at fontSize 12→11 with tighter padding. Keeps the denser tiles clean instead of letting fixed-size overlays/text dominate.

No behavior change beyond layout; the poster body is shared but `poster: true` is used only by these two TV grids (verified).
