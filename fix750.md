> **fix749 was authored and then WITHDRAWN — do not ship it.** Adversarial review found it introduces a real regression: it removes fix747's blanket `!_enginePlaying` bail, so an **OS/audio-focus pause** (phone call, notification, another app taking focus) would freeze the position with no pause-intent flag set, and the watchdog would fire a **spurious reconnect on a live channel after 15s**. fix747 was immune to that only by accident. This fix ships **no behavior change at all** — it adds the measurements needed to design the real fix from evidence.

## 1. Why a diagnostic instead of a fix

The silent-stall arc keeps failing for one reason: **we keep reasoning from mpv's `playing` flag.**

- **fix747** assumed a silent stall keeps `playing=true`. The S938U wedge reports `playing=false` — so its watchdog bailed, *and* refreshed its own baseline every 3s, making it structurally incapable of ever firing. (ABC30 Fresno froze >2 min, log 20260714-102840.)
- **fix749** (mine) tried to fix that by keying on pause *intent* instead. But intent can't see an **OS-induced** pause, so it trades a missed stall for a spurious reconnect during a phone call. Rejected.
- A **cache-depth** discriminator was also proposed and **tested against the field log — it fails**: `demuxer-cache-duration` reads `0.000000` during the wedge *and* during ordinary `playing=false` samples. Depth cannot separate them.

Each attempt added another guess about *why* playback might legitimately stop. That list is unbounded (user, OS focus, Bluetooth, doze, media_kit internals…), and each missing entry is a new bug.

**The hypothesis worth testing instead:** don't ask *why playback stopped* — ask **whether the demuxer is still reading**. In every legitimate pause the demuxer is healthy (idle or filling); in a wedge it is starved. If true, that single signal replaces the entire pause-reason allowlist, and likely deletes state rather than adding it.

**It is a hypothesis, not a fact — and it is unverified.** The current STATS line has no demuxer-progress field, so the existing logs cannot prove or kill it. Hence this fix.

## 2. What this adds (measurement only)

Four properties to the existing debug STATS poll (`hwdec`, `avsync`, `cache`, … — same 1s tick, full-screen + debug only):

| Property | Why |
|---|---|
| `demuxer-cache-time` | PTS the demuxer has read **up to** — should keep advancing while paused, must go flat in a wedge |
| `cache-speed` | bytes/sec arriving — the direct "is data flowing" signal |
| `core-idle` | mpv's own view of "not actively playing" |
| `eof-reached` | distinguishes a wedge from a clean provider EOF |

The property map is built by zipping `keys[i]: values[i]` and read **by string key** in both consumers — verified, so inserting keys mid-list cannot corrupt existing fields. No watchdog, no reconnect, no player logic is touched.

## 6. Data collection plan (this is the point of the fix)

Enable the debug overlay, then capture **all four** scenarios in one log if possible. Each STATS line will now carry `demuxTime`, `cacheSpeed`, `coreIdle`, `eof`.

1. **Baseline — normal playback.** Any live channel, 30s. *(Expect: demuxTime advancing, cacheSpeed > 0.)*
2. **User pause.** Pause a live channel with the play/pause button, hold **60s+**, resume. *(The key control: does `demuxTime` keep advancing while paused? Does `cacheSpeed` stay non-zero? If it goes flat here, the hypothesis is DEAD and the redesign changes shape.)*
3. **OS/audio-focus pause.** While a live channel plays, trigger a focus steal — easiest is to **call the phone from another line**, or play a video in another app. Let it sit ~30s. *(The case fix749 would have broken. Must be distinguishable from #4.)*
4. **The wedge.** Reproduce a silent freeze (ABC30 Fresno, or whichever channel freezes — frozen frame, position stuck, no buffering spinner). Let it sit **60s+ without touching it** so several samples land. *(Expect, if the hypothesis holds: demuxTime FLAT + cacheSpeed 0.)*

Send the log. If #2/#3 show a healthy demuxer and #4 shows a dead one, the real fix is small and evidence-backed. If they look alike, the hypothesis dies here — cheaply — instead of in production.
