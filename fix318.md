## What this does
Adds an optional Android foreground service so a **source refresh** keeps running (with a progress notification) when the user switches away from the app, instead of being suspended. Gated by a new **"Keep refreshes running in background"** setting in Playback — **default OFF**. Phase 1 of the larger request; EPG refresh/rematch and stream scan will route through the same wrapper in a later fix once this is validated on-device.

## How it's built
- New dep `flutter_foreground_task: ^8.17.0` (resolves cleanly; no win32 conflict).
- `BackgroundTaskService.run(enabled, title, work)` — when enabled+Android, requests notification permission, starts the foreground service, runs `work` (which still executes on the MAIN isolate — no DB/media_kit/cache moved to a background isolate), mirrors progress to the notification, and always stops the service in `finally`. Best-effort: any failure (permission denied, non-Android, plugin error) falls back to running the work foreground-only — it never throws into the caller.
- `sources_refresh_dialog.dart` wraps `Utils.refreshAllSources` in `BackgroundTaskService.run`, reading `SettingsService.cached?.backgroundProcessing ?? false`. The in-app progress dialog is unchanged; the notification just mirrors it.
- Settings: `backgroundProcessing` field (default false) + serialization + backup round-trip + Playback toggle.
- Manifest: `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_DATA_SYNC`, `POST_NOTIFICATIONS`, `WAKE_LOCK`. (The `flutter_foreground_task` plugin contributes its own `<service>` via manifest merge — no manual service declaration needed.)

## Risk (read before releasing)
- **The native side is UNTESTED here** — the sandbox has no Android SDK, so only the Dart analyzes clean. Foreground-service startup, the Android-13+ notification permission prompt, manifest merge of the plugin's `<service>`, and UI/isolate re-attach on return are device-only behaviours. **Validate on a real phone before trusting it.** Because it's default-OFF and best-effort (falls back to foreground-only), a failure should degrade to current behaviour rather than break refresh.
- If the plugin's service fails to merge/build, the APK build will fail at the native step (not caught by `flutter analyze`). If that happens, the rollback is trivial: revert this fix (it's self-contained; nothing else depends on it).

+/// fix318: thin wrapper around flutter_foreground_task used to keep long
+/// operations (currently source refresh) alive when the user switches away
+/// from the app on Android.
+///
+/// Design notes:
+/// - The actual work keeps running on the MAIN isolate (we do NOT move the DB
+///   / media_kit / cache singletons into a background isolate). The foreground
+///   service exists only to promote the process so Android does not suspend it
+///   while backgrounded, and to show a progress notification.
+/// - Best-effort: if the service can't start (permission denied, non-Android,
+///   plugin error) the caller still runs the work in the foreground as before.
+///   Nothing here should throw into the caller.
+class BackgroundTaskService {
+  const BackgroundTaskService._();
+
+  static bool _inited = false;
+
+  static Future<void> _ensureInit() async {
+    if (_inited || !Platform.isAndroid) return;
+    FlutterForegroundTask.init(
+      androidNotificationOptions: AndroidNotificationOptions(
+        channelId: 'free4me_background',
+        channelName: 'Background processing',
+        channelDescription:
+            'Keeps refreshes running when the app is in the background.',
+        onlyAlertOnce: true,
+      ),
+      iosNotificationOptions: const IOSNotificationOptions(),
+      foregroundTaskOptions: ForegroundTaskOptions(
+        eventAction: ForegroundTaskEventAction.nothing(),
+        autoRunOnBoot: false,
+        allowWakeLock: true,
+        allowWifiLock: true,
+      ),
+    );
+    _inited = true;
+  }
+
+  /// Run [work] with a foreground service active for its duration when
+  /// [enabled] (the user's backgroundProcessing setting) is true on Android.
+  /// [title] is the notification title; [work] receives an `update(text)`
+  /// callback to refresh the notification body with progress. The service is
+  /// always stopped when [work] completes or throws.
+  static Future<T> run<T>({
+    required bool enabled,
+    required String title,
+    required Future<T> Function(void Function(String text) update) work,
+  }) async {
+    final useService = enabled && Platform.isAndroid;
+    if (!useService) {
+      // No service — just run the work (foreground-only, prior behaviour).
+      return work((_) {});
+    }
+    var started = false;
+    try {
+      await _ensureInit();
+      // Request notification permission (Android 13+); ignore the result —
+      // if denied, startService below simply won't show a notification.
+      try {
+        await FlutterForegroundTask.requestNotificationPermission();
+      } catch (_) {}
+      final res = await FlutterForegroundTask.startService(
+        notificationTitle: title,
+        notificationText: 'Starting…',
+      );
+      started = res is ServiceRequestSuccess;
+      if (!started) {
+        AppLog.warn('BackgroundTaskService: startService not successful — '
+            'running in foreground only');
+      } else {
+        AppLog.info('BackgroundTaskService: started — "$title"');
+      }
+    } catch (e) {
+      AppLog.warn('BackgroundTaskService: start failed ($e) — '
+          'running in foreground only');
+      started = false;
+    }
+
+    void update(String text) {
+      if (!started) return;
+      try {
+        FlutterForegroundTask.updateService(
+          notificationTitle: title,
+          notificationText: text,
+        );
+      } catch (_) {}
+    }
+
+    try {
+      return await work(update);
+    } finally {
+      if (started) {
+        try {
+          await FlutterForegroundTask.stopService();
+          AppLog.info('BackgroundTaskService: stopped — "$title"');
+        } catch (e) {
+          AppLog.warn('BackgroundTaskService: stopService failed — $e');
+        }
+      }
+    }
+  }
+}
