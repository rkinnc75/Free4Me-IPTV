## Why (from the 13:17 log, all on current main 1.26.37)
fix321 correctly preserved data, but the log exposed follow-ups bundled here:
1. **Retry success was discarded.** Z2U (13:06): the bulk live/vod/series calls timed out → counted empty → `failCount=3`; fix321 retried and the retries SUCCEEDED (full dumps written), but `failCount` was never decremented, so the old `failCount>=3` check **threw and aborted the commit**, throwing away recovered data.
2. **Preserved-everything still threw a hard error.** When all three types were empty but had prior data, fix321 preserved them (good) yet the refresh still surfaced `all content types failed (3/3)` to the user.
3. **Background service didn't start on the device** — `BackgroundTaskService: startService not successful` with no reason (this is why Emjay's backgrounded movies came back empty: the service didn't hold the process).
4. **Cache-build query logged SLOW** on large catalogues (per-row correlated `hide_dividers` subquery).
5. **Exports from different devices collide** (same `free4me-…-<timestamp>` name).

## What this does
1. **Retry reconciliation (xtream.dart):** when a fix321 retry recovers a type, decrement `failCount`; only throw when `failCount>=3` **AND** nothing was preserved (genuinely dead/new source). A recovered-on-retry type now commits instead of being discarded.
2. **No false failure when preserved:** if empties were preserved from prior data, log it and keep the catalogue — no exception.
3. **Background diagnostics (background_task_service.dart):** log the notification-permission result and the `startService` result object so the failure cause is visible; clearer "won't survive backgrounding" warning. (Still best-effort; this is diagnosis, not a native rewrite.)
4. **Cache query speedup (sql.dart):** `getAllChannelNamesForCache` uses `LEFT JOIN sources` instead of a per-row correlated subquery. Identical results (verified), faster on huge catalogues.
5. **Device-tagged exports:** `DeviceDetector.deviceTag()` (filename-safe model, e.g. `onn4kplus`, `shieldandroidtv`) + `SettingsIo.stampWithDevice()`; backup/settings/log/zip export names become `…-<deviceTag>-<timestamp>`.
