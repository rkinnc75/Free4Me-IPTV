# fix378 — Categories view groups by source when sort_mode is `provider`

**Target version:** `1.34.7+378` (from `1.34.6+377`)
**Branch:** `main`
**Toolchain (verified in sandbox):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.
**Written against:** `v1.34.6+377` on main.

## What & why

The Categories home view (`Sql.searchGroup` in `lib/backend/sql.dart`)
had a hard-coded `ORDER BY` that always sorted categories A–Z
after the favorites/enabled tiers. It ignored the user-chosen sort
mode set in Edit Source → Channel order (`alpha` / `provider` /
`category`). With multiple provider-ordered sources, a user's enabled
categories appeared in one big A–Z list rather than grouped by source —
different from the Favorites view (fix377) and the rest of search.

fix378 makes `searchGroup` route the `provider` mode through a
source-grouped ORDER BY (favorites → enabled → source name A–Z →
category name A–Z), matching the source-name grouping used in
Favorites (fix377). The `alpha` and `category` modes (and the
null/mixed fallback) keep the existing A–Z form: there's no natural
"category" order for a list of categories, and the no-op for
null/mixed avoids forcing a correlated subquery on a rare path.

## Files changed
- `lib/backend/sql.dart` — `searchGroup` now resolves
  `_uniformSortMode(filters.sourceIds!)` and, when it returns
  `'provider'`, emits a source-grouped ORDER BY. Other modes
  (`alpha`, `category`, null) keep the existing A–Z form.
- `lib/whats_new_modal.dart` — `'1.34.7'` changelog entry (real UTF-8).
- `pubspec.yaml` — `1.34.6+377` → `1.34.7+378`.
- `test/sql_searchgroup_sort_mode_test.dart` — **new**: 4 real-`Sqlite`
  tests seeding a real DB with two sources and asserting the EXACT
  row order produced by the emitted SQL for each branch.

## On-device integration check (not sandbox-provable)
- Single source, any sort_mode: Categories view should look identical
  to before (alpha is the unchanged default; provider with one source
  groups by source A–Z which has only one entry, so functionally the
  same as A–Z).
- Two sources, **both** on `Provider order`: Categories view should
  now interleave categories by source name — `Alpha` source
  categories before `Beta` source categories, name A–Z within each
  source. Compare to non-favorites browse in Provider mode: should
  share the same source-grouped shape.
- Two sources, one on `Provider order` and one on `Alphabetical`:
  null/mixed fallback. Categories should keep the existing A–Z
  behavior (no source grouping). Compare to before fix378: identical.
- One source on `By category`: A–Z within each tier, unchanged
  (categories are the grouping already).
- Enabled/disabled tiles: behavior unchanged. Disabled categories
  still appear grayed-out, sorted below enabled ones within each
  source or globally depending on mode.

+bool _sqliteAvailable() {
+  try {
+    s3.sqlite3.openInMemory().dispose();
+  } catch (_) {
+    return false;
+  }
+  return true;

+  // Two sources so we can exercise the "group by source" branch
+  // (provider uniformMode case). The seed also includes favorite and
+  // enabled columns so the favorites/enabled tiers can be tested.
+  void seed(s3.Database db) {
+    db.execute('''
+      CREATE TABLE sources(id INTEGER PRIMARY KEY, name TEXT, sort_mode TEXT);
+      CREATE TABLE groups(
+        id INTEGER PRIMARY KEY,
+        name TEXT,
+        image TEXT,
+        source_id INTEGER,
+        media_type INTEGER,
+        favorite INTEGER,
+        enabled INTEGER
+      );
+    ''');
+    // Both sources are 'provider' mode for the provider-order tests.
+    db.execute("INSERT INTO sources VALUES (1, 'Beta', 'provider')");
+    db.execute("INSERT INTO sources VALUES (2, 'Alpha', 'provider')");
+    final ins = db.prepare(
+        'INSERT INTO groups VALUES (?,?,?,?,?,?,?)');

+    // id, name, image, source_id, media_type, favorite, enabled

+    ins.execute([2, 'Sports',   null, 2, null, 0, 1]);

+        List.filled(sourceIds.length, '?').join(',');

+      SELECT id, name FROM groups
+      WHERE (name LIKE ?)
+        AND (media_type IS NULL OR media_type IN (?, ?, ?))
+        AND source_id IN ($placeholders)
+        $orderByTail
+      LIMIT 0, 100''';
+  }
