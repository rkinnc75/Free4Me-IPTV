# fix597 — Live guide redesign: swapping rail + 70% preview + full-grid EPG

**Version:** v2.2.8+597
**Date:** 2026-06-28
**Type:** TV UX redesign (replaces the fix584/585 rail-collapse)

## What changed (`lib/tv/tv_guide_view.dart`)

Replaces the collapsing-rail Live guide with the user's layout:
- **Top band:** a 70%-size **preview** (top-left) + a **channel-info** band to its
  right (name + NOW/NEXT).
- **Below:** a single **rail** (left, 210 px) + a **full EPG grid** (right) with NO
  channel-name column — the names live in the rail.
- **Rail swaps** (`enum RailMode {categories, channels}`):
  - *categories* (default): the live category list. Moving the highlight
    **follows** the grid to that category's channels (debounced 280 ms; the
    preview stays on the last-watched channel — no engine churn). OK → channels.
  - *channels*: the selected category's channel names. Focusing one
    **highlights + scrolls** its row in the passive grid (align-on-focus, not a
    2nd synced controller) and arms the muted preview + info. OK plays it.
    **LEFT** → back to categories, restoring focus to the selected category.

## Focus model (every fix585-class trap closed)

- Mode swaps use **explicit `requestFocus`** on pre-created nodes (immediate +
  post-frame backstop after the `setState`-scheduled frame) — never autofocus
  re-fire, never `focusInDirection`.
- LEFT is **always handled** with a Favorites/first-category fallback (never
  strands), and is a no-op outside channels mode.
- `_onChannelFocused` is gated to channels mode so a stale event can't drive the
  preview while browsing categories.
- Channel/category FocusNodes persist across swaps so `requestFocus` lands on
  rebuild. Removed `_railCollapsed` + the `AnimatedContainer`/`OverflowBox`.
