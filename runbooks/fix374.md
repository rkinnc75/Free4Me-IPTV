# fix374 — Redact source credentials in the debug log

**Target version:** `1.34.3+374` (from `1.34.2+373`)
**Branch:** `main`

## Summary

The debug log can contain provider usernames/passwords (Xtream credential
fields, and the same literals embedded in stream/EPG URLs). When a log is shared
for troubleshooting, those credentials leak.

This adds a debug setting **"Log User/Pass"** (default **OFF**). With it OFF,
`AppLogger.log()` replaces each source's username/password literal with a
labelled token built from the source name with spaces stripped — e.g.
`<A3000_USER>` / `<Dino_PASS>` — before writing to console and file. With it ON,
the log is written verbatim (developer's own testing).

The redaction table is built from **all** sources regardless of enabled state,
via `getSources()` (the read chokepoint hit at startup and after every
add/edit/delete/import), plus an explicit startup call in `main()`.

## Step 4 — `lib/backend/app_logger.dart` : add the redaction methods

FIND:
```

  void warn(String message) => log(message, level: LogLevel.warning);
  void error(String message) => log(message, level: LogLevel.error);
```

REPLACE:
```
  void info(String message) => log(message, level: LogLevel.info);
  void warn(String message) => log(message, level: LogLevel.warning);
  void error(String message) => log(message, level: LogLevel.error);

  /// fix374: rebuild the redaction table from [sources]. Checks EVERY source
  /// regardless of enabled state. Empty/whitespace credentials are skipped (an
  /// empty secret would otherwise match everywhere). The source name has spaces
  /// stripped to form the token tag, e.g. "A 3000" -> <A3000_USER>.
  void setSourceSecrets(List<Source> sources) {
    final out = <MapEntry<String, String>>[];
    for (final s in sources) {
      final tag = s.name.replaceAll(' ', '');
      final u = s.username ?? '';
      final p = s.password ?? '';
      if (u.trim().isNotEmpty) out.add(MapEntry(u, '<${tag}_USER>'));
      if (p.trim().isNotEmpty) out.add(MapEntry(p, '<${tag}_PASS>'));
    }
    out.sort((a, b) => b.key.length.compareTo(a.key.length));
    _secrets = out;
  }

  /// fix374: replace every known credential literal in [s] with its token.
  /// String.replaceAll uses literal (non-regex) matching for a String pattern,
  /// so credential values containing regex metacharacters are handled safely.
  String _redactSecrets(String s) {
    if (_secrets.isEmpty) return s;
    for (final e in _secrets) {
      if (s.contains(e.key)) s = s.replaceAll(e.key, e.value);
    }
    return s;
  }
```

## Notes / scope

- Redaction matches the `username`/`password` **fields** of every source. For
  Xtream sources those literals also appear inside stream/EPG URLs and are
  redacted there too (same literal). M3U sources that embed credentials only in
  a URL query string (no parsed username/password) are out of scope for this fix.
- Tokens derive from the source name with spaces stripped. Two names that
  collapse to the same tag share a token — accepted.
- Empty/whitespace credentials are skipped so an empty secret never matches
  everywhere. Longest-secret-first ordering avoids partial-overlap clobbering.
