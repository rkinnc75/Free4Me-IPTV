# fix500 — TV tabbed shell + Default-View landing routing

**Version:** 1.35.22+423 → **2.0.0+500**
**Tag:** v2.0.0
**Type:** Feature (large). First fix of the **2.x TV-mode redesign** arc.
**Status:** Patch authored below (real code) — pending CI analyze + release verification. NOT locally verified: no flutter/dart on the authoring box.
**Base:** current `origin/main`. New major version + new fix sequence (build 500-series; for 2.x, version patch = build − 500).

## The arc (fix500 → fix503)
A ground-up TV-mode UI to replace the two-level giant-button menu. Each fix ships as a complete release.
- **fix500** — tabbed shell + Default-View routing *(this file)*
- **fix501** — Live category rail + per-source left-flush edge bar
- **fix502** — EPG title search + `epgSearchHours` + FTS5 (epg.sqlite)
- **fix503** — net-new EPG guide grid (channel × time), perf-gated

**Hard constraint across all four:** phone (touch) mode must not regress. TV changes are confined to `tv_home.dart` + new `lib/tv/` widgets; shared backend changes are additive + back-compatible. Perf NFR: every user-visible **read** query < 1000 ms **cold on the slowest device** (cheap Android TV box / Fire TV Stick).

## What
Replace `lib/tv_home.dart` (the TV entry — `main.dart:182` returns bare `TvHome()` when `forceTVMode || isTV || (!hasTouchScreen && Android/iOS)`) with a persistent top tab bar: **Live TV · Movies · Series · Favorites · History · Search** + a Settings gear. Each tab hosts the existing `Home`/`HomeManager` body via a per-tab `Filters`, with `hasTouchScreen:false`. The landing tab is chosen from `settings.defaultView`.

## Phone-mode safety
- `tv_home.dart` is the TV entry and is **never** mounted on phone (`main.dart:182` vs the phone branch `main.dart:183-189`). Rewriting it cannot affect phone.
- New shell widgets live under a new `lib/tv/` dir.
- `Home` is hosted unchanged with `hasTouchScreen:false`, so `BottomNav` stays suppressed (`home.dart:914-940`). **Do not edit `bottom_nav.dart`** — copy its brand-color consts (`bottom_nav.dart:7-43`) into a shared const if needed, never reuse the widget.
- `main.dart` TV branch gains a `settings` arg into the shell; the phone else-branch (`:187`) is **not** touched. No `Filters`/`HomeManager` model edits.

## Migration
**None.** Pure widget/navigation work.

## Perf gate
No new query. The one risk: **lazily** build/keep-alive tab bodies so launch fires **one** cold `Home.load → Sql.search` (`home.dart:259-262`), not six. Verify a single cold catalogue scan at launch on the slowest box.

## Touch-points (verified)
- `lib/tv_home.dart` — **rewrite**. Replace `getHome`/`navNested`/`navigateHome`/`getMediaTypeSelectNested` (`:53-68,153-207`) with the tabbed shell. **Re-host `runVersionStartupTasks(context)`** (the `!nested` hook, `:31-35`, fix403 What's New + log rotation) at the shell root.
- `lib/main.dart:177-190` — TV branch: pass `settings` into the shell for landing selection. Phone branch untouched.
- **NEW** `lib/tv/tv_shell.dart` — scaffold + per-tab `Filters` (Live=`mediaTypes:[livestream]`(+`groupId` later); Movies=`[movie]`; Series=`[serie]`; Favorites/History=`viewType`) + defaultView→landing map. Route `searchMethod` from `SettingsService.cached` (not the `Filters` `inMemory` default) to match phone (`home.dart:228-232`).
- **NEW** `lib/tv/tv_top_tab_bar.dart` — the tab bar; imports copied brand-color consts.
- `lib/settings_view.dart:624-645` — read-only reference: `_showDefaultViewDialog` exposes `ViewType.values.take(4)` = {all, categories, favorites, history}. No edit.
- `lib/models/filters.dart` — read-only (already carries `mediaTypes`+`groupId`+`viewType`+`copy()`). Optional: fix stale doc-comment `:15`.

## Decision — defaultView → landing tab (resolved)
`ViewType` enum (`view_type.dart:1`) has no livestream/movie/serie member, so the design's wording maps onto Filters as:
| `settings.defaultView` | Landing |
|---|---|
| `all` | Live TV (guide; channel list until fix503) |
| `categories` | Live TV, category rail focused (rail lands fix501) |
| `favorites` | Favorites tab (`ViewType.favorites`) |
| `history` | History tab (`ViewType.history`) |

`ViewType.settings` excluded (gear is separate).

## Approach
1. Build the shell scaffold: top tab bar + a body host that **lazily builds + keep-alives** one `Home` per tab from per-tab `Filters`.
2. Implement the defaultView→tab map above; pick the initial tab from `settings.defaultView`.
3. Re-host `runVersionStartupTasks` at the shell root (non-nested) so TV keeps What's New + log rotation.
4. Wire `main.dart` TV branch to pass `settings`.
5. Settings gear keeps pushing `SettingsView(showNavBar:false)` for now (TV Settings screen arrives in fix502).
6. Add a `whats_new_modal.dart` entry for 2.0.0.

## Tests
- `flutter analyze` = 0 issues; `flutter test` passes.
- New: widget test that the shell mounts the correct landing tab for each of the 4 defaultView values; that exactly one `Home` body loads at launch (lazy tabs).
- **Phone regression:** assert `main.dart` phone else-branch unchanged; `Home(hasTouchScreen:true)` still builds `BottomNav` (`home.dart:914`); existing phone tests green.

## Risks
- defaultView→tab is net-new (no TV codepath reads it today) — wrong mapping is the likeliest regression.
- Eager tab instantiation → six cold queries at launch. Use lazy/keep-alive bodies.
- Dropping `runVersionStartupTasks` in the rewrite loses TV What's New + log rotation — re-host it.
- Orphaned old nav paths (`navNested`/`navigateHome`) — re-wire or remove (TV-only breakage risk).
