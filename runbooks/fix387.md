# fix387 — Edit Source: renaming a source now saves (Sql.updateSource omitted `name = ?`)

**Target version:** `1.34.16+387` (from `1.34.15+386`)
**Branch:** `main`
**Toolchain (verified in sandbox):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.
**Written against:** `v1.34.15+386` on main (commit `894520d`).

## What & why

**Symptom.** In the Edit Source dialog (made name-editable in fix385), changing the source name and saving does nothing — the old name persists. (Reported for name; analysis below shows it is name-only.)

**Root cause.** The dialog side is correct: on Save it reads `_nameController.text.trim()` and passes it as `Source.name` into `Sql.updateSource` (`lib/edit_dialog.dart`). `Sql.updateSource` then runs an UPDATE whose SET clause is:

    SET url = ?, username = ?, password = ?, max_connections = ?, color = ?,
        sort_mode = ?, last_live_count = ?, last_movie_count = ?,
        last_series_count = ?, hide_dividers = ?
    WHERE id = ?

There is **no `name = ?`** in the SET clause. The statement succeeds and updates every other column, but never writes `name`, so the rename is silently discarded. `updateSource` predates editable names — before fix385 the name was read-only, so the UPDATE never needed it; fix385 added the editable field and dialog plumbing but did not extend the SET clause.

**Scope — name only.** Every other field the dialog edits is present in the SET clause and saves correctly: `url`, `username`, `password`, `color` (fix385/#4), `sort_mode`, `hide_dividers`, `max_connections`. `enabled` / `epg_url` / `epg_discovery_state` are intentionally not edited in this dialog and are carried through untouched (correct — `updateSource` not writing them preserves them). So only `name` was affected.

**Fix.** Add `name = ?` as the first column in the SET clause and bind `source.name` first. The statement is extracted to a `Sql.updateSourceSql` const so the regression test executes the *exact* statement the app runs (not a copy that can drift); a future change that drops a SET column then changes the bind/placeholder count and fails the test.

## Files changed
- `lib/backend/sql.dart` — extract the UPDATE to `static const String updateSourceSql` with `name = ?` added at the front of the SET clause; `updateSource` binds `source.name` first. No other column or bind order changed.
- `test/update_source_name_test.dart` — **new** (2 tests): runs `Sql.updateSourceSql` against a seeded in-memory sqlite DB and asserts the rename of source id 7 persists (`name` → "New Name") alongside url/color, while sibling id 8 is untouched; plus a guard that `Sql.updateSourceSql` contains `name = ?`.
- `lib/whats_new_modal.dart` — `1.34.16` changelog entry.
- `pubspec.yaml` — `1.34.15+386` → `1.34.16+387`.

+// fix387: Sql.updateSource silently dropped the name. fix385 made the Edit
+// Source dialog's name field editable and the dialog passes the new name into
+// Sql.updateSource, but the UPDATE's SET clause omitted `name = ?`, so renames
+// were never persisted (every other editable field WAS persisted).
+//
+// Rule-8 test: it executes the EXACT statement the app runs — the canonical
+// `Sql.updateSourceSql` const that Sql.updateSource binds against — with the
+// same bind order, against a seeded sqlite DB, and asserts the rename lands.
+// Because it runs the real const, a future regression that removes a SET
+// column changes the placeholder count (→ sqlite bind error) or the value
+// (→ assertion failure), so the test fails instead of silently passing.
+import 'package:flutter_test/flutter_test.dart';
+import 'package:sqlite3/sqlite3.dart' as s3;
+import 'package:open_tv/backend/sql.dart';
+
+bool _sqliteAvailable() {
+  try {
+    s3.sqlite3.openInMemory().dispose();
+    return true;
+  } catch (_) {
+    return false;
+  }
+}
+
+s3.Database _seed() {
+  final db = s3.sqlite3.openInMemory();
+  // Column order matches the production sources table for the columns the
+  // UPDATE touches; positional INSERT below sets them.
+  db.execute('''
+    CREATE TABLE sources(
+      id INTEGER PRIMARY KEY,
+      name TEXT,
+      url TEXT,
+      username TEXT,
+      password TEXT,
+      max_connections INTEGER,
+      color INTEGER,
+      sort_mode TEXT,
+      last_live_count INTEGER,
+      last_movie_count INTEGER,
+      last_series_count INTEGER,
+      hide_dividers INTEGER
+    )''');
+  // Target row (id 7) + a sibling control (id 8) that must stay untouched.
+  db.execute(
+    "INSERT INTO sources VALUES (7,'Old Name','http://old','u','p',1,0,'provider',10,20,30,0)",
+  );
+  db.execute(
+    "INSERT INTO sources VALUES (8,'Sibling','http://sib','u2','p2',1,0,'category',1,2,3,1)",
+  );
+  return db;
+}
+
+// Same bind order as Sql.updateSource.
+List<Object?> _binds({
+  required String name,
+  required String url,
+  required int color,
+  required int id,
+}) =>
+    [name, url, 'u', 'p', 1, color, 'provider', 10, 20, 30, 0, id];
+
+void main() {
+  group('Sql.updateSource persists the name (fix387)', () {
