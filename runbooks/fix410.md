# fix410 — Disable Impeller (Skia/GL compositing) — NVIDIA Shield black-screen DIAGNOSTIC

**Version:** 1.35.8+409 → **1.35.9+410**
**Type:** ⚠️ DIAGNOSTIC build — single-variable test. One manifest flag. No Dart logic change.
**Tag:** v1.35.9

## What & why (one screen)
- Shield (Tegra X1 / NVIDIA 495.00) renders a **black screen** on EPIX HD with **both** HW (`mediacodec-copy`/nv12) and SW (`hwdec=no`/yuv420p) decode. Decode/audio/clock are healthy in every log — fault is **downstream, in GPU compositing**.
- Same +409, same channel, the **S24 (Adreno) plays** the *identical* render path (`vo=gpu` GLES → Flutter `SurfaceProducer` surface → Impeller/Vulkan). Cross-device control isolates the fault to the **GLES-producer → Impeller/Vulkan-consumer** interop on the Tegra driver.
- This build sets `io.flutter.embedding.android.EnableImpeller=false` → Flutter composites with **Skia/GL**, so producer AND consumer are both GL and the GL↔Vulkan interop disappears.
- **Expected result split:** Shield no longer black → Tegra GL↔Vulkan interop confirmed as root cause. Still black → Shield was already on Skia (Tegra weak Vulkan) and mpv's GL output never reaches the surface → deeper media_kit SurfaceProducer issue, different fix.
- **Tradeoff:** the flag is **app-global** (Impeller is chosen at engine init, can't be per-device at runtime). Skia/GL is fully supported in Flutter 3.44. Test other devices (S24, Pixel, onn) for any regression before keeping.
