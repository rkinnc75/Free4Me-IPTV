# fix578 — TV player Mode B: D-pad navigation of the control bars

**Version:** v2.1.5+578
**Date:** 2026-06-27
**Type:** TV/remote UX (player) — completes the agreed full D-pad model

## Context

fix576/fix577 gave the player **direct-map** D-pad (▲▼ channel, ◀▶ seek, OK =
play-pause + reveal bars), verified on-device. The agreed full model adds **Mode B**:
once the bars are up, the D-pad navigates the bar buttons (Cast / PiP / Subtitles /
Audio / Aspect / transport) and OK activates them.

## Why media_kit's bars need help

media_kit 2.0.1's controls auto-hide on a `_timer` that resets only on tap / mouse
hover — **never on key or focus activity** (material.dart:690), and `onTap()` toggles.
So on a TV the bars would vanish mid-navigation. The user chose to navigate the real
bars anyway, so fix578 disables auto-hide while navigating.

## Change (all in player.dart)

A `_barsNav` flag drives a mode switch:

- **Enter (OK in direct-map):** `_togglePlayPause()` + `_enterBarsNav()` —
  `setState(_barsNav=true)` (so the theme rebuilds with a days-long
  `controlsHoverDuration` → no auto-hide), then on the next frame synthesize the
  reveal tap (arms media_kit's hide timer with the new long duration) and
  `FocusScope.of(context).nextFocus()` to push focus into the bar buttons.
- **In nav mode:** `_onPlayerKey` returns `ignored` for everything, so Flutter's
  default directional focus traversal moves between the (focusable) bar `IconButton`s
  and Enter/OK activates the focused one.
- **Exit (Back):** the `PopScope` handler calls `_exitBarsNav()` when `_barsNav` —
  toggles the bars off, restores auto-hide, and `requestFocus()`es the player so the
  D-pad direct-maps again — instead of leaving the player.

## Risk / verification

The unverified piece is whether media_kit's bar buttons are reachable + directionally
traversable by `nextFocus()` / arrow keys. That is only testable on a shipped build
(no local keystore). **On-device (post-ship):** `adb install -r`, tune from the guide,
OK → bars stay up, D-pad moves the yellow focus ring between Cast/PiP/Subtitles/Audio/
Aspect, OK activates, Back returns to channel control. If traversal does not reach the
buttons, the fallback is a custom focusable overlay (Option B/C from the scoping
question) — fix579.

  (`playerKeyAction`) is unchanged and still covered by `test/fix576_player_key_action_test.dart`.
