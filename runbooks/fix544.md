# fix544 — Live TV shows stale channels after changing enabled sources

**Base:** apply AFTER fix543 (this stacks on it; both touch `tv_guide_view.dart`).
**Version:** `2.0.43+543` → `2.0.44+544`
**Scope:** one-line behavior fix in the Live guide's reload entry point + changelog. Full suite green.

> ⚠️ Apply order: **fix543 then fix544.** fix544's patch is generated on top of fix543. If fix543 has already landed/tagged, branch fix544 from that tag.

---

## Problem (from device report)

Disabling ALL sources correctly emptied Movies, Series, and Categories — but **Live TV still showed channels (stale)**.

Cause: the shell calls `TvGuideViewState.reloadGuide()` when the Live tab is re-selected (the guide is kept alive by the shell's `IndexedStack`, so `initState` does not re-run). `reloadGuide()` called only `_loadGuide(null)`, which re-queries channels but **reuses the cached `_sourceIds` and `_groups` (the category rail) captured at the last `_init()`**. So after the enabled-source set changed, the guide re-queried with the STALE source list and kept showing channels. The other tabs rebuild fully (fresh `_reloadGen` key), so they updated correctly.

## Fix

`reloadGuide()` now re-runs `_init()`, which re-reads `getEnabledSourcesMinimal()`, rebuilds the category rail, and then loads the guide — so the enabled-source set and rail are always current on re-entry. With all sources disabled the guide correctly goes empty (`source_id IN ()` returns no rows in SQLite — no error).
