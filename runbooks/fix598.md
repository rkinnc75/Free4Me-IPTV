## Focus bugs (fix597 redesign) — fixed

On-device testing found: OK-into-channels didn't land focus on the top channel,
and LEFT returned to the WRONG category. Root cause: fix597 used cross-rebuild
`requestFocus` on nodes detached during the rail's mode swap — fragile (the
fix585 class). Replaced with the **proven autofocus-on-remount** pattern:

- The rail `ListView` is keyed per mode (`'rail-channels'` / `'rail-categories'`),
  so a categories↔channels swap **remounts** the list.
- The target tile uses `autofocus: true` — first channel in channels mode, the
  **selected** category in categories mode — which fires reliably on that fresh
  mount.
- `_loadGuide` just sets `_railMode` (the remount + autofocus do the focusing);
  `_onChannelLeftKey` just swaps to categories (selected-category autofocus
  restores focus). All the cross-rebuild `requestFocus`/post-frame code removed.
- Entering channels, the first channel's `autofocus` also fires `onFocusGained`
  → arms the preview (on multi-connection sources) + highlights the grid.
