# fix650 — TV: control overlay for VOD + progress row (v2.2.50)

**Builder:** desktop, applying the phone coder's work order `fix649.md`
(renumbered per VERIFY_643-647 protocol). **Live base:** v2.2.49.
**Ships as:** v2.2.50+639.

## Problem (from the work order; evidence captured in VERIFY §6)

On TV, OK during VOD playback revealed media_kit's own bars (fix577 synth
tap) — visible but NOT D-pad focusable (the reverted fix578 problem), so a
remote could see controls it couldn't press. Live got the custom focusable
overlay in fix580; VOD never did. No position/length shown anywhere.

## Change (applied verbatim, markers renumbered fix649→fix650)

- `_onPlayerKey` playPauseReveal: drops the `mediaType == livestream` gate —
  TV single-cell opens the custom focusable overlay for VOD too; phone
  unchanged.
- `_buildTvOverlay`: seek buttons gate on `_canSeekTransport` (DVR or VOD);
  `Back to live` stays DVR-only; now/next EPG label only on live (spacer on
  VOD).
- New `_buildOverlayProgress()`: display-only `position ── bar ── duration`
  row above the buttons, `positionStream`-driven, hidden when the engine
  reports no duration (plain live). Display-only by design — a focusable
  slider would steal D-pad LEFT/RIGHT from the overlay traversal.

## Verify on device (onn)

1. Movie on TV, press OK: playback toggles AND the dark overlay appears
   (subtitles/audio, rewind/forward, play-pause, aspect, mini-player — all
   focusable) + `mm:ss ── bar ── h:mm:ss` row. No `Back to live`, no ▲/▼
   (not surfable), no EPG label.
2. D-pad traverses; OK activates; Back closes overlay first, second Back
   exits playback (fix580 semantics).
3. Live+DVR: unchanged except the progress row. Plain live: NO progress row.
4. Overlay auto-hides ~8 s idle.
