## Source
Cold-eyes review of 1.31.2+359. CRIT-1 reproduced by execution and re-confirmed against current main before fixing — this is a regression I introduced in fix355.

## CRIT-1 — every series view is empty
fix355 added `AND series_id IS NULL` to the three `Sql.search` query paths to keep episodes out of Movies. But that line went into the **static WHERE base**, while the series view appends `AND series_id = ?` afterwards — producing `... series_id IS NULL ... AND series_id = <id>`, never satisfiable. A series view uses an empty query → the SQL branch → 0 episodes on every device, every search method. fix353's test only checked the table-level upsert, never `Sql.search` with a seriesId, so it passed.
**Fix:** the episode exclusion is now **conditional** in all three paths (FTS-match base, no-query base, `_searchLike`): the `seriesId != null` branch emits only `series_id = ?` (series view → its episodes); the `else` branch emits `series_id IS NULL` (everything else → episodes excluded). The two can never AND together again.

## HIGH-1 — episodes still leak into search on the default path
fix355 missed `ChannelSearchCache` — the **default** (`SearchMethod.inMemory`) path. Its filter excluded episodes only when *inside* a series; typing a query in Movies/All/Favorites surfaced matching episodes.
**Fix:** symmetric guard in `ChannelSearchCache.search` — `if (seriesId == null && e.seriesId != null) continue;`.

## Regression coverage (the gap that let this ship)
New `test/series_search_query_test.dart` exercises the emitted WHERE semantics directly: series view returns that series' episodes; movies browse and movies SEARCH exclude episodes; and the in-memory cache filter is symmetric (episode-in-series kept, episode-in-movies dropped). +3 tests (20 → 23).

## Rule 8 (executed)
Seeded DB (series parent + 2 episodes + 2 movies): series-view query → 2 episodes; movies browse → 2 movies (no episodes); movies LIKE-search → episodes excluded; cache keep-logic verified for all four cases. ALL ASSERTIONS PASS.

+// fix362: regression tests for the series-view / episode-leak query semantics
+// that fix353's table-level test never exercised. These build the exact WHERE
+// fragments Sql.search emits and run them against a seeded DB:
+//   - series view (seriesId set) returns that series' episodes
+//   - movies browse + movies search (seriesId null) EXCLUDE episodes
+// CRIT-1 was: an unconditional "series_id IS NULL" in the base ANDed with the
+// series view's "series_id = ?" -> every series empty. HIGH-1 was the parallel
+// gap in the in-memory cache filter.
+import 'package:flutter_test/flutter_test.dart';
+import 'package:sqlite3/sqlite3.dart' as s3;
+
+bool _has() {
+  try { s3.sqlite3.openInMemory().dispose(); return true; } catch (_) { return false; }
+}
+
+void main() {
+  if (!_has()) {
+    test('sqlite3 unavailable — skipped', () => expect(true, isTrue));
+    return;
+  }
+  late s3.Database db;
+  setUp(() {
+    db = s3.sqlite3.openInMemory();
+    db.execute('''CREATE TABLE channels(id INTEGER PRIMARY KEY, name TEXT, url TEXT,
+      source_id INT, media_type INT, series_id INT, favorite INT DEFAULT 0);''');
+    db.execute('''INSERT INTO channels(name,url,source_id,media_type,series_id) VALUES
+      ('Breaking Bad','42',5,2,NULL),('S01E01','u1',5,1,42),('S01E02','u2',5,1,42),
+      ('Some Movie','m1',5,1,NULL),('Another Film','m2',5,1,NULL);''');
+  });
+  tearDown(() => db.dispose());
+
+  // Mirrors Sql.search append logic: seriesId set -> "series_id = ?";
+  // else -> "series_id IS NULL".
+  List<String> run({int? seriesId}) {
+    var q = 'SELECT name FROM channels c WHERE media_type IN (1)'
+        ' AND source_id IN (5) AND url IS NOT NULL';
+    final p = <Object>[];
+    if (seriesId != null) {
+      q += ' AND series_id = ?';
+      p.add(seriesId);
+    } else {
+      q += ' AND series_id IS NULL';
+    }
+    return db.select(q, p).map((r) => r['name'] as String).toList()..sort();
+  }
+
+  test('series view returns that series\' episodes (CRIT-1 guard)', () {
+    expect(run(seriesId: 42), ['S01E01', 'S01E02']);
+  });
+
+  test('movies browse/search excludes episodes (fix355 intent preserved)', () {
+    expect(run(seriesId: null), ['Another Film', 'Some Movie']);
+  });
+
+  // HIGH-1: the in-memory cache filter must be symmetric.
+  bool cacheKeep(int? eSeriesId, int? seriesId) {
+    if (seriesId != null && eSeriesId != seriesId) return false;
+    if (seriesId == null && eSeriesId != null) return false;
+    return true;
+  }
+
+  test('in-memory cache excludes episodes outside a series, keeps them inside', () {
+    expect(cacheKeep(42, null), false); // episode in movies search
+    expect(cacheKeep(null, null), true); // movie in movies search
+    expect(cacheKeep(42, 42), true); // episode in its series
+    expect(cacheKeep(99, 42), false); // other series' episode
+  });
+}
