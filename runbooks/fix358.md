## Two bugs

### 1. Multi-view channel picker: no focus highlight on TV (`channel_picker_screen.dart`)
Each row is a `ListTile` wrapped in an opaque `Container` painting the source tag tint (fix228). That opaque fill sat ON TOP of `ListTile`'s built-in focus highlight, so D-pad navigation gave no visible indication of the selected row.
**Fix:** wrap the row in a non-focusable `Focus` + `Builder`; when `Focus.of(context).hasFocus`, the row blends 30% primary over the source tint and draws a 2.5px primary border (`AnimatedContainer`, 120ms). The `ListTile` (extracted to `_pickerListTile`) keeps its `onTap` and `autofocus`; `canRequestFocus:false` on the wrapper means traversal still lands on the tile, so existing D-pad order and the fix252 first-tile autofocus are unchanged.

### 2. QR/LAN sources-import dropped source-level settings (`settings_io.dart` + `sql.dart`)
The portal import path (`importSourcesOnly`) already stages channel preserves + category state + resume positions (fix355), but the per-source EXPORT map (`_sourceToMap`) only wrote 7 fields — `maxConnections`, `color`, and `sortMode` were omitted; the import `Source(...)` ctor omitted them too; and `getOrCreateSourceByName` never wrote those columns even when present.
**Fix, all three layers:** `_sourceToMap` now exports `maxConnections`/`color`/`sortMode`; `importSourcesOnly` reconstructs them; `getOrCreateSourceByName` writes them on INSERT and `COALESCE`-updates them on UPDATE (a null in a re-import never clobbers a locally-tuned value). EPG URL, favorites, category state and positions already flowed — this closes the remaining source-level gap.
