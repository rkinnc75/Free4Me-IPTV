# fix586 — #6: held-OK opens the channel context menu on TV

**Version:** v2.2.3+587 (with fix585 #4, fix587 #23)
**Date:** 2026-06-28
**Type:** TV interaction (on-device-found gap)

## Problem

On-device on the onn: the channel context menu — and its fix584 "Open in
Multi-view" entry, plus favorite/category/mini-player/history — is opened only
by `InkWell.onLongPress`, which is a TOUCH gesture. A TV remote cannot fire it:
- A long-press of OK (`adb input keyevent --longpress 23`) just registered as a
  tap → opened the player.
- KEYCODE_MENU (82) did nothing (no-op).

So every context-menu item was unreachable by remote. #6's Multi-view entry was
correct in code but could never be opened on TV.

## Fix (`lib/channel_tile.dart`)

Added a timer-based long-press detector in `_focusNode.onKeyEvent` for the
select keys (`select`/`enter`/`numpadEnter`/`gameButtonA`):
- KeyDown(select): start a 450 ms hold timer, mark `_selectActed=false`, and
  **consume** the event so InkWell's default `ActivateIntent` (which fires onTap
  on key-down) never runs.
- Hold timer fires (still focused): `_onLongPress()` → opens the menu; mark acted.
- KeyUp(select): cancel timer; if not acted → `_activate()` (the former onTap:
  play, or category toggle). 
- KeyRepeat(select): swallowed (the timer drives the menu).

`onTap` now calls the extracted `_activate()` so touch + quick-D-pad share one
path. Timer cancelled in `dispose()`. Phone is unaffected (touch uses
onTap/onLongPress gestures, not key events).

  "Open in Multi-view"; selecting it opens multi-view seeded with the channel.
