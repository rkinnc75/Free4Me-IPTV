# fix720 — TV GUI Phase 5 unit 3: bottom-sheet menu restyle (v4.1.24)

**Builder:** desktop. **Ships as:** v4.1.24+720. TV only; phone/touch UI
byte-identical. Autonomous ("keep going, pick recommended"). Plan:
`docs/TV_GUI_REDESIGN.md` §4.10/§4.11 (tokenize the pre-existing menus/dialogs).

## What
The channel context menu (fix586: held-OK → `showModalBottomSheet`) — and every
other modal bottom sheet — rendered as the flat default Material sheet, reading
as un-migrated next to the redesigned dialogs/OSD. fix720 gives TV bottom sheets
the redesign glass look (dark card + rounded top corners).

## How (one theme-level change, no per-call-site edits)
`main.dart` `ThemeData` gains:
```
bottomSheetTheme: hasTouchScreen
    ? null                          // phone → Material default (byte-identical)
    : const BottomSheetThemeData(
        backgroundColor: Color(0xF00B0F19),      // opaque F4 glassFill
        modalBackgroundColor: Color(0xF00B0F19),
        surfaceTintColor: Colors.transparent,    // kill M3 elevation tint
        shape: RoundedRectangleBorder(
          borderRadius: BorderRadius.vertical(top: Radius.circular(20)), // F4Radius.modal
        ),
      ),
```
- Values mirror `F4Tokens` as literals (ThemeData is built without an
  `F4.of(context)` in scope). `0xF00B0F19` is the near-opaque glass (the OSD uses
  `0xCC…`; a modal sheet over content reads better nearer-opaque).
- Theme-level → covers ALL `showModalBottomSheet` calls (the 2 in `channel_tile`
  + any others) uniformly; no logic touched.
- Phone gets `null` → the Material default sheet, so the touch UI is unchanged.
