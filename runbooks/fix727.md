# fix727 — Player Actions Bar completion: speed + sleep timer (mock §4.6)

TV OSD only; phone/touch bar unchanged. Task #45. Chrome-only addition — the
playback engine, channel-surf, and reconnect logic are untouched (mock §4.6:
"keep the existing engine … this is a chrome rebuild").

## What (the two genuinely-missing Actions Bar entries)
The three-bar Peer2 OSD (Info/Channel bars) shipped in Phase 4 (fix714/716) and
the button row already had Play/Pause, Subtitles, Audio, Zoom/Aspect, PiP,
Mini-player. fix727 fills the last two of the mock's `[…][Sleep Timer]` set:
- **Playback speed** — presets {0.25,0.5,0.75,1.0,1.25,1.5,2.0} (mock §4.6).
  Button shown only on **seekable** (VOD/DVR) surfaces with track support.
- **Sleep timer** — {Off,15,30,45,60,90} min. Arms a one-shot Timer that
  pauses playback then exits the player; bedtime icon fills while armed.

## How
- `player_engine.dart`: abstract gains `double get playbackRate => 1.0` +
  `Future<void> setRate(double) async {}` (default no-op / fixed rate).
- `mpv_engine.dart`: overrides via media_kit `_player.setRate`, caches `_rate`.
- `player.dart`:
  - `_openSpeedFromOverlay()` / `_openSleepTimerFromOverlay()` — reuse the
    existing `SelectDialog` (glass theme fix720/721 + D-pad traversal) with the
    same auto-hide-cancel wrapper as the subtitles/audio pickers; `selectedId`
    pre-highlights the active preset (speed = nearest index; sleep = minutes,
    0 = Off).
  - `_armSleepTimer(int)` — cancels any prior timer; `minutes>0` arms
    `Timer(Duration(minutes:))` → `_engine.pause()` then guarded `onExit()`
    (both mounted-checked). `dispose()` cancels `_sleepTimer`.
  - bottomBar: speed button `if (seekable && tracks)`; sleep button always
    (last), icon `bedtime`/`bedtime_outlined` by armed state.
