# fix705 — TV GUI redesign Phase 3 unit 2: guide NOW emphasis (v4.1.10)

**Builder:** desktop. **Live base:** v4.1.9. **Ships as:** v4.1.10+705. TV only;
phone UI byte-identical. Standing autonomous GUI mandate.

## What
`lib/tv/tv_guide_view.dart` — two changes making "now" legible at 10 feet, both
from EPG programme times only (no data layer, no new `Program` field):

1. **NOW-line glow (`_nowLine`)** — the 2px full-height "now" indicator gains a
   `BoxShadow` (blur = token `F4.of(context).focus.nowGlowRadius` = 8, spread
   0.5). Colour kept `scheme.primary` (blue) — **deliberately distinct** from the
   white accent focus ring (fix704), so the time indicator and the focus
   indicator never read as the same thing.
2. **Progress-within-cell fill (`_block`, on-now cell)** — the elapsed fraction
   of the current show's runtime tints the left portion of its cell a little
   stronger (a built-in progress bar). `Stack` → `Positioned.fill(
   FractionallySizedBox(alignment: centerLeft, widthFactor: elapsedFrac,
   child: Container(primary @0.18)))` behind the existing `Padding(Align(Text))`.
   `elapsedFrac = (isNow && dur>0) ? ((now - start)/dur).clamp(0,1) : 0` —
   guarded against zero/negative duration (bad EPG data).

### Perf note (why the clip is gated)
`Material.clipBehavior` is `isNow ? Clip.antiAlias : Clip.none`. Only on-now
cells (1 per row) add the fill layer + need the clip; non-now cells keep
`Clip.none` so there is **no per-cell clip layer across the dense grid**. The
`_nowLine` glow is drawn once **per visible row** (the now-line is per-row inside
each row's `Stack`), i.e. ~10-20 `BoxShadow` blurs on a full screen — flagged for
an on-device scroll-fps check on the onn (low-RAM 32-bit). If it juddered, the
fallback was a smaller radius / gradient / single-overlay line; **verify on
device.**

### Badges dropped (scope honesty)
`Program` (`lib/models/program.dart`) has id/epgChannelId/sourceId/title/
description/category/start/stop/episodeNum — **no HD/SD/quality field.** The
doc's "badges" would require a title heuristic = a separate, riskier concern.
Not fabricated here.

### Kept / untouched
Passive-grid `ExcludeFocus` (finding 75) + `onTap: _play(ch)`; rail↔grid
Y-alignment (`_rowHeight=56` — Stack sizes to the non-positioned `Padding(Text)`,
so cell size is unchanged); :00/:30 snap; live preview; place-memory; held-OK.
