# fix571 — framedrop=decoder on low-RAM Android (smooth high-fps playback)

**Version:** v2.1.0+571
**Date:** 2026-06-27
**Type:** Playback smoothness (low-RAM Android boxes)

## Problem

On low-RAM / weak-GPU Android boxes (onn 4K Plus, software decode), high-frame-rate
streams (50/60 fps) judder. fix564 attributed this to the Mali-G310 not keeping up
with full-rate frame uploads at the video-output (VO) stage. The intended remedy —
a 60→30 `vf=fps` cap — was infeasible on the stock bundled libmpv (no `fps`
libavfilter filter; see runbooks/fix566–570 and the libmpv-no-vf-filters note), so
fix570 disabled the cap and left the judder unaddressed.

## Finding (on-device ADB sweep, onn 4K Plus)

A controlled sweep on a heavy 1080p50 ~9 Mbps stream, measured via the per-stream
mpv `STATS` logcat line (`voDrop`/`decDrop`; `vfFps`/estimated-vf-fps is unreliable
on this build — ignore it), isolated the libmpv `framedrop` mode as the lever:

| Frame drop mode | A/V sync (any) | VO drops/min |
|---|---|---|
| `vo` (libmpv default) | audio / display-resample / display-resample-drop | 528–849 (judder) |
| `decoder` | audio / display-drop | **0** (smooth, ~0.4% decoder drops) |

Two independent `decoder` windows (4.1 min, 5.0 min) held 0 VO drops; three `vo`
windows all dropped heavily. The A/V sync mode made no measurable difference —
`framedrop` is the whole story. `framedrop=decoder` sheds the rare late frame
before the upload stage so the VO never backs up, keeping ~full frame rate (better
than a hard 30 fps cap, which would discard ~40% of frames).

## Change

Apply-time gate in `lib/player/mpv_engine.dart` `_applyMpvOptions()`:

```dart
final lowRamFramedrop = Platform.isAndroid &&
    s.devFramedrop.value == 'vo' &&
    await DeviceDetector.isLowRamDevice();
final framedropMode = lowRamFramedrop ? 'decoder' : s.devFramedrop.value;
await np.setProperty('framedrop', framedropMode);
```

- Default `devFramedrop` stays `vo` (libmpv upstream). On low-RAM Android a `vo`
  setting is auto-upgraded to `decoder`; capable devices/phones are unchanged.
- Because it gates on the stored `vo`, existing low-RAM installs benefit with no
  settings migration. Explicit `no`/`decoder` in Developer settings still wins.
- Help/label/subtitle text in `lib/settings_view.dart` documents the auto-upgrade.

## Caveat

mpv flags `decoder` mode as able to "produce unwatchable distorted output" in
theory; the 5-min sweep showed none and 0 errors. Watch real-world use.

## Notes

- Ships alongside the custom LGPL-max libmpv (commit 66045f3) merged earlier;
  `framedrop=decoder` is libmpv-independent and works on stock and custom builds.
- The orthogonal `vf=fps=30` cap (now possible on the custom libmpv) is parked as
  an opt-in toggle (default OFF) on branch `libmpv-lgplmax-verify` — not in this
  release.
