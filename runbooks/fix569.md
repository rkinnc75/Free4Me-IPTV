# fix569 — name the `select` option explicitly (device libavfilter rejects positional args)

Base: **v2.0.66** (fix567/568 were never-released dry-run iterations).
Ships as **v2.0.69+569**.

## What fix568 got wrong (onn 4K Plus dry-run)

fix568 set `vf=lavfi=[select=not(mod(n\,2))]`. The comma escape was correct
(no "No such filter" this time), but the device stalled with:

```
graph = 'select=not(mod(n\,2))'
AVFilterGraph: No option name near 'not(mod(n,2))'  →  Creating filter 'lavfi' failed  →  stall
```

This build's (older) libavfilter **rejects positional filter arguments** — it
requires `key=value`. It's the exact same rule that broke `fps=30` in fix566
("No option name near '30'"). My local mpv 0.41 is newer and accepts positional
args, which is why fix568 passed locally but failed on the box.

Crucially, the error reached *argument* parsing → the **`select` filter exists**
on this build (unlike `fps`, which gave "No such filter").

## Fix

Name the option explicitly: **`vf=lavfi=[select=expr=not(mod(n\,2))]`** (filter
`select`, option `expr`=the every-other-frame expression). Validated on mpv
0.41 (creates + reconfigures the VO). Named option satisfies the device's
strict parser; `select` exists → it creates and runs. Dart raw string keeps the
`\,` literal.
