# fix607 — v2.2.18: Live-TV held-OK diagnostic easter egg + shared IssueReporter

User request: "If you hit long press on Live TV, I want to add an easter egg
that will send a diagnostic report automatically, if diagnostic mode is enabled.
For subject set a timestamp, for body it can be blank."

## What it does
HELD-OK (long-press) on the **Live TV** tab (tab 0) auto-submits a diagnostic
report when **diagnostic mode is on** — subject = `DateTime.now().toIso8601String()`,
body = empty. Uses the SAME scrubbed payload as Settings › Report an issue.

## Gating (safety)
`SettingsService.cached.debugLogging == true` AND `logUserPass == false` — the
latter mirrors the Settings report flow so a log that captured raw provider
credentials is never sent. If not gated, the held-OK is a silent no-op on the
Live TV tab. Feedback via a SnackBar ("Sending diagnostic report…" →
"Diagnostic report sent." / "Report failed: …").

## Pieces
1. **`lib/backend/issue_reporter.dart` (new)** — `IssueReporter.submit({subject,
   details}) → IssueReportResult{success, errorMsg}`. Extracted verbatim from
   SettingsView's `_submitIssueReport` network/payload block (worker URL + app
   secret + scrubbed log + scrubbed settings export + device/version/clientId).
2. **`settings_view.dart`** — `_submitIssueReport` now calls
   `IssueReporter.submit(...)` and keeps its dialogs; removed the duplicate
   `_issueWorkerUrl`/`_issueAppSecret` consts and the now-unused `dart:convert`
   + `http` imports.
3. **`tv_top_tab_bar.dart`** — `_TabButton` gained a held-OK detector on its own
   FocusNode (TV remotes can't fire `InkWell.onLongPress`, which is touch-only —
   so the existing History-clear long-press was ALSO unreachable by D-pad).
   Mirrors fix603: `_selectDown` orphan guard + open-on-RELEASE (timer only marks
   `_heldLong`; KeyUp decides held=onLongPress vs quick=onTap), 600ms. Touch
   `onLongPress` kept. NOTE: this makes EVERY tab's quick-OK activate on key-UP
   (was key-DOWN) — consistent with the channel tiles, low risk on the onn where
   key-up is reliable.
4. **`tv_shell.dart`** — `_onTabLongPress`: tab 0 (Live TV) → gated
   `IssueReporter.submit(subject: timestamp, details: '')` + SnackBar; History
   tab branch unchanged (and now actually reachable by remote).

## On-device verify
- adb can't inject a true key-hold, so the held-OK gesture itself needs the real
  remote (same as the channel menu). adb CAN confirm the touch path: `input
  swipe X Y X Y 700` on the Live TV tab with debugLogging on → "Sending
  diagnostic report…" then a result; a report should land at the
  free4me-issue-reporter worker / private repo.
- With debugLogging OFF, held-OK on Live TV does nothing.
