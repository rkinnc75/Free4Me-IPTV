# fix668 — Scheduled Recording capture engine (v3.0.12)

**Builder:** desktop. **Live base:** v3.0.11. **Ships as:** v3.0.12+668.

Second phase of the Scheduled Recording (SR) arc (667 schema+scheduling → **668
capture** → 669 UI → 670 optional transcode). Wires real capture behind the
exact alarm from fix667. "SR" = record-to-file; distinct from the app's live
"DVR" (rewind-within-live-stream) — this runbook also folds in the fix667
comment rename (DVR→SR) that shipped slightly late in v3.0.11.

## Architecture (why a native service)

android_alarm_manager_plus runs the alarm callback in a short JobIntentService
window (seconds–minutes under Doze) — NOT long enough for a multi-hour capture.
So the Dart alarm callback only STARTS a native foreground service, which holds
its own wake+wifi locks for the full duration and outlives the alarm window and
app task.

## What

- **`RecordingCaptureService.kt`** (native foreground `Service`,
  `foregroundServiceType=dataSync`, `stopWithTask=false`): buffered
  `HttpURLConnection` inputStream → MediaStore outputStream copy for
  `durationMs`; PARTIAL_WAKE_LOCK + WIFI_MODE_FULL_HIGH_PERF held for the run;
  output to **MediaStore `Movies/Free4Me/*.ts`** (`IS_PENDING` until finalized,
  scoped-storage clean on Android 10+); manual stop via a cancel flag
  (ACTION_STOP). On finish/failure it writes `status` + `output_path` + `error`
  **directly to `context.filesDir/db.sqlite`** (the same file DbFactory opens),
  UPDATE on the `recordings` row. Status strings match `RecordingStatus.name`
  ('recording'/'done'/'failed'). <64KB captured ⇒ failed (keeps the file).
- **MainActivity**: new `me.free4me.iptv/recording` MethodChannel
  (startCapture / stopCapture).
- **`recording_capture.dart`**: thin Dart bridge (start/stop); no-op off
  Android.
- **`recordingAlarmCallback`** (was a stub): looks up the row, skips if
  cancelled, flips to `recording`, and starts the native service via the
  bridge; marks `failed` if start throws.
- **Manifest**: `<service>` declaration.

## DB write-back contract (verified in-sandbox)

Native writes columns `status` / `output_path` / `error` keyed on `id` — these
match migration 44's schema exactly (confirmed by diffing the CREATE TABLE
against the service's ContentValues). The status enum strings match the Dart
`RecordingStatus.name` values.

+    private fun runCapture(id: Int, url: String, durationMs: Long, name: String) {
+        val pm = getSystemService(Context.POWER_SERVICE) as PowerManager
+        val wakeLock = pm.newWakeLock(PowerManager.PARTIAL_WAKE_LOCK, "$TAG:wl:$id")
+        val wifiManager =
+            applicationContext.getSystemService(Context.WIFI_SERVICE) as WifiManager

+        )
+
+        var mediaUri: Uri? = null
+        var conn: HttpURLConnection? = null
+        try {
+            wakeLock.acquire(durationMs + 60_000L)
+            wifiLock.acquire()

+            mediaUri = createMediaStoreEntry(name)
+            if (mediaUri == null) {
+                updateStatus(id, "failed", null, "Could not create output file")
+                return
+            }
+
+            conn = (URL(url).openConnection() as HttpURLConnection).apply {
+                connectTimeout = 15_000
+                readTimeout = 30_000
+                instanceFollowRedirects = true
+            }
+            conn.connect()
+            if (conn.responseCode !in 200..299) {
+                updateStatus(id, "failed", null, "HTTP ${conn.responseCode}")
+                return
+            }
+
+            val deadline = System.currentTimeMillis() + durationMs
+            var total = 0L
+            contentResolver.openOutputStream(mediaUri)?.use { out ->
+                conn.inputStream.use { input ->
+                    val buf = ByteArray(64 * 1024)
+                    while (System.currentTimeMillis() < deadline &&
+                        cancelFlags[id] != true
+                    ) {
+                        val n = input.read(buf)
+                        if (n < 0) break // stream ended
+                        out.write(buf, 0, n)
+                        total += n
+                    }
+                    out.flush()
+                }
+            }
+
+            finalizeMediaStore(mediaUri)
+            if (total < 64 * 1024) {
+                // Almost nothing captured — treat as failure but keep the file.
+                updateStatus(id, "failed", mediaUri.toString(),
+                    "Captured too little data (${total} bytes)")
+            } else {
+                updateStatus(id, "done", mediaUri.toString(), null)
+            }
+        } catch (e: Exception) {
+            Log.w(TAG, "capture $id failed: ${e.message}")
+            mediaUri?.let { runCatching { finalizeMediaStore(it) } }
+            updateStatus(id, "failed", mediaUri?.toString(), e.message ?: "error")
+        } finally {
+            runCatching { conn?.disconnect() }
+            runCatching { if (wakeLock.isHeld) wakeLock.release() }
+            runCatching { if (wifiLock.isHeld) wifiLock.release() }
+            active.remove(id)
+            cancelFlags.remove(id)
+            stopSelfIfIdle()
+        }
+    }

+            )

+                put("status", status)
+                if (outputPath != null) put("output_path", outputPath)
+                if (error != null) put("error", error)
+            }
+            db.update("recordings", values, "id = ?", arrayOf(id.toString()))
+        } catch (e: Exception) {
+            Log.w(TAG, "updateStatus($id,$status) failed: ${e.message}")
+        } finally {
+            runCatching { db?.close() }
+        }
+    }
