# fix152.md — Content filter: make "All" white so it's visually distinct from "Live" (currently both blue)

> **Version:** Free4Me-IPTV 1.22.13 build.
> **Issue:** In the content-type filter cycle (All → Live → Movies → Series),
> **"All" and "Live" use the SAME color** (`0xFF4E9FE5`, blue), so you can't
> tell at a glance which one is active. Make **All white**; leave **Live** blue
> (and Movies/Series unchanged).
>
> Also serves as a small end-to-end release smoke test that the build Mac can
> tag and publish cleanly after the recent fix146/148/150 changes.

---

## Root cause

`lib/bottom_nav.dart`, the `_filterColors` map (lines 24–29):

```dart
const _filterColors = {
  ContentTypeFilter.all:     Color(0xFF4E9FE5), // blue (matches old All)
  ContentTypeFilter.live:    Color(0xFF4E9FE5), // blue
  ContentTypeFilter.movies:  Color(0xFF8BC34A), // lime
  ContentTypeFilter.series:  Color(0xFFE040FB), // magenta
};
```

`all` and `live` are both `0xFF4E9FE5`. The color is used at line 155
(`final filterColor = _filterColors[filter]!;`) for the active filter's chip/
icon/label, so All and Live are indistinguishable.

---

## fix152.1 — Make "All" white, keep "Live" blue

**File:** `lib/bottom_nav.dart`

**Current (lines 24–29):**

```dart
const _filterColors = {
  ContentTypeFilter.all:     Color(0xFF4E9FE5), // blue (matches old All)
  ContentTypeFilter.live:    Color(0xFF4E9FE5), // blue
  ContentTypeFilter.movies:  Color(0xFF8BC34A), // lime
  ContentTypeFilter.series:  Color(0xFFE040FB), // magenta
};
```

**Replace with:**

```dart
const _filterColors = {
  ContentTypeFilter.all:     Color(0xFFFFFFFF), // fix152: white (distinct from Live)
  ContentTypeFilter.live:    Color(0xFF4E9FE5), // blue
  ContentTypeFilter.movies:  Color(0xFF8BC34A), // lime
  ContentTypeFilter.series:  Color(0xFFE040FB), // magenta
};
```

Only the `all` entry changes (`0xFF4E9FE5` → `0xFFFFFFFF`). `live` stays blue;
`movies`/`series` unchanged.

> The UI is dark-themed (the filter chips sit on a dark bottom nav, per the
> existing palette), so pure white reads clearly. If on your device white looks
> too stark next to the others, `0xFFECEFF1` (very light blue-grey) is a softer
> alternative — but white is what was requested.
