# fix182 â€” land D-pad focus on the first element when a screen/dialog opens

> **Version:** 1.23.14+182 (phone/even build; patch bump from shipped 1.23.13+180)
> **Source verified against:** `Free4Me-IPTV-1.23.13` (1.23.13+180).
> **Request:** the yellow focus ring (fix168) works, but several surfaces open with focus on *nothing*, so the user must hunt with the D-pad to find where focus is. Make focus land on the first visible/sensible element when each surface opens.
> **Surfaces:** Home channel grid (and category/group lists â€” same grid), Settings list, and all ~20 dialogs. On destructive dialogs, focus the **safe** option (not the confirm) to avoid accidental data loss on an OK press.

## Mechanisms (all verified against api.flutter.dev / repo)

Three different surface shapes need three approaches; the repo already uses `autofocus` in 8 places (e.g. `multi_view_cell.dart:810` `autofocus: cellIndex == 0`, `menu_tile.dart`, `channel_picker_screen.dart:269`), so this matches existing patterns.

1. **Widgets that expose `autofocus`** (InkWell-based tiles, buttons): set `autofocus: true` on the intended first/safe target. (`InkWell`, `TextButton`, `FilledButton`, `ElevatedButton` all support it.)
2. **Widgets that do NOT expose `autofocus`** (`ExpansionTile` â€” confirmed no such param on api.flutter.dev; `ListTile`): request focus into the scope after first frame with `FocusScope.of(context).nextFocus()` in an `addPostFrameCallback`. Per Flutter docs, a focus scope with no focused child uses `nextFocus()` to move to the **first** focusable in traversal order â€” exactly "focus the first element."
3. **Only one widget per route may have `autofocus: true`.** Don't double up.

# Fix 182.2 â€” Settings list (first row)

**File:** `lib/settings_view.dart`

The first focusable in the list is the "Playback" `ExpansionTile` (~line 1630). `ExpansionTile` has **no `autofocus`** (verified, api.flutter.dev), so use the post-frame `nextFocus()` approach in the State.

In `_SettingsState`, add a post-frame focus request. If there's an `initState`, add it there; if not, add one. Add:

```dart
  @override
  void initState() {
    super.initState();
    // fix182: land D-pad focus on the first settings row when the screen
    // opens (ExpansionTile has no autofocus param; nextFocus() on a scope
    // with no focused child moves to the first focusable in traversal order).
    WidgetsBinding.instance.addPostFrameCallback((_) {
      if (mounted) FocusScope.of(context).nextFocus();
    });
  }
```

> If `_SettingsState` already overrides `initState`, insert just the `addPostFrameCallback(...)` block at its end instead of adding a second `initState` (a duplicate override is a compile ERROR). Verify by grepping `void initState` in the file first.
> `WidgetsBinding` is in `package:flutter/widgets.dart` (re-exported by material, already imported). `FocusScope` likewise. No new import.

**Key changes:** opening Settings moves focus to the first row (Playback section header), visibly highlighted.
