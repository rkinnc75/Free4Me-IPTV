# fix722 — multi-view accent focus ring (v4.1.26)

**Builder:** desktop. **Ships as:** v4.1.26+722. TV only; phone/touch
byte-identical. Autonomous ("keep going, pick recommended"). **Completes the
accent focus-ring language** — multi-view was the LAST TV surface still on the
old `colorScheme.primary` ring. Plan: `docs/TV_GUI_REDESIGN.md` §4.11.

## What
The focused-cell ring in multi-view (and the empty-cell "+" button focus border)
were `Theme.of(context).colorScheme.primary` / `Colors.white`. On TV they now
paint with the shared `AccentScope` accent, matching tabs(702)/tiles(703)/
rails(704)/buttons(707)/recordings(718). Milestone: every TV focus indicator now
speaks the accent language.

## How (ambient-gated — no plumbing)
`multi_view_cell.dart` `_MultiViewCellState`:
- `bool get _isTvLike => widget.settings.forceTVMode || DeviceDetector.isTvCached
  == true || Utils.hasTouchScreenCached == false;` (mirrors channel_tile
  finding-107; `widget.settings` already available → no SettingsService import).
- `Color _ringColor(BuildContext c, Color fallback) => _isTvLike ?
  AccentScope.of(c) : fallback;`
- Two cell focus-ring `Border.all` colors → `_ringColor(context,
  Theme.of(context).colorScheme.primary)`.
- The "+" button focused border → `_ringColor(context, Colors.white)`.

No `MultiViewScreen`/`MultiViewCell` constructor or call-site changes (both
construction sites — the static opener + home.dart — untouched). On a phone
`_isTvLike` is false → the original colors render (**byte-identical**).

## P0 safety
ONLY color expressions changed. The `_CellMenuIntent` single-press menu bindings,
the audio-focus-follows-D-pad-focus hook (fix170), and cell-0 autofocus (fix172)
— the three behaviors §4.11 warns not to regress — are NOT touched. The "+"
button FILL stays `colorScheme.primary` (a colored bg behind the white "+", so a
white accent doesn't hide the icon). Ring width unchanged (3).

## Tests / review
