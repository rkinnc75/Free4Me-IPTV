# fix587 — #23: Confirm-to-exit setting (re-do)

**Version:** v2.2.3+587 (with fix585 #4, fix586 #6)
**Date:** 2026-06-28
**Type:** Feature (re-do of the earlier reverted #23)

## Background

The first #23 attempt put a `PopScope` at the root `MaterialApp`, which did NOT
survive the phone bottom-nav's `Navigator.pushAndRemoveUntil` (it rebuilds the
root) — so it was reverted/deferred. This re-do wraps the individual top-level
screens instead, and adds a guard so the user is never trapped.

## Implementation

- **New widget** `lib/confirm_exit_scope.dart` (`ConfirmExitScope`): wraps a
  screen with a `PopScope`. Double-back-to-exit with a 2 s window and a "Press
  Back again to exit" SnackBar. Critically, it only intercepts when
  `!Navigator.of(context).canPop()` — i.e. Back would actually EXIT the app.
  When the enclosing route can still pop (e.g. Settings opened as a sub-route on
  TV), Back falls through normally → no trapping.
- **Setting** `confirmToExit` (default false): added to `settings.dart`
  (field + ctor), `settings_service.dart` (`confirmToExitProp` + read + write),
  and `settings_io.dart` backup round-trip.
- **UI**: a "Confirm to exit" `_switchTile` in Settings (next to Safe mode).
- **Wired** into `home.dart` (phone home), `tv_shell.dart` (TV shell), and
  `settings_view.dart` (phone Settings is root via pushAndRemoveUntil). Each
  passes `enabled: <settings>.confirmToExit`; the canPop guard makes wrapping
  safe everywhere.

## Why no hero-preview teardown on exit

The adversarial review flagged "tear down TvHeroPreview before exit." Not
needed: a confirmed exit pops the root → the activity finishes → the widget tree
disposes (engine torn down via normal dispose), and the OS reclaims the process.
There is no lifetime leak introduced by the exit path itself.

## Verification

- `flutter analyze` clean; `flutter test` green (272).
- **On-device (post-ship):**
  - onn (TV): Settings › General → enable "Confirm to exit". At the TV shell,
    press Back → SnackBar "Press Back again to exit", app stays. Back again
    within 2 s → app closes. Wait >2 s after first Back → it re-prompts.
  - onn: open Settings (sub-route) with the setting ON → Back returns to the
    shell normally (NOT trapped) — the canPop guard.
  - With the setting OFF → Back exits immediately (unchanged).
