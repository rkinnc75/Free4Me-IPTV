# fix579 — revert broken Mode B; restore verified v2.1.4 D-pad

**Version:** v2.1.6+579
**Date:** 2026-06-27
**Type:** Regression fix (revert)

## Problem (regression introduced by fix578 / v2.1.5)

fix578 added "Mode B" — D-pad navigation of the player's control bars — by setting a
`_barsNav` flag on OK that was meant to disable media_kit's auto-hide and push focus
into the bar buttons. **On-device testing on the onn (v2.1.5) showed it does not work
and regresses the working v2.1.4 D-pad:**

1. Auto-hide was NOT actually disabled — the bars still vanished after ~3 s
   (`_barsNav`→long `controlsHoverDuration` did not take effect for the synth-tap
   timer).
2. Focus never entered media_kit's buttons (`FocusScope.nextFocus()` did not reach
   them), so the D-pad did not navigate the bars.
3. **Regression:** once the bars auto-hid, `_barsNav` stayed `true`, so `_onPlayerKey`
   returned `ignored` for everything → the ENTIRE D-pad went dead (no channel up/down,
   no seek, no play/pause) until Back. Confirmed via ADB: after OK + auto-hide, a
   further OK logged no `pause`/`play` and changed nothing.

Root cause: media_kit 2.0.1's controls auto-hide on a pointer/hover timer that ignores
key/focus activity, and its button subtree is not reachable by Flutter directional
traversal from our ancestor `Focus`. The "navigate media_kit's bars" approach is not
viable; see runbooks/fix578.md.

## Change

`git checkout v2.1.4 -- lib/player.dart` — restores player.dart to its exact, on-device-
verified v2.1.4 state: direct-map D-pad (▲▼ channel, ◀▶ seek, OK = play-pause + reveal
bars via synth tap) with NO `_barsNav` machinery. fix577's guide-playlist fix and
fix575's stream-info label are unaffected (those are in other files).

## Next (real Mode B)

Reaching Cast/PiP/Subtitles/Audio/Aspect by D-pad requires a **custom focusable
overlay** we fully control (deterministic visibility, our own auto-hide reset on key,
explicit FocusNodes + FocusTraversalGroup) — NOT media_kit's bars. That is a separate,
larger build; deferred.
