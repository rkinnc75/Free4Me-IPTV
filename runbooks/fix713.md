# fix713 — "Re-match all channels" fails when the EPG feed is unchanged

**Base:** main @ `5c877168d7b69a8a1a9744a6e665edce70059d6a` (v4.1.16+712)
**Version:** 4.1.16+712 → **4.1.17+713**
**Files:** epg_service.dart, xmltv_parser.dart, settings_view.dart, whats_new_modal.dart, pubspec.yaml
**Analyze:** run on the full modified files with pinned Flutter 3.44.5 / Dart 3.12.2 — result: **No issues found!** (0 INFOs)

## 1. Symptom (S938U log 20260712-095426)

"Re-match all channels" reported `⚠ Trex: failed to download EPG` / `⚠ A3000: failed to download EPG` on every attempt. Log shows Trex returned HTTP 200 with `body unchanged (hash match)` and A3000 returned `HTTP 304 Not Modified` — both then `download returned null`.

## 2. Root cause

fix695's unchanged-feed optimization (conditional GET validators + body-hash skip) makes `downloadAndParseEpg` return **null** when the feed is identical to the previous refresh. `_runEpgRematch` treats null as a download failure and skips the source; `refreshSource` silently bails before `matchChannels`. So re-match only works when the provider's feed *changed* — inverted from its purpose (matcher updated, feed identical). Secondary: the plain EPG refresh mislabels the unchanged skip as `⚠ 0 programs loaded`.

## 3. Fix

- `XmltvParser.parse`: new `forceParse` param — suppresses the If-None-Match / If-Modified-Since validators and the body-hash short-circuit, so a full parse always yields the channelMap.
- `EpgService.downloadAndParseEpg`: returns tri-state `EpgDownloadResult` (channelMap / unchanged / failed) instead of conflating unchanged and failed as null; threads `forceParse`.
- `EpgService.refreshSource`: returns `EpgRefreshOutcome {refreshed, unchanged, failed}`; passes `forceParse: forceRematch`. Other callers (utils.dart, setup.dart, refreshAllSources) ignore the return — no changes needed there.
- `settings_view.dart`: re-match passes `forceParse: true` and consumes the tri-state; refresh summary shows `✓ feed unchanged — kept existing data` (INFO) instead of the bogus `⚠ 0 programs loaded` warning when a source's feed is unchanged.
- R4 whats_new entry for 4.1.17; version bump.

Behavior preserved: validators are still persisted after a forced parse, so the *next* ordinary refresh benefits from 304/hash-skip again. The unchanged path still records a clean refresh in `epg_refresh_log` (launch-refresh debounce, finding 37) and still skips the write+checkpoint+match tail.
