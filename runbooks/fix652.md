## Change (applied verbatim, markers renumbered fix651→fix652)

New setting `devSkipBackOnResumeSecs` (int seconds, default 0 = off, range
0–30) wired through all five standard layers: `models/settings.dart` (field +
ctor default + reset), `settings_service.dart` (storage key + load/save),
`settings_io.dart` (backup export/import — fix573 lesson), `settings_view.dart`
(slider + help under "Controls auto-hide"). `player.dart._togglePlayPause`:
on resume, if enabled AND mediaType != livestream AND position > N, seek back
N seconds before play(). VOD-only by design — a paused live-DVR stream is
already behind the live edge by the pause length.

## Verify on device (onn)

1. Settings → "Skip back on resume" = 10. Movie, pause ~1 min in, resume:
   restarts ~10 s earlier (chipless — resume seek, not a transport seek).
2. Live TV pause/resume: NO rewind.
3. Setting 0: resume continues exactly where paused.
4. Settings backup → restore round-trips the value.
