## Source
Remaining actionable findings from the 1.31.2+359 cold-eyes review (CRIT-1/HIGH-1 already shipped in fix362).

## MED-1 — DVR cache dir shared across engines, wiped wholesale (mine, fix357)
One fixed `…/free4me_dvr` for every engine; teardown deleted its entire contents. Rapid full-screen A (DVR on) → Back → open B before A's async cleanup finished → A's cleanup unlinks B's in-use cache file, corrupting B's live window. **Fix:** per-engine subdir `…/free4me_dvr/e<identityHashCode>`; cleanup deletes only this engine's subdir, never siblings.

## MED-2 — picker search loaded the entire result set
`channel_picker_screen` looped `page++` with no cap; a short query on a 700k-channel catalog pulled the whole table into memory. **Fix:** cap at `maxPages = 10`; narrowing the query reaches the rest.

## LOW-1 — dead `Media.extras`
`Media.extras = {force-seekable:no}` isn't forwarded to libmpv by media_kit 1.2.6 (no-op). Removed; `force-seekable` is set authoritatively in `_applyMpvOptions` (live: no; DVR: yes); misleading comment corrected.

## LOW-2 — export portal "← Back" dropped the access token
Both result-page back links pointed at `/` (no token) → 403 after fix347's gate. Now include `?t=<token>`.

## LOW-3 — UpdateChecker leaked HttpClient on the error path
`close()` ran only on success. Hoisted the client above the try; closed in a `finally`.

## LOW-4 — `_dfAvailableMB` mis-parsed a wrapped `df` line (mine, fix357)
`df` wraps a long device name onto its own line, so a fixed column index could read the wrong token. Now joins post-header output and takes the 3rd numeric token (Available) — wrap-safe.
