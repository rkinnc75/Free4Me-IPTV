# fix399 тАФ Live DVR: replace deprecated `cache-dir` with `demuxer-cache-dir`

**Target version:** `1.34.27+399` (from `1.34.26+398`)
**Branch:** `main` ┬╖ **Base:** `ccaef76` (v1.34.26+398)
**Toolchain (verified):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.

## What & why

A device log (`v1.34.25+397`, S24) showed libmpv warning on every DVR-enabled open:

> `Warning: property 'cache-dir' was replaced with 'demuxer-cache-dir' and might be removed in the future.`

The Live DVR path in `MpvEngine` sets the on-disk cache location via the old `cache-dir` property (`mpv_engine.dart`, in the `dvrBackMB > 0` block). mpv renamed it to `demuxer-cache-dir`; the old name still works today but logs the deprecation and is slated for removal, so the DVR cache directory would silently stop being honoured on a future libmpv bump. This renames the single call to `demuxer-cache-dir` (verified present in the current mpv option list / `mpv --list-options`). The adjacent `cache-on-disk = yes` is already the current property and is unchanged. Pure property-name correction тАФ no behavior change.
