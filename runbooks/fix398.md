# fix398 — Remove dead `video-sync-min-fps` dev tile + default TLS verification to OFF

**Target version:** `1.34.26+398` (from `1.34.25+397`)
**Branch:** `main` · **Base:** `869e280` (v1.34.25+397)
**Toolchain (verified):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.

## What & why

Two developer-options changes.

**1. Remove the dead `video-sync-min-fps` tile (completes the fix394 review).**
The fix394 review flagged six invalid/mistyped libmpv tunables. The shipped fix394 correction (`509e8b7`) removed five and fixed `audio-spdif`, but **missed `video-sync-min-fps`** — not a real libmpv property (verified against `mpv --list-options`; real video-sync knobs are `video-sync-max-audio-change` / `-max-factor` / `-max-video-change`). The engine still called `setProperty('video-sync-min-fps', …)` (silently ignored by `media_kit`) behind a live "Min resample FPS" slider that did nothing. Removed the field and tile across all sites, matching the pattern used for the other five, plus one stale help-text line pointing at the removed `target-colorspace` tile.

**2. Default TLS certificate verification to OFF.**
`Settings.devTlsVerify` now defaults to `false` (was `true`). Rationale: many IPTV providers serve HTTPS with self-signed certificates, and there is **no per-source TLS toggle** in the UI — the Developer → TLS Verify switch is the only user-facing TLS control. Engine logic is unchanged (`if (ignoreSsl) → no; else → devTlsVerify ? yes : no`); only the default flips. The per-channel `ignoreSsl` (read from import headers, e.g. VLC http-ignore-ssl) still forces `tls-verify=no` regardless of the toggle.

**Security note (explicit):** with this default, certificate verification is off for **all** HTTPS streams unless the user re-enables the toggle — not just self-signed ones, and there is no per-source escape hatch. This is a deliberate product default for an IPTV app; users can turn verification back on under Settings → Developer → TLS Verify.

Both changes are non-breaking on upgrade: a persisted `devTlsVerify` value is still honoured (only the *default* for fresh installs / unset changes), and a leftover `devVideoSyncMinFps` key in an old backup is simply ignored on load. No migration.
