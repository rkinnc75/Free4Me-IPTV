## The two bugs
1. **Search shows hidden categories + divider rows.** The SQL browse path (fix272/278) filters disabled categories (`groups.enabled`) and unplayable `#### ####` dividers (`is_divider`), but the in-memory keystroke-search path (`_searchInMemory`) fetched rows by ID with no such filter — so typing "yes" surfaced hidden Israeli "YES" categories and divider rows.
2. **Picker shows duplicate "Favourites"/"History" headers.** `channel_picker_screen._sectionHeader` emits a header whenever the section label changes vs the previous row, assuming the list is contiguously ordered Favourites → History → All. But the comparator sorted by per-source mode first, so favourites from different sources/modes scattered, making the same header reappear lower down (the two "Favourites" blocks in the screenshots).

## Verified (real sqlite_async + comparator test + compile)
- In-memory fetch with the added filters: searching "yes" returns only the enabled US YES channel; the disabled ISR category and the `##### YES #####` divider are excluded.
- Section-rank-first comparator: favourites from multiple sources collapse under ONE "Favourites" header; each header appears exactly once.
- Full tree compiles: 2 tolerated INFOs, no errors/warnings. Changelog matches the generator.

## Fix 294.1 — `lib/backend/sql.dart`: filter the in-memory search fetch
### Current code (verbatim)

### Replacement code (verbatim)
```dart
    final db = await DbFactory.db;
    // fix294: apply the same divider + disabled-category filters the SQL browse
    // path uses, so the in-memory (keystroke) search does not surface hidden
    // categories or unplayable "#### ####" divider rows.
    final sqlQuery =
        'SELECT * FROM channels c WHERE c.id IN (${generatePlaceholders(ids.length)})'
        ' AND c.url IS NOT NULL'
        '\nAND NOT (COALESCE(c.is_divider,0) = 1 AND '
        'COALESCE((SELECT hide_dividers FROM sources WHERE id = c.source_id),0) = 1)'
        '\nAND COALESCE('
        '(SELECT g.enabled FROM groups g WHERE g.id = c.group_id), 1) = 1';
    final rows = await db.getAll(sqlQuery, [...ids]);
```

### Replacement code (verbatim)

/// fix294: the picker's section grouping (Favourites → History → All channels)
/// in _sectionHeader assumes the list is contiguously ordered by these three
/// tiers. If it isn't, the same header (e.g. "Favourites") repeats wherever a
/// tier resumes. This returns that global section rank so it can be the PRIMARY
/// sort key across ALL sources/modes, keeping each section contiguous.
int _sectionRank(Channel ch) {
  if (ch.favorite) return 0;
  if (ch.lastWatched != null) return 1;
  return 2;
}

/// fix258: provider-aware multi-source sort for the channel picker.
/// For sources in 'provider' mode: favorites first, then provider_order
/// (NULLs last) — matching the SQL browse sort. For 'alpha' mode sources,
/// the 6-tier sort. Between sources, group by provider mode (provider first),
/// then apply within-source sort.
int _pickSortWithProvider(Channel a, Channel b, Set<int> providerSourceIds,
    Set<int> categorySourceIds) {
  // fix294: section grouping first — ALL favourites, then ALL history, then ALL

  // scatter and _sectionHeader emits duplicate "Favourites"/"History" headers.
  final secA = _sectionRank(a);
  final secB = _sectionRank(b);
  if (secA != secB) return secA.compareTo(secB);
