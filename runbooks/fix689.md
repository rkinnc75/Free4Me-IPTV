# fix689 — Pass AVRational by value to av_packet_rescale_ts (v4.0.5)

**Builder:** desktop (Mac). **Live base:** v4.0.4. **Ships as:** v4.0.5+689.

**Why:** fix688 got the re-mux to the packet loop; fix687's `[SRDBG]`
diagnostics then showed `write_frame rc=-22 (after 0 frames)` for every
recording, both MP4 and MKV. Tag re-verified at start and before the diff —
latest remote tag = **v4.0.4**.

---

## Root cause (proven, not inferred)

`av_packet_rescale_ts(pkt, AVRational src, AVRational dst)` takes its two
time-base rationals **by value** (8-byte `{num,den}` structs). The FFI typedef
decomposed each into separate int32 args — the wrong arm64 ABI. Verified against
real libavformat:

- Decomposed-int call → `pts/dts = INT64_MIN` (= `AV_NOPTS_VALUE`).
- Correct by-value `AVRational` → rescales properly (`pts 90000 → 15360`).

Corrupted (NOPTS) timestamps make the first `av_interleaved_write_frame` fail
with EINVAL — exactly the device symptom. And the full algorithm, run in C on a
real H.264+AAC `.ts`, remuxes to a valid MP4 (138 frames) once the rational is
passed correctly — so this is the last blocker.

## Fix
- Add an `AVRational` FFI struct (`{Int32 num; Int32 den}`).
- Change the `av_packet_rescale_ts` typedef to take both rationals **by value**.
- Populate reusable src/dst rationals from the in/out stream time bases at the
  call site.
- fix687 `[SRDBG]` diagnostics retained (permanent, debug-gated).

## Verification

**Performed in sandbox (real):**
- **Proved the ABI bug and the fix** against system libavformat 6.1: decomposed
  args corrupt pts→INT64_MIN; by-value `AVRational` is correct; the complete
  remux of a genuine H.264+AAC `.ts` produces a valid MP4.
- **`dart analyze` (project Dart 3.12.2 + repo `analysis_options`) → "No issues
  found!"**
- Patch applies clean to a **pristine v4.0.4 clone**.

**NOT performed (Mac/CI owns):** full `flutter analyze`, tests, native compile,
arm64 runtime. (Dart FFI by-value struct passing is verified by ABI + the C
reference, not a live arm64 run — the device smoke is the final check.)
