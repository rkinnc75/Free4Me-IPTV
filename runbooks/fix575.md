# fix575 — stream-info appended to the channel name (backlog #9)

**Version:** v2.1.2+575 (shipped with fix573 + fix574)
**Date:** 2026-06-27
**Type:** Player UX (long-standing stream-info label fix)

## Problem

The stream's resolution + codec ("720p H.264") never reliably showed in the
full-screen top bar. The fix515/522 `PlayerStreamInfoLabel` failed for two reasons:

1. **Layout:** it sat AFTER an `Expanded` `PlayerEpgNowLabel` that consumed the row,
   squeezing the stream-info widget to ~zero width.
2. **Timing:** it relied on a one-shot broadcast (`streamInfoStream`) that raced the
   `MaterialVideoControlsTheme` topButtonBar mount (the bar list is read ONCE at
   mount); the fix522 latch seed did not reliably cover the race.

## Change

Append the stream-info onto the always-rendered channel-NAME slot (which sits
BEFORE the greedy `Expanded` EPG label), using the proven self-updating/polling
pattern of `PlayerEpgNowLabel`:

- `lib/widgets/player_channel_name_label.dart` (new): a `StatefulWidget` that seeds
  from `engine.lastStreamInfo`, subscribes to `streamInfoStream` for immediacy, AND
  polls `engine.lastStreamInfo` once a second until it has a value (self-cancelling
  timer). Renders `name` or `name  (720p H.264)`. media_kit freezes the topButtonBar
  list, but a StatefulWidget in that slot freely rebuilds its own subtree — so
  polling sidesteps the frozen-slot race entirely.
- `lib/player.dart`: the topButtonBar's static `Text(channel.name)` is replaced with
  `PlayerChannelNameLabel`; the old `PlayerStreamInfoLabel` block is removed.
- Deleted `lib/widgets/player_stream_info_label.dart` + `test/player_stream_info_label_test.dart`
  (no longer referenced). `streamInfoStream` / `lastStreamInfo` stay on the
  `PlayerEngine` interface and `_publishStreamInfo` is unchanged — the new widget
  consumes the same latch.

## Notes / trade-off

On a very long channel name the appended `(…)` can be ellipsized away inside the
`Flexible(maxLines:1)` — acceptable per the decided approach (the name is never lost,
and most names leave room). Mid-stream resolution changes are not re-emitted (the
first-frame sample is one-shot, same scope as fix515).

## Verification

- `flutter analyze` clean; `flutter test` green (the removed widget's test was
  deleted; no remaining references).
- On-device (manual): tune a live stream; once the first frame lands the top bar
  shows e.g. "ESPN HD  (1080p H.264)".
