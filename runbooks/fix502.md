# fix502 — EPG "what's on" search + epgSearchHours + FTS5 (programmes)

**Version:** 2.0.1+501 → **2.0.2+502**
**Tag:** v2.0.2
**Type:** Feature. New setting + epg.sqlite migration 2 (FTS5) + TV search view.
**Status:** Implemented + verified locally on Flutter 3.44.0 (analyze 0; 168 tests incl. the FTS perf test); shipped via CI.
**Base:** fix501.

## What
- `epgSearchHours` setting (default 3, clamped 1..`epgForecastDays`*24, forward-only), exposed as a "Search window (hours)" slider in Settings → EPG.
- epg.sqlite **migration 2**: `programmes_fts` FTS5 (trigram, external content). **Trigger-free** — rebuilt once after each EPG refresh (`Sql.rebuildProgrammesFts`), so the ~1M-row bulk insert pays NO per-row trigger cost (no refresh regression).
- `Sql.searchPrograms(...)` — forward-only programme-title search (FTS MATCH + window + source filter); `Sql.getLiveChannelsByEpg(...)` resolves matches to live channels.
- TV Search tab (`lib/tv/tv_search_view.dart`): combines channel-name matches with EPG-title→channel matches, grouped Channels / Movies / Series, reusing `ChannelTile` (play, now/next, source edge bar).

## Tokenizer decision (revised)
Trigram, not unicode61+prefix — matches the existing `channels_fts` tokenizer and reuses `Sql.search`'s MATCH builder + ≥3-char gate. Consistent + lower risk.

## Perf gate (passed)
`test/programmes_fts_search_test.dart`: in-memory seed + ANALYZE; EXPLAIN shows `programmes_fts` MATCH (VIRTUAL TABLE) + base reached by PRIMARY KEY, no base-table scan; forward-window + source-filter semantics verified. Absolute <1000 ms cold on the cheapest box remains an on-device confirmation.

## Phone-mode safety
UI is TV-only (`tv_search_view` mounted only by the shell's Search tab). Backend additive: migration 2 is CREATE-only; `searchPrograms`/`getLiveChannelsByEpg`/`rebuildProgrammesFts` are new (no phone callers); `epgSearchHours` uses the null-guarded settings pattern (old blobs default to 3). The "Search window" slider sits in the shared EPG settings group — inert on phone (phone search is channel-name only), not a regression.
