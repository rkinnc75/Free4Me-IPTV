# fix305 — Scanner start-tile math off by row pitch + add category line to long-press menu → v1.26.21+305

## What this does
1. **Scanner start-tile fix.** fix304 computed the first-visible tile with `offset / 100` and no top-padding offset. The real grid row pitch is `mainAxisExtent(100) + mainAxisSpacing(12) = 112`, after a 5px `SliverPadding` top inset. The undersized divisor made the computed row run ahead, so the scan started several tiles *below* the first visible one (reported: set to "Ultra Nature 4K" at top, scan began well below). Now `row = ((offset - 5) / 112).floor()`.
2. **Category line in long-press menu.** Long-pressing a channel now shows a non-interactive header at the top of the menu with the channel's category (its `group`, or "Uncategorized" if none). Display-only.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Built against the uploaded 1.26.20 zip, confirmed byte-identical to live main `home.dart` (fix304 is on main); patch applies cleanly to a fresh clone.
- Scan math checked: at the scroll position where the old `/100` formula returned row 44, the new `/112` (minus 5px pad) returns row 40 — the actually-visible tile. Offset 0 → row 0.

## Note
- The pitch constants (100 extent, 12 spacing, 5 top pad) are duplicated from the grid's `SliverGridDelegate` / `SliverPadding`. If those grid values change later, update `_startScan` to match.
