## Specs (confirmed by Rich, 2026-06-12)
1. **Favorites view:** group by source (A–Z), channels A–Z within each source.
2. **Categories grid:** favorited categories A–Z on top, remaining enabled A–Z below; disabled categories stay hidden (unchanged behaviour).
3. **Settings groups:** expand/collapse state remembered for the current app session only (no persisted setting, not in backups).

## What this does
- **`lib/backend/sql.dart` — Favorites ordering (#5):** in BOTH channel query paths (Sql.search FTS/plain + the LIKE search), the Favorites view (`viewType == favorites`, not inside a series) now gets a dedicated ORDER BY — source name then channel name, both `COLLATE NOCASE` — instead of the BrowseOrder browse clause. The correlated source-name subquery is safe here: favorites lists are tens of rows, not the 270k-row catalogue (the fix344 index concern does not apply to this branch). History ordering and all non-favorites browse paths are byte-identical.
- **`lib/backend/sql.dart` — Categories ordering (#6):** `searchGroup` ORDER BY changes from `favorite DESC, id` (rowid order within tiers, fix308) to `favorite DESC, name COLLATE NOCASE, id` — favorites A–Z, then the rest A–Z; `id` retained as a deterministic tiebreaker. Disabled filtering untouched.
- **`lib/settings_view.dart` — session expand state (#7):** a `static final Map<String, bool> _groupOpen` on the State (static = survives the settings route closing, dies with the process). All 8 ExpansionTiles (`playback, buffering, multiview, content, epg, backuprestore, reset, diagnostics`) now read `initiallyExpanded` from the map and write back via `onExpansionChanged`. No Settings model change, no persistence, no backup involvement — exactly the session-only spec.
- `whats_new_modal.dart` — 1.30.0 changelog (3 entries). `pubspec.yaml` — 1.30.0+356.
- No schema changes — migration ceiling stays **28**, backup schemaVersion stays **4**.

## Rule 8 (executed — both new ORDER BY forms)
Seeded sources Zeta/Alpha + favorites across both + curated groups: favorites query returns Alpha-source channels first, A–Z case-insensitive within each source; groups query returns favorited A–Z then non-favorited A–Z; disabled excluded. ALL ASSERTIONS PASS.
