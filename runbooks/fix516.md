# fix516 — diagnostic logging for the missing stream-info label

**Version:** 2.0.15+515 → **2.0.16+516**
**Type:** Pure diagnostic. No behavior change, no user-visible difference.
**Tag:** v2.0.16
**Base:** v2.0.15 (2.0.15+515).

## Why
fix515's stream-info label ("720p H.264") didn't appear on real-device screenshots for two channels. The log (`free4me_log-sms938u-20260623-173039.txt`) confirms both were **direct/fresh channel opens** (`CREATED engine`, not an adopted/swapped engine), and confirms `_observeFirstFrame`'s `dwidth` observer fired correctly (3–4 times per channel, all `dwidth=1280` — non-zero, passing the guard).

But `_publishStreamInfo()` — the function that reads `dheight`/`video-codec` and emits the label — shipped with **zero logging** (it was written as fully silent/best-effort by design). That made it impossible to tell from the log whether it ran, what it read, where it stopped, or whether the emission ever reached a subscriber. Continuing to theorize from there would be guessing, not diagnosing.

## What
Every branch of `_publishStreamInfo()` now logs:
- the disposed/native-player skip case
- the disposed-mid-read case
- the raw `dheight`/`video-codec` values read and the formatted label (or `null`)
- **whether the emission actually happened, and `_streamInfoCtrl.hasListener` at that exact moment** — this directly tests the live hypothesis: `PlayerStreamInfoLabel` lives inside `topButtonBar`, which `MaterialVideoControlsTheme` only reads once at its own mount (the same fix367/fix405 landmine already documented in `player.dart`); if that mount happens *after* the first `dwidth` callback fires (~2.3s post-open in the captured log), the label's subscription is too late and `StreamController.broadcast()` silently drops the event with no listener to deliver it to.
- any exception from the property reads

No logic changed — this is purely additive logging so the next report from this device is conclusive instead of another round of inference.

## Testing
No test changes (pure logging, no new logic surface). `flutter analyze` clean, **205 tests pass** (unchanged from fix515).
