## #4 — collapse category rail on select (tv_guide_view.dart)

- `_railCollapsed` (init in field decl, NOT reset by `_init`/`reloadGuide`, so a tab
  re-entry keeps the chosen state). The rail is an `AnimatedContainer` 210↔18 px; its
  content is kept laid out at 210 via `OverflowBox` + `ClipRect`, so **every category
  FocusNode stays mounted** (the rail is a non-lazy `ListView`) and LEFT-restore can
  target it.
- `_loadGuide(groupId, {fromUser})`: a user pick (`fromUser:true`, from the rail tap)
  collapses the rail and, post-frame (guarded by `_inv`), `requestFocus`es
  `_firstChannelNode` so focus moves into the channel list (autofocus won't re-fire on
  a list rebuild). The initial load (`fromUser:false`) stays expanded.
- LEFT from any channel (`_onChannelLeftKey` on the channel-column `_FocusTile`):
  **ALWAYS `handled` + ALWAYS** `setState(_railCollapsed=false)` + post-frame
  `requestFocus(_railNodes[_selectedGroupId])`. No fall-through to directional
  traversal (banned here — fix558/562/563 stranded-focus class). `_FocusTile` gained
  `focusNode` + a non-focusable `Focus(skipTraversal)` `onKeyEvent` wrapper (mirrors
  the proven `_surfKeyFocus` observer).

## #6 — long-press → Multi-view (channel_tile.dart + Search/Home + MultiViewScreen)

- `ChannelTile.onOpenMultiView` (a per-caller closure — there is NO shared opener;
  `home._openMultiView` reads Home-only state). A live-gated "Open in Multi-view"
  `ListTile` is added after the mini-player entry.
- Wired at the two live-channel surfaces: `home.dart` (live list) and
  `tv_search_view.dart` (live results), each pushing via the new static
  `MultiViewScreen.openWithChannel(context, settings, sourceIds, channel)` with its
  LOCAL `sourceIds`. (tv_categories is category tiles; tv_browse is Movies/Series —
  the livestream gate means the entry never shows there.)
- `MultiViewScreen.initialChannel`: after `_restoreChannels` completes (via `.then`,
  covering all its exit paths), the channel is dropped into the **first empty cell**
  (cell 0 if all full) — preserves the saved layout. Uses the last layout, default 2×2.
