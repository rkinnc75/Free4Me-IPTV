# fix609 — v2.2.20: startup loading splash ("Preparing…")

The 4th of the Shield startup-freeze fixes (the one deferred from fix608). The
backend fixes (fix608) reduce the freeze but don't remove the ~75s
ChannelSearchCache rebuild on 3 GB+ boxes; this splash MASKS the remaining
warm-up with branding instead of a blank/frozen screen.

## Behaviour
While the startup DB warm-ups run, the home shows a full-screen splash:
- background `assets/tv_background.webp` (same as the TV shell) + an **85% scrim**
  (`Color(0xD9000000)`), a centred spinner, and "Preparing for the best Free4Me
  experience…".
- Fades out (AnimatedSwitcher, 400ms) and is removed when warm-up completes.

## Wiring
- `appWarmupDone` (top-level `ValueNotifier<bool>` in main.dart), false at start.
- main() flips it true in the SAME `Future.wait(warmups)` that gates fix608's
  refreshIfStale — `.timeout(4 min)` safety cap so a hung warm-up can't trap the
  splash. (warmups = ChannelSearchCache.ensureBuilt + warmBrowseCache +
  runPendingIndexMaintenance, each error-swallowed.)
- `_RootPageState.build()`: the Setup path returns early (no splash); the
  TvHome/Home path is wrapped in a Stack with a `ValueListenableBuilder` overlay
  that shows `_StartupSplash` until done. The home builds underneath meanwhile.

## Verify
