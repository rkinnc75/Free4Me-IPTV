# fix142.md — Validated green highlight doesn't persist across sessions: read sites check only the in-memory scan map, not the persisted stream_validated flag

> **Version:** Free4Me-IPTV 1.22.11 build (fix130/132).
> **This is a BUG, not intentional.** Stream validation IS persisted correctly —
> a scan calls `Sql.setStreamValidated(id, ok)` which writes the
> `stream_validated` DB column (and updates the in-memory ChannelSearchCache), and
> channels load that column into `channel.streamValidated`. **But every green-
> highlight READ site checks ONLY the in-memory `StreamScanner.results` map**,
> which is a `static final Map` cleared on every app start. So after a restart the
> DB still says validated, `channel.streamValidated == true`, yet the green
> highlight is gone because `StreamScanner.results` is empty.

---

## Root cause

Validation has TWO sources that must both count as "validated":
- **Persisted:** `channel.streamValidated == true` (the `stream_validated` DB
  column; survives restart).
- **This-session:** `StreamScanner.results[id] == true` (in-memory map; cleared
  on app start — `static final Map<int,bool> results = {}` in
  `stream_scanner.dart`, no load-from-DB).

The scan writes BOTH (`stream_scanner.dart`: `results[ch.id!] = ok;` then
`await Sql.setStreamValidated(ch.id!, ok);`). The PERSISTED side is correct. The
bug is purely on the READ side — three highlight sites check only the session map:

1. **`channel_tile.dart:290`** (Home list green border):
   ```dart
   final scanOk = widget.channel.id != null &&
       StreamScanner.results[widget.channel.id] == true;   // ← ignores streamValidated
   ```
2. **`channel_picker_screen.dart:287`** (picker tile green check + green text):
   ```dart
   final scanOk = ch.id != null &&
       StreamScanner.results[ch.id] == true;               // ← ignores streamValidated
   ```
3. **`channel_picker_screen.dart:170`** (picker section grouping — also touched
   by fix138).

After restart, `StreamScanner.results` is empty → `scanOk` is false everywhere →
no green, despite `stream_validated = 1` in the DB.

> This is the READ-SITE twin of the fix138 sort bug (same two-source mismatch).
> If fix138 is being applied, it introduces a shared `_isValidated(ch)` predicate
> in the picker; fix142 reuses that and adds the equivalent to the Home tile.

# Fix 142.1 — Home tile green border honors the persisted flag

**File:** `lib/channel_tile.dart`

**Current (~lines 289–291):**

```dart
  Widget build(BuildContext context) {
    final scanOk = widget.channel.id != null &&
        StreamScanner.results[widget.channel.id] == true;
```

**Replace with:**

```dart
  Widget build(BuildContext context) {
    // fix142: a channel is validated if EITHER the persisted stream_validated
    // flag is set (survives restart) OR it was validated in this session's scan
    // (in-memory map, cleared on app start). Previously only the session map was
    // checked, so the green highlight vanished after every restart even though
    // the DB still had the channel validated.
    final scanOk = widget.channel.streamValidated == true ||
        (widget.channel.id != null &&
            StreamScanner.results[widget.channel.id] == true);
```

(`channel_tile.dart` already imports `stream_scanner.dart` and has
`widget.channel.streamValidated` — no new imports.)

# Fix 142.2 — Picker tile green check honors the persisted flag

**File:** `lib/channel_picker_screen.dart`

**Current (~lines 287):**

```dart
                    final scanOk = ch.id != null &&
                        StreamScanner.results[ch.id] == true;
```

**Replace with:**

```dart
                    // fix142: persisted OR session validation (see channel_tile).
                    final scanOk = ch.streamValidated == true ||
                        (ch.id != null &&
                            StreamScanner.results[ch.id] == true);
```

> **If fix138 is applied** (it adds `bool _isValidated(Channel ch)`), prefer
> reusing it for consistency:
> ```dart
>                     final scanOk = _isValidated(ch);   // fix142 + fix138
> ```
> `_isValidated` already encodes `streamValidated == true || StreamScanner.results
> [id] == true`. Use whichever form matches what's in the file; the result is
> identical.

## Test plan

### Green highlight persists across restart (THE bug)
1. Scan some channels → they get the green highlight/border.
2. Fully close and relaunch the app.
3. **Expected:** the same channels STILL show the green highlight (read from
   persisted `stream_validated`). Before fix142 the green was gone after restart.

### Fresh session scan still highlights immediately
1. In a new session, run a scan.
2. **Expected:** green appears immediately (session map path still works).

### Clear stream validation
1. Settings → Reset → "Clear stream validation".
2. **Expected:** green highlights disappear (DB flags cleared to NULL) and stay
   gone after restart. (Confirms the persisted read reflects clearing too.)

### Picker matches Home
1. Open the multi-view picker after a restart (with previously-validated live
   channels).
2. **Expected:** validated channels show the green check, sort to the top of
   their group, under the correct header (per fix138 if applied).

### Source refresh
1. Validate channels, refresh the source.
2. **Expected:** validation persists per existing behaviour (the scanner /
   stream_validated survive refresh); green reflects the persisted state.
