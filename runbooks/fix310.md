## What this does
The update checker already detected new versions and showed a dialog whose button opened the GitHub release page in a browser. fix310 makes that button **download the release APK in-app (with a progress dialog) and launch the Android installer** — one tap, then the user confirms the system install prompt. Falls back to opening the release page if no `apkUrl` is present or the download fails.

How:
- `scripts/update_version_json.py` now writes `apkUrl` into `version.json`, pointing at the release asset (`Free4Me-IPTV-${VERSION}-arm64.apk`, matching `release.yml`).
- The dialog's button (now "Update") calls `_downloadAndInstall`: streams the APK to app storage with a % progress + Cancel, then `OpenFilex.open(path)` to hand it to the package installer.
- `AndroidManifest.xml` gains `REQUEST_INSTALL_PACKAGES`. `open_filex` provides its own FileProvider, so no extra provider/XML is needed.

## One-time device requirement (expected, not a bug)
Android sideload installs are never fully silent: the first time, the user must grant Free4Me-IPTV permission to "install unknown apps," then tap Install on the system prompt. The flow shows a guidance snackbar if the installer can't open.

## Verified before delivery
- `flutter pub get` resolves `open_filex ^4.5.0`; `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Built against live main (1.26.25+309); patch applies cleanly to a fresh clone.

- Download path: streamed write to `getApplicationSupportDirectory()`, progress from `contentLength`, Cancel deletes the partial file; failures dismiss the dialog and show a snackbar + leave the browser fallback intact.

## Note
- The `apkUrl` only appears in `version.json` from the **next** release onward (this build regenerates it). The in-app updater reads the live `version.json` on `main`, so once 1.26.26 is released the field will be present for clients.
- Assumes the arm64 APK asset name stays `Free4Me-IPTV-${VERSION}-arm64.apk`. If `release.yml` renames it, update the generator string to match.
