# fix704 ‚Äî TV GUI redesign Phase 3, unit 1: rail focus rings (v4.1.9)

**Builder:** desktop. **Live base:** v4.1.8. **Ships as:** v4.1.9+704. TV only;

## What
The **three copy-pasted `_FocusTile` rails** on the browsing surface ‚Äî the guide
rail (`tv_guide_view.dart`, categories / channels / frozen channel column), the
browse rail (`tv_browse_view.dart`), and the Categories rail
(`tv_categories_view.dart`) ‚Äî focus ring:
- **Ring:** `Colors.yellow` (3px) ‚Üí `AccentScope.of(context)` (3px), read at
  draw time. Matches the tab bar (fix702) and channel/poster tiles (fix703) ‚Üí
  the accent-ring focus language now covers the whole browsing surface.
- `AccentScope.of` is null-safe (`scope?.notifier?.value ?? white`), so even a
  build with no `AccentScope` ancestor is safe (falls back to white). The app
  wires `AccentScope` above the route tree in `main.dart` (fix701).

## Scope boundary ‚Äî global button chrome deferred
The 4 global TV button focus themes in `main.dart` (Filled/Icon/Text/Outlined
`ButtonStyle.side`, all `!hasTouchScreen`-gated ‚Üí the Settings/dialog/gear
chrome) still use `Colors.yellow`. They are a **separate, higher-blast-radius
unit** (every TV button app-wide) that also needs a live-accent reactivity
decision ‚Äî the theme is built once, so reading the accent there means either a
non-reactive `appAccentNotifier.value` read or wrapping the theme build in an
`AnimatedBuilder` on the notifier. Deferred to a dedicated chrome pass with its
own broad dialog/settings device-verify. `test/fix704_guide_focus_test.dart`
asserts `main.dart` still contains `Colors.yellow` so a future merge of the two
scopes is flagged.

## Why this is the first (and smallest) Phase-3 unit
The guide is the 1405-line flagship, highest-risk file (¬ß4.3 of
`docs/TV_GUI_REDESIGN.md`). The Phase-3 plan is explicitly **one concern per
unit, verify each**. Unit 1 is the pure focus-ring color change ‚Äî no layout, no
typography, no data layer ‚Äî so the token wiring is proven to reach the guide
before the riskier units (NOW-line glow + progress-within-cell fill + badges;
genre tint, which needs a net-new `Program.category`‚Üíbucket normalizer;
branded/gap fallbacks; `F4GuideFocusState` formalization) land.

### Width 3 kept on purpose
The rail's 56px `_rowHeight` itemExtent has a tight ~38px content budget (the
18px chrome was tuned in fix603 to fit a 2-line channel name without a
RenderFlex overflow). Changing the border width would move that budget, so the
width stays 3 ‚Äî color is the only change.

### Untouched (kept wins, verify each)
Program grid cells (`_block`) are `ExcludeFocus`/passive ‚Üí no ring, unaffected.
Rail‚Üîgrid Y-alignment (`_rowHeight=56` itemExtent + header spacer + one-way
scroll mirror), :00/:30 timeline snap, 12/24h `guideClockFmt()`, dwell-gated
single muted MPV live preview + low-RAM gate, place-memory, rail SWAP
categories‚Üîchannels, Back-in-channels‚Üícategories, player channel-surf, bounded
queries, error+Retry, and the held-OK **fire-on-release** menu (the P0 win) are
all unchanged.
