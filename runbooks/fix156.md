# fix156.md — TV/D-pad focus: multi-view layout picker (1×2/2×2 unreachable) and settings rows where focus lands only on the info icon

> **Version:** Free4Me-IPTV 1.23.2 build.
> **Device:** onn 4K Plus (Google TV, ARM Cortex-A55, 2GB RAM) — D-pad navigation.
>
> Two D-pad focus bugs, same underlying cause (**non-focusable `GestureDetector`
> for the real action + a focusable widget stealing focus**):
> 1. **Multi-view layout picker:** the 1×2 and 2×2 cards can't be focused — only
>    "Disable multi-view" is reachable. The cards use `GestureDetector`, which is
>    NOT focusable; only the `TextButton` (Disable) takes D-pad focus.
> 2. **Settings rows (e.g. "Default view"):** D-pad focus lands ONLY on the info
>    `IconButton`, so select opens the help dialog and the row's real action
>    (e.g. the Default-view picker) is unreachable. The row's primary action is
>    on `ListTile.onTap`/trailing, but the only focusable child is the inner
>    help `IconButton`, which captures focus.

---

## Root cause

**#1 — `lib/multi_view_picker_dialog.dart`:** each `_LayoutCard` is a
`GestureDetector` (line ~88). `GestureDetector` has no focus node and does not
participate in D-pad traversal. The only focusable widget in the dialog is the
`TextButton` ("Disable multi-view"), so D-pad can reach only Disable.

**#2 — `lib/settings_view.dart`:** the help-icon pattern. Rows like "Default
view" (line ~1736), `_switchTile` (~1042), and `_bufferSlider` (~1071) wrap the
title in a `GestureDetector` containing a **focusable `IconButton`** (the
`_helpIcon`, line ~1023). On TV the focus engine lands on that inner
`IconButton`; the `ListTile.onTap` (the actual picker) and trailing controls are
not directly focusable as the row's primary action, so D-pad select only fires
the help dialog.

> Touchscreen masks both bugs (tap hits whatever you touch). Only D-pad exposes
> them.

---

# Fix 156.1 — Make multi-view layout cards focusable

**File:** `lib/multi_view_picker_dialog.dart`

Replace the `GestureDetector` in `_LayoutCard.build` with an **`InkWell`**
(focusable + D-pad-activatable, gives a focus highlight) and **autofocus the
currently-selected card** so the dialog opens with focus on a real choice, not
on Disable.

**Current (`_LayoutCard.build`, ~line 88):**
```dart
    return GestureDetector(
      onTap: onTap,
      child: AnimatedContainer(
```
**Replace with:**
```dart
    return InkWell(
      onTap: onTap,
      autofocus: isSelected,          // fix156: focus the selected card on open
      borderRadius: BorderRadius.circular(12),
      focusColor: colorScheme.primary.withValues(alpha: 0.25),
      child: AnimatedContainer(
```

> `InkWell` is focusable and responds to D-pad center/Enter. `autofocus:
> isSelected` lands initial focus on the current layout (1×2 or 2×2), so the
> user can immediately D-pad left/right between the cards and down to Disable.
> If `none` is current, neither card autofocuses — fine; D-pad still reaches
> them. Consider also giving the "Disable" `TextButton` a lower visual priority
> so it's clearly the third option.

**Also make the focus order sane:** the two cards are in a `Row` and "Disable"
is below — natural D-pad order is left-card → right-card → down to Disable, which
`InkWell` + the existing layout already yields once the cards are focusable.

# Fix 156.2 — Settings rows: separate the help icon from the row's primary action

**File:** `lib/settings_view.dart`

The fix is to stop nesting a focusable `IconButton` inside a row whose real
action is elsewhere. Two clean approaches — **A (recommended)**: make the help
icon NOT steal the row's primary focus by moving it to `trailing` and making the
ListTile's own tap the focus target; **B**: keep layout but make the whole row a
single focusable target whose select fires the primary action, with help on a
long-press / secondary. A is simplest and matches TV expectations.

### 156.2a — "Default view" row (line ~1736)

**Current:**
```dart
                      ListTile(
                        title: GestureDetector(
                          onTap: () => SettingHelpDialog.show(
                            context,
                            title: _helpDefaultView.title,
                            body: _helpDefaultView.body,
                          ),
                          child: Row(
                            children: [
                              const Text("Default view"),
                              const SizedBox(width: 4),
                              _helpIcon(
                                title: _helpDefaultView.title,
                                body: _helpDefaultView.body,
                              ),
                            ],
                          ),
                        ),
                        subtitle: Text(viewTypeToString(settings.defaultView)),
                        onTap: () async => await _showDefaultViewDialog(context),
                      ),
```
**Replace with:**
```dart
                      ListTile(
                        // fix156: title is plain text so the ListTile itself is
                        // the focusable primary target (D-pad select opens the
                        // picker). Help moves to trailing as a separate focus
                        // stop, so it no longer steals the row's action on TV.
                        title: const Text("Default view"),
                        subtitle: Text(viewTypeToString(settings.defaultView)),
                        trailing: _helpIcon(
                          title: _helpDefaultView.title,
                          body: _helpDefaultView.body,
                        ),
                        onTap: () async => await _showDefaultViewDialog(context),
                      ),
```
Now the row has two focus stops on TV: the ListTile body (select → Default-view
picker) and the trailing info icon (select → help). Both reachable.

### 156.2b — `_switchTile` helper (line ~1034)

**Current:**
```dart
    return ListTile(
      title: GestureDetector(
        onTap: () => SettingHelpDialog.show(
          context, title: help.title, body: help.body,
        ),
        child: Row(
          children: [
            Text(label),
            const SizedBox(width: 4),
            _helpIcon(title: help.title, body: help.body),
          ],
        ),
      ),
      trailing: Switch(value: value, onChanged: onChanged),
    );
```
**Replace with:**
```dart
    return SwitchListTile(
      // fix156: SwitchListTile makes the whole row the focusable target and
      // D-pad select toggles the switch. Help icon is a secondary trailing
      // focus stop via secondary:.
      title: Text(label),
      value: value,
      onChanged: onChanged,
      secondary: _helpIcon(title: help.title, body: help.body),
    );
```
> `SwitchListTile` is the idiomatic focusable switch row: D-pad select toggles
> it; `secondary:` places the help icon as its own focus stop. Cleaner than the
> manual `ListTile`+`Switch`.

### 156.2c — `_bufferSlider` helper (line ~1063)

**Current:**
```dart
    return ListTile(
      title: GestureDetector(
        onTap: () => SettingHelpDialog.show(
          context, title: help.title, body: help.body,
        ),
        child: Row(
          children: [
            Text(label),
            ...
```
**Replace the title** so the label is plain text and the help icon is a separate
trailing/secondary focus stop (mirror 156.2a/b). Keep the slider itself as the
row's interactive control (sliders are D-pad-adjustable with left/right once
focused). Minimal change:
```dart
    return ListTile(
      title: Text(label),                 // fix156: plain text, not a GestureDetector
      subtitle: /* existing slider widget */,
      trailing: _helpIcon(title: help.title, body: help.body),
    );
```
> Verify the slider is reachable/adjustable via D-pad after this; if the slider
> sits in `subtitle`, ensure it has a focus node (Flutter `Slider` is focusable
> and responds to D-pad left/right). If the slider proves awkward on D-pad,
> that's a separate enhancement — the focus-steal fix is the priority here.

   focusable and D-pad left/right adjusts it; info icon reachable separately.

### Touch regression (phone)
1. On a phone, tap the layout cards, default-view row, switches, sliders, and
   info icons.
2. **Expected:** all behave as before — taps hit the right targets.

## Notes for the implementer
- **Root cause is `GestureDetector` (non-focusable) for primary actions + inner
  focusable `IconButton` stealing focus.** The fix makes the PRIMARY action the
  focusable target and demotes help to a separate trailing/secondary focus stop.
- **`InkWell` (not `GestureDetector`) for anything that must be D-pad
  reachable** — going forward, TV-facing tappables should be `InkWell`/
  `TextButton`/`ListTile`/`*ListTile`, never bare `GestureDetector`.
- **`autofocus: isSelected`** on the layout cards gives a sane initial focus.
- **Two files.** Build `1.23.2+156`, tag `v1.23.2`, even = phone.
- Bug #3 (backup/log export "no app" on TV) is a SEPARATE, critical fix —
  tracked separately (it needs a non-SAF export path for TV).
