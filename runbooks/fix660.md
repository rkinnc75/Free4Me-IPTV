# fix660 — first-run pre-warm defaults OFF on phones (v3.0.5)

**Builder:** desktop. **Live base:** v3.0.4. **Ships as:** v3.0.5+649.

## Problem

`preWarmOnFocus` is `true` on first run for EVERY device, phones included —
so phones spend a redirect-HEAD (AppHttp.resolveRedirects) on the grid's
autofocused index-0 tile on every load, which is almost never the channel a
touch user taps. Trace:
- `_readFromDb()` builds `Settings()` → constructor default **true**
  (settings.dart:330).
- The stored value is only applied `if (prewarm != null)`, so a fresh install
  (nothing stored) keeps **true**.
- The device-aware `preWarmOnFocus = isTV` lives ONLY in
  `Settings.optimisedFor` (settings.dart:~500), which runs ONLY from the
  Optimise button — never on first-run load.

So the code's own intent (settings.dart comment: "TVs benefit … phones less so
— touch scrolling sweeps focus rapidly") never takes effect on a normal phone.
This is the same log finding behind the sms938u analysis: prewarm fired on the
autofocused CRIME tile (index 0) while the user played Yankees (`prewarmed=
false`). On touch, the tapped channel has no pre-tap focus/hover signal, and
onTapDown can't win the race against open(), so prewarm can't help touch play —
it's a TV-dwell optimization.

## Fix

In `_readFromDb`, give the UNSET case an explicit OFF fallback instead of
inheriting the constructor's `true`:

```dart
if (prewarm != null) {
  settings.preWarmOnFocus = int.parse(prewarm) == 1;
} else {
  settings.preWarmOnFocus = false; // first run / unset
}
```

Default OFF is unconditional (not device-aware) on purpose: `DeviceDetector
.isTvCached` seeds AFTER settings load in main.dart (getSettings @49 / reload
@54 vs isTV() @85), so the device isn't known this early. A TV defaulting
prewarm OFF on cold first-run is harmless — prewarm is an optimization, not
correctness, and Optimise restores `true`. Mirrors the sibling first-run
fallbacks right below (miniDemuxer/bufferSize/searchMethod).

**Scope:** only the first-run/unset path changes. Existing users with a stored
value (either state) are untouched — including anyone who has deliberately
turned it ON. This does NOT flip a currently-ON phone; that still needs the
Playback → "Pre-warm streams on focus" toggle (or a settings reset).
