## Why (S24 2026-06-12 VOD log series)
Dino delivers /series/ files at ~0.8× realtime: mpv's cache can never get ahead, and with the default 1s resume threshold playback stutters every 2–5 s forever. Z2U delivers above realtime but ramps slowly (~30 s of startup churn). Client config was already correct (VOD branch: cache-secs=60, demuxer 264MB) — the missing piece is mpv's pre-buffer pause: hold playback until N seconds are cached, and refill to N after every underrun. At 0.8× delivery a 15 s cushion yields ~75 s of smooth playback per pause; continuous micro-stutter becomes one short startup wait plus rare, well-spaced refills. Z2U's rough first 30 s disappears behind the initial fill.

## What this does
- **`lib/player/mpv_engine.dart`** (VOD branch only): when `vodPrebufferSecs > 0`, set `cache-pause-initial=yes` and `cache-pause-wait=<secs>`. Live playback untouched (a long pause-wait at the live edge would just freeze the picture).
- **New setting `vodPrebufferSecs`** (default **15**, range 0–60, 0 = old start-immediately behaviour): model + persistence (`settings.dart`, `settings_service.dart`), backup export/import (`settings_io.dart`; old backups import with the default), reset-to-defaults, and a slider tile with full help text in Settings right after "VOD/Movie cache (seconds)".
- `whats_new_modal.dart` — 1.29.2 changelog. `pubspec.yaml` — 1.29.2+354.
- No DB changes (settings live in the kv map) — Rule 8 N/A. Migration ceiling stays 28.
