# fix721 — TV dialog glass + CI pub-cache (v4.1.25)

**Builder:** desktop. **Ships as:** v4.1.25+721. TV only for the UI change;
phone/touch byte-identical. Autonomous ("keep going, pick recommended"). Bundles
two folded changes (owner asked the pub-cache ride along with the in-flight fix,
not a standalone release).

## (a) TV dialog glass — Phase 5 unit 4
Companion to fix720 (bottom-sheet glass): `AlertDialog`/`SelectDialog` surfaces
(confirm dialogs, pickers, the "Re-match complete" notice) were the flat Material
dialog. Added `dialogTheme` to `main.dart` `ThemeData`:
```
dialogTheme: hasTouchScreen
    ? null                                  // phone → Material default (identical)
    : const DialogThemeData(
        backgroundColor: Color(0xF00B0F19), // opaque F4 glassFill (== fix720)
        surfaceTintColor: Colors.transparent,
        shape: RoundedRectangleBorder(
          borderRadius: BorderRadius.all(Radius.circular(20)), // F4Radius.modal, all corners
        ),
      ),
```
Reuses the fix720 glass values (device-verified good on the onn first). Theme-level
→ covers every dialog uniformly, no per-call-site edits. `DialogThemeData` is the
current Flutter type (analyze-confirmed).

## (b) CI pub-cache speedup (folded in, per owner request)
Added an `actions/cache@v4` step (`path: ~/.pub-cache`, `key:
pub-${{ runner.os }}-${{ hashFiles('pubspec.lock') }}`, `restore-keys:
pub-${{ runner.os }}-`) AFTER the Flutter setup step and BEFORE `flutter pub get`
in:
- `.github/workflows/analyze.yml` (analyze job)
- `.github/workflows/release.yml` — the analyze **gate** job AND the
  **build-and-release** job.

Purely additive dependency-download speedup. **Untouched:** the analyze gate
invocation, `needs: analyze`, triggers, concurrency groups, APK_BEFORE_VERSIONJSON
logic — verified by grep + an additions-only diff. Both YAML files validated with
`ruby -ryaml` before commit. Lands in THIS fix's own build (folded, not a
standalone release).
