# fix690 — aac_adtstoasc: make converted recordings play in the default player (v4.0.6)

**Builder:** desktop (Mac). **Live base:** v4.0.5. **Ships as:** v4.0.6+690.

**Why:** recordings convert and play in VLC, but the stock Android "Video
Player" reports *audio codec not supported* (video plays). Tag re-verified at
start and before the diff — latest remote tag = **v4.0.5**.

---

## Root cause (confirmed from device + proven in sandbox)

- VLC Codec Info on the recording: audio = **AAC-LC, 2ch, 48 kHz**; video H.264.
- Live-TV AAC is **ADTS-framed**. Our pinned ffmpeg **n6.0** mp4 muxer does not
  synthesize a valid AudioSpecificConfig (`esds`) from ADTS by itself — it needs
  the `aac_adtstoasc` bitstream filter. The device `.so` even contains the
  message *"use the audio bitstream filter 'aac_adtstoasc'"*. (Sandbox ffmpeg
  6.1 auto-handles it, which is why earlier tests didn't reproduce it — the
  device runs 6.0.)
- Tolerant players (VLC) decode the malformed AAC; strict ones (Android default)
  reject it → "audio codec not supported."
- **Proven:** ran the full `aac_adtstoasc` remux flow against libavformat — 88
  audio frames filtered, output decodes cleanly with a valid `esds`.

## Fix
For **AAC audio into MP4** only:
- `av_bsf_get_by_name("aac_adtstoasc")` → `av_bsf_alloc` → copy input codecpar
  into `par_in`, set `time_base_in` → `av_bsf_init`.
- Copy the filter's **`par_out`** (now carrying the ASC) into the output stream
  **before `write_header`**, so the MP4 `esds` is correct.
- Route audio packets through `av_bsf_send_packet` / `av_bsf_receive_packet`
  (EAGAIN/EOF handled) before writing; video and the MKV path are unchanged.
- Filter unavailable → falls back to a plain copy (fail-open); fix687 `[SRDBG]`
  diagnostics retained.

## Verification

**Performed in sandbox (real):**
- End-to-end `aac_adtstoasc` remux proven against libavformat (clean decode,
  valid `esds`).
- BSF symbols (`av_bsf_get_by_name/alloc/init/send_packet/receive_packet/free`)
  + the `aac_adtstoasc` filter confirmed **EXPORTED** in the vnext arm64 `.so`.
- `AVBSFContext` offsets derived from n6.0 headers (par_in@24, par_out@32,
  time_base_in@40).
- **`dart analyze` (project Dart 3.12.2 + repo `analysis_options`) → "No issues
  found!"**
- Patch applies clean to a **pristine v4.0.5 clone**.

**NOT performed (Mac/CI owns):** full `flutter analyze`, tests, native compile,
arm64 runtime. (BSF flow verified against libavformat + ABI; device is the final
check.)
