## Defects fixed

### A â€” HIGH: held-OK handler desynced from the channel on list shrink/reorder

the then-current `widget.focusNode`. Channel rail tiles were UNKEYED, so
`ListView.builder` reconciled by index: on a list shrink/reorder (e.g. a
favorite toggle that drops a channel) Flutter reused the State at each index and
only swapped its widget â€” `initState` did NOT re-run. The handler then sat on
the old node while the tile rendered a different channel (cross-wired menu/tap),

**Fix:** key the channel tiles â€” `key: ValueKey('gch-${ch.id ?? ch.name}')` in

channel, so a dropped channel's State disposes (clearing its node handler) and
survivors keep their correct Stateâ†”node pairing. The handler closure reads

the old node there would race across pooled nodes on reorder). channel_tile
doesn't hit this because it owns its node per-State; the guide pools nodes in
`_channelNodes` by id.
