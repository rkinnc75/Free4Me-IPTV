# fix543 — Live TV: selecting a category shows "No channels" when you have no live favorites

**Base tag:** `v2.0.42`
**Version:** `2.0.42+542` → `2.0.43+543`
**Scope:** TV Live guide favorites-filter scoping. No schema/query change. Full suite green.

---

## Problem (confirmed from onn device + log)

On Live TV, arrowing to any category (e.g. "USA CBS Locals") and pressing OK showed **"No channels"** — for every category. Device diagnostics ruled out data issues: fix533 stats show all groups enabled (`g_en == g_n` for every source/media_type) and `orphan=0` (joins healthy).

Root cause: fix539 made the Live guide default to favorites-only (`_favOnly = true`). The guide's `_visibleChannels` applied that favorites filter **even when a specific category was selected**, so drilling into a category loaded its channels into `_channels` and then stripped every non-favorite — leaving an empty list for anyone with no live favorites.

## Fix

The favorites filter now applies **only to the top-level "All channels" view** (`_selectedGroupId == null`). Selecting a specific category shows that category's channels regardless of the Favorites toggle. The Favorites pill is also hidden inside a selected category (it would be a no-op there). The favorites-default landing behaviour (kept intentionally) is unchanged for the "All channels" view.

+// fix543: the Live guide's Favorites filter must apply ONLY to the top-level
+// "All channels" view (groupId == null). Inside a selected category the user
+// explicitly drilled in and wants that category's channels regardless of the
+// Favorites toggle — otherwise (with fix539's _favOnly=true default and no live

+// filter rule that _visibleChannels implements.
+import 'package:flutter_test/flutter_test.dart';
+
+// Mirror of TvGuideViewState._visibleChannels' rule, kept tiny and pure so it
+// can be unit-tested without the full widget/DB.
+List<_Ch> _visibleChannels(List<_Ch> all,
+    {required bool favOnly, required int? selectedGroupId}) {
+  return (favOnly && selectedGroupId == null)
+      ? all.where((c) => c.favorite).toList()
+      : all;
+}
+
+class _Ch {
+  final String name;
+  final bool favorite;
+  const _Ch(this.name, this.favorite);
+}
+
+void main() {
+  const cbs = _Ch('CBS Local', false);
+  const espn = _Ch('ESPN', true); // a favorite
+  const all = [cbs, espn];
+
+  test('All view + favOnly → only favorites (the landing default)', () {
+    final v = _visibleChannels(all, favOnly: true, selectedGroupId: null);
+    expect(v.map((c) => c.name), ['ESPN']);
+  });
+
+  test('All view + favOnly OFF → everything', () {
+    final v = _visibleChannels(all, favOnly: false, selectedGroupId: null);
+    expect(v.length, 2);
+  });
+
