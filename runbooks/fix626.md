# fix626 — Redact a source's custom EPG host in logs (was leaking / mislabeled)

**Target:** v2.2.37+626 (diff against live tag **v2.2.36**, verified `git ls-remote --tags`)
**Apply check:** `git apply --check` PASSED on a fresh v2.2.36 clone.
**Analyze:** run on the build machine — gate not asserted here. (Manual: braces + parens balanced.)

## Why (found while diagnosing a Dino EPG failure)
A batch "re-match EPG (all)" failed on Dino with `EPG: download failed … Connection closed while receiving data` — a transient network drop on Dino's EPG endpoint; the manual retry succeeded. NOT an app bug. BUT the log rendered Dino's EPG URL host as `<Trex_HOST>`, which is wrong and misleading.

Root cause: `AppLogger.setSourceSecrets` builds the credential/host redaction table from each source's `url` and `urlOrigin` only — the **manual EPG-URL override host (`source.epgUrl`) is never added**. Dino's guide is served from a different provider's XMLTV host, so that host either (a) leaked in cleartext, or (b) — as here — happened to match a DIFFERENT source's streaming host string and got replaced with that source's tag (`<Trex_HOST>`). fix415's goal was to stop hosts leaking; this was a gap in it, and the mislabeling actively misleads log analysis.

(Xtream's automatic fallback EPG URL is built from the source's own `url` host, which is already redacted — so only the manual override host was exposed.)

## Fix
In `setSourceSecrets`, also resolve `source.epgUrl`'s host and add it to the table under the source's own tag `<NAME_EPG_HOST>`. Longest-first sorting is preserved so it doesn't collide with streaming-host substrings.

## Files
- `lib/backend/app_logger.dart` — add the manual EPG host to the redaction table.
- `pubspec.yaml`, `lib/whats_new_modal.dart` — version bump + changelog.

## Validation plan (build machine)
1. `flutter analyze` → expect "No issues found!".
2. With a source that has a custom `epgUrl` on a distinct host, trigger an EPG download and confirm the log shows `<NAME_EPG_HOST>` for that source's EPG line — not the raw host and not another source's tag.
3. Confirm sources WITHOUT a manual EPG override (Xtream auto-fallback) still redact their EPG host as before (via `<NAME_HOST>`).
4. Confirm normal streaming-host/user/pass redaction is unchanged.
