# fix385 — Edit Source dialog: name + color editable, counts at top, Cancel-default focus, URL validator, Test connection

## What & Why

The Edit Source dialog (reached from a source row's pencil icon) had several gaps that came out of the in-conversation review (issue numbering matches the review list):

- **Bug #1:** Name was not editable in the dialog — title showed it, Save preserved it from the source object, but there was no form field. A rename required delete-and-re-add.
- **UX #4:** Color was not displayed or editable in Edit — only via the leading icon on the source list. The Edit dialog should at least show the current color and let you change it.
- **UX #6:** Save had `autofocus: true`, so D-pad Center / Enter on dialog open fired Save immediately on an unchanged form. Accidental save risk.
- **UX #9:** Read-only info (Connections, Live TV, Movies, Series counts) sat at the bottom of the form, after the editable fields. A user opening Edit to *check counts* had to scroll past the form.
- **UX #10:** The "Counts appear after the next refresh." placeholder only appeared when *all three* counts were null. If one was set and two were `—`, no hint — confusing.
- **UX #11:** URL field had only a `required()` validator. A user could save `"abc"` or whitespace and the source would break on next refresh. The Setup wizard has a real URL validator; Edit should match.
- **UX #16:** No way to test whether a connection still works before saving. For Xtream providers, `fetchXtreamMaxConnections` already does a cheap user-info probe (calls `player_api.php?action=` with empty action). For M3U URL sources, a HEAD on the URL is cheap. For M3U file sources, a `File.existsSync` is cheap. A "Test connection" button in the form, in the dialog, surfaces the result inline.
- **UX #21:** Title `'Edit source ${name}'` had no truncation — a 40-char source name overflowed the dialog title on TV.

Skipped (per review): EPG URL (#2, "no issue for now"), enabled toggle (#3), maxConnections (#5), password pre-reveal (#8), D-pad on-device tests (#7, #19, #20), dirty-check on Cancel (#18), username/password char validation (#12), edge-case checks (#13–#15, #17).

## Design choices

- **#1 rename validator:** Fire-and-forget `Sql.getSources()` in `initState` populates `_otherSourceNames` (a `Set<String>` of all source names except the one being edited). The name field's validator checks `required + non-empty + not in _otherSourceNames`. The original name is allowed through (early return) so opening Edit and hitting Save without renaming works.
- **#4 color swatch:** Reused the existing `showSourceColorPicker` modal (the same one the source list uses) — a tap on the swatch opens the picker, the new color is held in `_color` and persisted on Save. For consistency, the color swatch lives in the info section at the top.
- **#6 autofocus:** Moved `autofocus: true` from Save to Cancel. Cancel is now the D-pad Center / Enter default on dialog open, with a `FocusNode` to ensure the focus moves correctly on TV.
- **#9 reorder:** Info section (color + Connections + Live TV + Movies + Series) is now the first child of the form's `Column`, with a `Divider` between it and the editable form fields. Editable fields (name, URL, username, password, test connection, sort mode, hide dividers) come below.
- **#10 any-null placeholder:** New `anyCountNull` boolean = `liveCount == null || movieCount == null || seriesCount == null`. Placeholder shows when this is true.
- **#11 URL validator:** `Uri.tryParse(value.trim())` then check `uri.host.isEmpty && !value.contains('.')`. Same shape as the Setup wizard's URL field.
- **#16 Test connection:** Single async function `_testConnection()` dispatches by `widget.source.sourceType`:
  - `xtream` → builds a temporary `Source` with the form's current values (URL, username, password) and calls `fetchXtreamMaxConnections`. If it returns null, the result is "Xtream login failed — check URL, username, password."
  - `m3uUrl` → `HttpClient.headUrl` with an 8-second timeout. Any non-2xx status is the error.
  - `m3u` (file picker) → `File(path).existsSync`. Missing file is the error.
  - During the probe, the button shows a spinner and is disabled. Result is shown inline in green (OK) or red (error).
- **#21 title truncation:** `maxLines: 1, overflow: TextOverflow.ellipsis` on the title `Text`.

## Risks

- **Builds on a user-machine state:** the test uses `_otherSourceNames` populated in `initState` via `Sql.getSources()`. If a test rig runs this in an isolated `WidgetTester` without a real DB, the Set stays empty and the rename validator silently allows through. The grep-based regression guard for the validator's structure is what catches the design intent; the actual uniqueness check at runtime is best-effort (the caller can also call `Sql.sourceNameExists` after Save if strictness is required).
- **Network probe in a UI test:** the Test connection button is wired but the actual HTTP/M3U/HEAD calls can't be exercised in a unit test. The widget test covers the *button is present* and the *dispatch logic structure*; the network behavior needs an on-device test (with real provider credentials) to confirm.
- **HttpClient connection state:** `client.close(force: true)` is called after the response is read. If the response stream isn't fully consumed, the close can be delayed. Tested with the simple `await req.close()` → read `statusCode` pattern, which is the standard dart:io idiom.
- **`enabled: false` on the URL field for M3U file sources:** the old code had the URL as a free-form field (it would let you change the file path through the URL field, breaking things). The new code disables the URL field and shows a helper text "Re-pick the file from the source list to change." This is a behavior change — old code allowed editing, new code disallows. Reverting is a single-line change if needed.
