## Root cause (proven from the provider dump + the import code)
During Xtream import, `processXtream` builds `catsMap` = (category_id → category_name) from the provider's `get_*_categories` response, then looks up each stream's `category_id` to set its group. When a stream references a `category_id` that is **absent** from that response, the lookup returns null → the channel's `group` is null → it never becomes a named `groups` row → it cannot be listed, searched, or enabled on the Categories screen.

The user's provider does exactly this: the dump confirms streams reference categories `2183`, `1631`, `680`, etc., but only `804` is actually defined in the category list. So those channels exist in the DB but are orphaned and un-enable-able.

## The fix (Option B — group by id, name by dominant prefix)
Before the main import loop, scan streams whose `category_id` is missing from `catsMap`. Group them **by category_id** (reliable — keeps each category whole), compute the **dominant cleaned name-prefix** among that category's streams, and inject it into `catsMap` as a synthetic name. The existing loop then resolves them normally.

- Prefix = text before the first `|`, with a trailing feed-number tail like `(Peacock 016)` collapsed to `(Peacock)` so numbered feeds of one category share a prefix instead of fragmenting.
- Same-prefix orphan categories **merge into one tile** by design (per the user's choice — fewer tiles).
- A name with no usable prefix falls back to `Category <id>`.
- Real, named categories are never touched.

**Effect timing:** this is in the import path, so it applies on the **next source refresh**. Existing orphaned channels stay null-grouped until re-import.

// fix299: derive a clean category-name prefix from a stream name. Splits on the
// first '|', then collapses a trailing feed-number tail like "(Peacock 016)" to
// "(Peacock)" so numbered feeds of one category share a prefix instead of
// fragmenting into one tile per feed.
String _categoryPrefix(String name) {
  var p = name.contains('|') ? name.split('|').first.trim() : name.trim();
  // "US (Peacock 016)" -> "US (Peacock)"; "Foo (Bar)" stays "Foo (Bar)".
  p = p
      .replaceAllMapped(
        RegExp(r'\s*\(([^)]*?)\s+\d+\)\s*$'),
        (m) => ' (${m[1]})',
      )
      .trim();
  return p;
}

// fix299: for every category_id referenced by a stream but missing from
// [catsMap], pick the dominant [_categoryPrefix] among that category's streams
// and add it to [catsMap] as the synthetic name. Streams whose name has no
// usable prefix fall back to "Category <id>". Mutates [catsMap] in place.
void _resolveMissingCategories(
  List<XtreamStream> streams,
  Map<String, String> catsMap,
) {
  // categoryId -> (prefix -> count)
  final prefixCounts = <String, Map<String, int>>{};
  for (final s in streams) {
    final cid = s.categoryId ?? "";
    if (cid.isEmpty || catsMap.containsKey(cid)) continue;
    final name = s.name?.trim() ?? "";
    if (name.isEmpty) continue;
    final prefix = _categoryPrefix(name);
    if (prefix.isEmpty) continue;
    (prefixCounts[cid] ??= <String, int>{})
        .update(prefix, (c) => c + 1, ifAbsent: () => 1);
  }
  for (final entry in prefixCounts.entries) {
    String? best;
    int bestCount = -1;
    entry.value.forEach((prefix, count) {
      if (count > bestCount) {
        best = prefix;
        bestCount = count;
      }
    });
    catsMap[entry.key] = best ?? 'Category ${entry.key}';
  }
  if (AppLog.enabled && prefixCounts.isNotEmpty) {
    AppLog.info(
      'Xtream.fix299: synthesized ${prefixCounts.length} missing'
      ' category name(s) from stream prefixes',
    );
  }
}
