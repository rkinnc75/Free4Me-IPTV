# fix302 — Opening a category shows all its channels regardless of the enabled checkbox → v1.26.18+302

> ## ⚠️ BUILD MACHINE — APPLY THE PATCH, then RELEASE (EXACTLY). No questions, no DB to query.
> **FIRST:** `find . -name 'fix296-DIAGNOSE*.md' -delete 2>/dev/null || true`

## What this does
The Categories enabled/disabled checkbox correctly hides channels from aggregated views (All / Live / search). But it was also hiding channels when the user **taps into a category** — so an unchecked category looked empty when opened. fix302 skips the enabled filter when `filters.groupId` is set (the user explicitly opened that category). The checkbox still governs every other view.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Patch applies cleanly to fresh main.
- Rule 8 (real SQLite): a disabled category is hidden in the All view (groupId=null) but its channels are returned when browsing into it (groupId set).
- One-line guard in `Sql.search`; no migration, no other view affected.
