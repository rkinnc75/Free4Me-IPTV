# fix715 — TV player OSD Phase 4 unit 2: action-button focus lift (v4.1.19)

**Builder:** desktop. **Live base:** v4.1.18. **Ships as:** v4.1.19+715. TV only;
phone/touch UI byte-identical. Overnight autonomous (owner: Option B + go).
Plan: `lib/player/ai/tv_osd.plan.md`.

## What (Option B — look only, no trigger change)
The overlay action buttons get the Peer2 focus **lift** on top of the accent
**ring** they already draw (fix707's global iconButtonTheme).
- New `lib/player/tv_osd/action_button.dart` — `OsdActionButton`: wraps the same
  `IconButton` in an `AnimatedScale` (1.0→1.15 on focus, `F4Motion.fast` /
  `easeOut`). Observes the IconButton's focus node (the caller's when supplied —
  e.g. play/pause's `_overlayFirstFocus` autofocus node — else an internal
  `_own`), and in dispose removes its listener + disposes ONLY a self-created
  node (a caller node is theirs). `onInteract` fires before `onTap` (preserving
  `_ovlButton`'s old onPressed → `_resetOverlayHideTimer`).
- `player.dart` `_ovlButton(...)` now returns `OsdActionButton(...)` — every
  overlay-button call site (back/cast/pip/subtitles/audio/surf/seek/aspect/
  play-pause/mini) is UNCHANGED.

## Why low-risk
`AnimatedScale` is a paint/hit-test transform, not a focus boundary, so D-pad
traversal, `OK` activation, the fix707 ring, hit-testing, and the play/pause
first-focus autofocus are all untouched. The reveal trigger (short OK →
play/pause + reveal), `_navMode`, `_onPlayerKey`, channel-surf, and seek are
untouched (Option B was chosen precisely to avoid the risky trigger change).

## Not in this unit
The long-press "actions on demand" trigger (Peer2 unit-2 default) — dropped per
Option B. Track/speed popups — deferred (subtitles/audio already open pickers;
speed presets are a separate feature). Next: **fix716 Channel Bar** (unit 3).
