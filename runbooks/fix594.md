# fix594 — Export: "Everything (incl. databases)" zip + max compression

**Version:** v2.2.7+596
**Date:** 2026-06-28
**Type:** Feature (backup/export)

## Changes (`lib/settings_view.dart` `_buildExportBundle`)

1. **Existing zip → level 9.** `ZipFileEncoder.addFile` previously used the
   `archive` package default (`DeflateLevel.bestSpeed`, level 1). Now the text
   bundle (source dumps + log + settings) deflates at **level 9**. The dumps
   dominate the zip and compress well; `addFile` streams the input (archive 4.x,
   32 KB deflate window) so max compression can't OOM. The zip's FILE SET is
   unchanged.
2. **New "Everything incl. databases" zip** (`free4me-export-all-<stamp>.zip`):
   the small zip's contents PLUS `db.sqlite` + `epg.sqlite` in one download.
   - Text files: deflate level 9.
   - DB files: **level 0 (stored, not compressed)** — SQLite barely compresses
     and deflating a multi-hundred-MB DB is pure cost; archive 4.x honors the
     per-file level (`level ?? file.compressionLevel ?? _data.level ?? 6`) and
     streams the input, so this is fast and OOM-safe.
   - **Credential-gated** (`includeCredentials`) — the DB embeds Xtream
     user/pass, same gate as the standalone DB downloads.
   - The original small zip is left exactly as-is (a separate item).

## Verification (on-device, manual export flow)

- Settings → export WITHOUT credentials → only the small `…-export-<stamp>.zip`
  (+ dumps/log/settings standalone); NO "Everything" zip; NO DBs.
- Export WITH credentials → both the small zip AND
  `…-export-all-<stamp>.zip` (which contains the text files + db.sqlite +
  epg.sqlite). Confirm the everything-zip builds without OOM on the onn and the
  small zip is meaningfully smaller than before (level 9 vs bestSpeed).
- Unzip the everything-zip on a desktop → DBs open in a SQLite viewer (stored
  intact).
