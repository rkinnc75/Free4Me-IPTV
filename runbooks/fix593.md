# fix593 — "On now"/"Coming up" search shelves empty (stale programmes_fts)

**Version:** v2.2.6+593 (with fix592 search diagnostics)
**Date:** 2026-06-28
**Type:** Search bug fix + diagnostics (diagnosed from the on-device log)

## Root cause

TV search builds "On now" / "Coming up" ONLY from `Sql.searchPrograms` (EPG
programme-title matches). The diagnostic log showed `epgProg=…/0` on EVERY query
(`programmes.length == 0`) even though the guide rendered NOW/NEXT — i.e. the
`programmes` table HAS data, but the `programmes_fts` (trigram, external-content)
index returns nothing. `programmes_fts` is only (re)built after an XMLTV EPG
refresh (`epg_service.dart` → `rebuildProgrammesFts`); it is NEVER rebuilt on a
plain source refresh or on app start. So an install whose FTS is empty/stale
silently shows empty On-now/Coming-up forever (the trigram query itself is fine
— it worked per fix556).

## Fix

- `lib/backend/db_factory.dart` (EPG db open, fix593): after migrations, compare
  `count(*) FROM programmes` vs `count(*) FROM programmes_fts`. If programmes
  exist but the FTS count disagrees, rebuild the FTS once
  (`INSERT INTO programmes_fts(programmes_fts) VALUES('rebuild')`) and log
  before/after counts. Cheap counts; the rebuild runs only when actually stale.
  Self-heals at the next app start.
- `lib/tv/tv_search_view.dart` (fix592): the `TvSearch._run` log line now prints
  the per-shelf breakdown `shelves[onNow= comingUp= channels= movies= series=]`
  so a "missing series/movies/On-now" report is diagnosable straight from the
  log (which media types the match returned), separating data from code.

## Verification

- `flutter analyze` clean; `flutter test` green (272).
- **On-device (onn):** launch v2.2.6 — the log shows
  `EpgDb: programmes=N programmes_fts=M`; if M was stale it logs a rebuild to N.
  Then search a programme title currently airing → "On now"/"Coming up" populate
  and `epgProg=…/>0`. The new `shelves[…]` log line confirms per-type counts
  (use a "fox" search to settle the series-count question — data vs name-match).
