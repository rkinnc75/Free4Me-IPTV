# fix643 — TV: D-pad LEFT-at-edge = Back, exit-confirm always on for TV

**Target tag:** `v2.2.42` (commit `905c1bed48502e640594a5ea6628f2b3c78dd50c`)
**New version:** `2.2.43+632`
**Migration:** none

## Summary

TV-mode navigation feature (backlog: back-button-to-Categories, scoped with
Rich 2026-07-02). Design agreed:

- **Back chain** (already works via pushed routes — verified, no change):
  player -> channel list -> Categories/parent tab -> root.
- **LEFT-at-edge = Back (new):** on a TV grid (drilled-in category / series /
  channel list — every drill-in pushes the `Home` body, from both the phone-style
  tabs and `TvBrowseView`), a D-pad LEFT from a tile in the **leftmost column**
  pops the route, exactly like Back. Non-edge tiles keep normal directional
  traversal. KeyDown only — a held LEFT cannot pop multiple screens.
- **At a root tab LEFT does nothing** (only the Back button arms the exit
  confirm) — a fast horizontal scroll can never accidentally exit.
- **"Press Back again to exit" always on for TV:** `TvShell`'s
  `ConfirmExitScope` is now unconditionally enabled (was gated on the
  `confirmToExit` setting, default OFF). The setting still governs the phone.

Implementation piggybacks the existing `ChannelTile` focus-node key handler
(same slot as the fix558/563 right-edge + up-escape handlers — deterministic
column math, NOT `focusInDirection` probing, per the fix558/562/563 lessons).
`Home` computes `leftEdge` as `index % crossAxisCount == 0` (TV only,
`!hasTouchScreen`) and the callback guards `Navigator.canPop()` so root tabs
ignore it.

## On-device test notes (onn)

1. Drill into a category -> focus a first-column tile -> LEFT -> should return
   to the parent grid (same as Back).
2. Held LEFT from a first-column tile -> should pop ONE level only.
3. At a root tab (Movies/Series/Categories), LEFT on the leftmost column ->
   nothing (focus stays; no exit toast).
4. Back at root -> "Press Back again to exit" toast (now regardless of the
   Confirm-to-exit setting) -> second Back exits.
