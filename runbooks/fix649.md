## Problem (from the work order)

On TV, ◀/▶ during full-screen playback seek only when a live DVR window is
active (`playerKeyAction` gated on `dvrActive`, fix576). On movies/series the
keys did nothing — media_kit's bars aren't D-pad reachable and the direct key
path was never wired for VOD. Seeks were also silent (log-only feedback).

## Change (applied verbatim from the work order diff, comments renumbered)

- `player_key_action.dart`: seek gate renamed `dvrActive` → `canSeek` (pure
  function; caller decides seekability).
- `player.dart`: `_canSeekTransport` getter (`dvrActive || mediaType !=
  livestream`); `_dvrSeekBy` → `_seekBy` (adds duration clamp for VOD; zero
  floor unchanged; mpv owns the live-edge clamp); all four call sites updated.
- New skip chip: centered `+10 s` / `−10 s` pill accumulating net seconds
  across rapid presses, fades 900 ms after the last; timer cancelled in
  dispose.
- `test/fix576_player_key_action_test.dart` updated for the rename (adds the
  VOD-true / plain-live-false cases).

Work-order provenance: diff generated mechanically by the phone coder,
chained `git apply --check` verified on a fresh v2.2.47 clone; code hunks
applied verbatim here (pubspec/whats_new hand-renumbered: their v2.2.48 →
this v2.2.49).
