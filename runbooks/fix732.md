# fix732 — Channel-zap black shutter (mock §4.7) — v4.1.35

TV only. Gap-audit item #3. Masks the fresh-play black-load with a clean fade
instead of a raw black flash.

## How
- `mpv_engine.dart`: new broadcast `firstFrameStream` fires once when the real
  first frame is sized (inside the existing `dwidth` 0→WxH observe, `_observeFirstFrame`,
  registered at open() for full-screen plays); closed in dispose.
- `player_engine.dart`: `Stream<void> get firstFrameStream => const Stream.empty()`
  default (non-mpv engines).
- `player.dart`: `_buildZapShutter()` = a full-bleed black `AnimatedOpacity`
  (F4Motion.shutter 150ms, easeOut) above the video, below the buffering
  spinner/banner/OSD. `_showShutter` starts true on a fresh play, cleared by the
  `firstFrameStream` event (`_clearShutter`); a 4s `_shutterTimeout` fallback
  clears it if the signal never fires (dead/audio-only stream). Adopted
  (already-rendering, swap) engines skip the shutter (`_showShutter=false`).

## Verify
`test/fix732_zap_shutter_test.dart` (4). Whole-project analyze clean.
- onn: open a channel / surf ▲▼ → brief black → fades to video on first frame
  (no previous-frame flash). Audio-only/dead stream → black ≤4s then clears.
  Swap/maximize (adopt) → no black flash.
