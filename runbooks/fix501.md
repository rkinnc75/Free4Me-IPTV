# fix501 — Per-source edge bar + TV focus ring (rail deferred to fix503)

**Version:** 2.0.0+500 → **2.0.1+501**
**Tag:** v2.0.1
**Type:** Feature (small, TV-only). Additive + gated.
**Status:** Implemented + verified locally on Flutter 3.44.0 (analyze 0, flutter test green); shipped via CI.
**Base:** fix500.

## What
TV-only, gated on `!hasTouchScreen` (phone byte-for-byte unchanged):
- **Per-source edge bar** — a left-flush, full-saturation source-color strip on each TV channel tile, over the existing `SourcePalette.tintOver` inner shade (fix196). Null source = no bar.
- **TV focus ring** — a focused TV tile shows the app-standard yellow border; phone keeps elevation-only focus.

Implemented by REUSING the proven `Home` browse (play/pagination/search/scan intact) via a new gated `ChannelTile.showSourceEdgeBar` param (default false) + one `Home` call-site flag (`!widget.hasTouchScreen`). No TV browse rewrite.

## Scope change (decided autonomously, stability-first)
The **category rail** originally planned here moved to **fix503**, where it is architecturally required to rail-scope the EPG guide grid. Shipping the edge bar alone keeps this release small and fully verifiable.

## Phone-mode safety
`ChannelTile.showSourceEdgeBar` defaults false; the phone `Home` passes false. `clipBehavior`, the yellow focus border, and the edge bar are all gated on that flag — phone rendering unchanged.

## Tests
`flutter analyze --no-fatal-infos` = "No issues found!"; `flutter test` = all 166 pass (verified locally on 3.44.0). CI `analyze.yml` + `release.yml` are the gate.
