# fix706 — TV GUI redesign Phase 3 unit 4: guide gap placeholders (v4.1.11)

**Builder:** desktop. **Live base:** v4.1.10. **Ships as:** v4.1.11+706. TV only;
phone UI byte-identical. Standing autonomous GUI mandate.

## What
`lib/tv/tv_guide_view.dart` — "never a blank/broken cell". Channels with no EPG
programmes in the current window (24/7 loop / VOD-style feeds with no XMLTV — the
common case for many bundle channels; verified on the onn: ABC affiliates + Hulu
Originals all render empty) previously showed a blank grid row. Now they show a
dim full-width **"No guide data"** placeholder.

- New `_emptyRowPlaceholder(width)`: a muted `surfaceContainerHighest`@0.35
  full-width `Container` (radius 4), left-aligned "No guide data" (fontSize 11,
  onSurface@0.45). Styled like a muted programme cell; the NOW line still draws
  over it.
- `_gridRow` `Stack`: `if (progs.isEmpty) _emptyRowPlaceholder(c.maxWidth)` added
  before the real-cell `for`-loop + `_nowLine`.

## Why reordered ahead of unit 3 (genre tint)
Units 3/4/5 are independent and reorderable (`docs/TV_GUI_REDESIGN.md` §Phase
plan). On the onn the guide is EPG-sparse (loop channels have no XMLTV), so:
(a) unit 3's on-now genre tint has no cells to tint → not device-verifiable now;
(b) unit 4 *fixes the exact blank-grid state* observed and IS verifiable now; and
(c) unit 4 is lower risk (no data-layer normalizer). So 4 shipped first.

## Purely additive (why populated rows are safe)
The placeholder is only built in the `progs.isEmpty` branch. When `progs` is
non-empty the collection-if contributes nothing and the real-cell `for`-loop +
`_nowLine` are byte-identical to fix705. No data layer, no Program change.
