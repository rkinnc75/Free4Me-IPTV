# fix258 — Provider sort: favorites first, then pure provider order (+ 1.26.0 changelog & back-fills)

> **Version:** this IS the **1.26.0** release (build 258). Tag `v1.26.0`.
> **Builds on:** the shipped **1.25.7+256** source.
> **Bug (seen after fix256 shipped):** in 'provider' sort mode, channels still looked out of order once some had been stream-validated or watched. Cause: the browse sort applied the full 6-tier sort (favorite / validated / watched) BEFORE provider_order, so a validated or watched channel jumped above the provider's `#### SECTION ####` header and fragmented the curated blocks.
> **Fix (your choice — "favorites on top, then provider order"):** in provider mode, collapse the tier sort to just "favorites first", then order by provider_order. The validated/watched tiers no longer reshuffle a provider-ordered list. Alpha mode keeps the full 6-tier sort unchanged.

## Fix 258.3 — `lib/channel_picker_screen.dart`: picker (multi-view + elsewhere) respects provider order

The channel picker (`ChannelPickerScreen`, opened by multi-view cell `+`/Replace and reused across the app) calls `Sql.search` (so it gets the SQL sort) but then RE-SORTS in Dart via `_pickSort`/`_channelTier`, which had no knowledge of provider order — throwing the SQL provider order away. This makes the picker (and therefore 1×2/2×2 channel assignment) honor the same rule: provider-mode sources → favorites first, then provider order.

### 258.3a — add a provider-aware comparator
Current code (verbatim — match exactly):
```dart
int _pickSort(Channel a, Channel b) {
  final ta = _channelTier(a);
  final tb = _channelTier(b);
  if (ta != tb) return ta.compareTo(tb);
  return a.name.toLowerCase().compareTo(b.name.toLowerCase());
}
```
Replacement code (verbatim):
```dart
int _pickSort(Channel a, Channel b) {
  final ta = _channelTier(a);
  final tb = _channelTier(b);
  if (ta != tb) return ta.compareTo(tb);
  return a.name.toLowerCase().compareTo(b.name.toLowerCase());
}

/// fix258: provider-aware comparator for the picker, mirroring the SQL browse
/// sort. For channels whose source is in 'provider' mode: favorites first,
/// then the provider's order (providerOrder); a channel with no providerOrder
/// falls back to name. For all other channels: the standard 6-tier sort.
/// [providerSourceIds] is the set of source ids currently set to provider mode.
int Function(Channel, Channel) _pickSortWith(Set<int> providerSourceIds) {
  bool isProvider(Channel c) => providerSourceIds.contains(c.sourceId);
  return (a, b) {
    final ap = isProvider(a);
    final bp = isProvider(b);
    // Keep each source's channels internally consistent; if exactly one is a
    // provider-mode channel, fall back to the tier sort for a stable mix.
    if (ap && bp) {
      // favorites first
      final fa = a.favorite ? 0 : 1;
      final fb = b.favorite ? 0 : 1;
      if (fa != fb) return fa.compareTo(fb);
      final oa = a.providerOrder;
      final ob = b.providerOrder;
      if (oa != null && ob != null && oa != ob) return oa.compareTo(ob);
      if (oa != null && ob == null) return -1;
      if (oa == null && ob != null) return 1;
      return a.name.toLowerCase().compareTo(b.name.toLowerCase());
    }
    return _pickSort(a, b);
  };
}
```
