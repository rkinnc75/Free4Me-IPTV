# fix714 — TV player OSD Phase 4 unit 1: Info Bar (v4.1.18)

**Builder:** desktop. **Live base:** v4.1.17. **Ships as:** v4.1.18+714. TV only;
phone/touch UI byte-identical. Overnight autonomous run (owner approved Phase 4).
Plan: `lib/player/ai/tv_osd.plan.md`.

## What
First unit of Phase 4 (Peer2 three-bar player OSD). Replaces the flat top-bar
info scatter with a bottom **Info Bar**.
- New `lib/player/tv_osd/info_bar.dart` — `PlayerInfoBar`: a display-only
  token-glass `Container` (glassFill + glassStroke, radius.card). Row: channel
  logo (`CachedNetworkImage` 44×44, contain, errorWidget→empty; only when
  `channel.image` non-empty) + `PlayerChannelNameLabel` (styled via
  DefaultTextStyle; the label has no own colour/size). Below: `PlayerEpgNowLabel`
  when `live` (self-hides when no EPG); the caller's `_buildOverlayProgress()`
  row when `seekable`.
- `player.dart` `_buildTvOverlay`: top bar slimmed to `[Back, Spacer, Cast, PiP]`
  (name + NOW/NEXT removed); Column now `[topBar, Spacer, PlayerInfoBar,
  SizedBox(8), bottomBar]`; the standalone `if (seekable) _buildOverlayProgress()`
  is folded INTO the Info Bar (`progress: seekable ? _buildOverlayProgress() :
  null`).

## Why low-risk (chrome only)
Reuses the existing self-updating label widgets + the caller's progress row — no
new data sources. `PlayerInfoBar` has **no focusable children** (no FocusNode /
InkWell / GestureDetector), so the overlay's `FocusTraversalGroup` D-pad
traversal, the `_overlayFirstFocus` playPause autofocus, `_navMode`,
`_onPlayerKey`, channel-surf (`_surf`), seek, and auto-hide are all untouched.
The **touch (non-TV) control path** (a different method, ~player.dart:2429) still
uses the labels directly — unchanged (imports stay valid).

## Not in this unit (later Phase 4 units)
fix715 Actions Bar (bottomBar buttons → TvFocusable + 2→8dp lift + long-press
trigger + track/speed popups); fix716 Channel Bar (D-pad Center → horizontal

badges deferred (no reliable per-programme quality data — same call as the guide).
