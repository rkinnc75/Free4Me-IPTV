# fix718 — TV GUI Phase 5 unit 1: Recordings accent focus ring (v4.1.22)

**Builder:** desktop. **Ships as:** v4.1.22+718. TV only; phone/touch UI
byte-identical. Overnight autonomous (owner: "go through the GUI, most-logical
order"). First Phase 5 unit. Plan: `docs/TV_GUI_REDESIGN.md` §4/§6 (Phase 5).

## What
The Recordings list rows were plain Material `ListTile`s whose focus showed only
Material's faint default overlay — the ONE remaining TV surface not on the accent
focus-ring language (tabs=702, tiles=703, rails=704, buttons/gear=707, player
OSD=715). fix718 gives the focused recording row the shared accent ring on TV.

## How (phone-safe by construction)
- `RecordingsView` + `final bool tv` (default **false**). Only `tv_shell.dart`
  (the `!hasTouchScreen` path) builds `RecordingsView(tv: true)`; phone
  `bottom_nav.dart` is untouched → `tv=false`.
- `_RecordingTile` threads `tv`; `_RecordingTileState` adds `bool _focused` fed by
  a `_node` focus listener (`addListener` initState / `removeListener` dispose,
  `mounted`-guarded, setState only on change).
- build(): assembles the `ListTile` into `tile`; **`if (!widget.tv) return tile;`**
  (phone = exactly the pre-change widget). On TV, wraps `tile` in an
  `AnimatedContainer` (`F4Motion.fast`, easeOut) with
  `Border.all(width: 2, color: _focused ? AccentScope.of(context) : transparent)`
  + `BorderRadius.circular(t.radius.card)`.

Design notes:
- The 2px border is **always allocated** (only its color changes on focus), so
  entering/leaving focus causes **no layout reflow** — no list jitter on D-pad
  travel. The `ListView.separated` Dividers sit outside each wrapped tile.
- The ring is paint-only around the `ListTile`; the tile keeps `focusNode`/
  `onTap`/`onLongPress` and the **fix693 held-OK model** (KeyDown 600ms timer →
  KeyUp fires onDetails if held else onTap) is untouched. The focus listener and
  `onKeyEvent` are independent.
