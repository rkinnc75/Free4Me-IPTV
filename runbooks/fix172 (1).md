# fix172 — Multi-view D-pad audio focus: compile break + TV focus seeding

> **Version:** 1.23.8+170 → **1.23.9+172**
> **Evidence:** `flutter analyze --no-fatal-infos` on current `main` source (zip dated
> 2026-05-30) → **7 fatal issues** at `lib/multi_view_cell.dart:859–860`
> (`expected_token` / `missing_identifier` / `unexpected_token` + `dead_code`).
> On-device log `free4me_log-20260530-134945.txt` was captured on **1.23.7 build=168**
> (line 11: `App started — version=1.23.7 build=168`), so it does **not** exercise
> fix170.2 at all.
> **Bug:** (1) `main` does not compile — fix170.2's `_buildVideoCell` wrapper has one
> orphan closing `)`. (2) Even once it compiles, multi-view has no initial focus target
> and no traversal containment, so on a pointer-less TV the D-pad cannot reliably move
> audio focus between cells.
> **Root cause:** the fix170.2 atomic re-apply was still off by one closer; and the
> `FocusableActionDetector` cells were added without `autofocus` or a
> `FocusTraversalGroup`, leaving the audio/visual focus model (`_focusedCell`) with no
> framework focus node seeded at entry.

## Root cause (current code)

### A. The compile break — `lib/multi_view_cell.dart`, `_buildVideoCell` (805–862)

The widget tree is `FocusableActionDetector( … child: GestureDetector( … child: Stack( …
children: [ … ] ) ) )` — three nested calls plus one list. The current tail closes one
level too many:

```dart
        ],            // 856  closes children: [   ✓
      ),              // 857  closes Stack(        ✓
        ),            // 858  closes GestureDetector(   ✓
      ),              // 859  closes FocusableActionDetector(   ✓
    );                // 860  ← ORPHAN: nothing left open before `return`
  }
```

`return FocusableActionDetector(...)` is fully closed at line 859; line 860's `);` has no
matching opener. The inner `Stack(` was also under-indented (line 817 `fit:` sits at the
same column as `child: Stack(`), which is what let the miscount slip through by eye.

### B. No initial focus on TV — `lib/multi_view_cell.dart` / `lib/multi_view_screen.dart`

`FocusableActionDetector` defaults `autofocus: false` (confirmed against
api.flutter.dev/…/FocusableActionDetector.html). Nothing in `multi_view_screen.dart`
autofocuses or wraps the grid in a focus scope. On a TV (no pointer to seed focus), at
entry **no node is focused**, while `_focusedCell = 0` already paints the blue border +
volume-up icon on cell 0. The first D-pad press therefore moves *real* focus (likely to
the AppBar close button) without `onShowFocusHighlight` ever having fired for cell 0 →
the audio/visual model desyncs from framework focus.

### C. No traversal containment — `lib/multi_view_screen.dart` body (≈235)

With cells focusable but ungrouped, D-pad traversal can escape to the AppBar between
cells instead of moving cell→cell. A `FocusTraversalGroup` around the grid keeps traversal
inside the cells.

                  decoration: BoxDecoration(
                    border: Border.all(
                      color: Theme.of(context).colorScheme.primary,
                      width: 3,

            Positioned(
              left: 0,
              right: 0,
              bottom: 0,
              child: _buildInfoBar(),

`widget.cellIndex` already exists on the widget (constructor field, line 44) — no new
param. On an all-empty grid, cell 0 renders `_buildEmptyCell()` (channel == null), so
`_buildVideoCell` isn't built and autofocus does **not** steal focus from the "+" FAB —
it only takes effect once cell 0 has a channel.

# Fix 172.3 — Contain traversal to the cells (`FocusTraversalGroup`)

**File:** `lib/multi_view_screen.dart`, `build()` body (≈233).
Wrap the existing `LayoutBuilder` in a `FocusTraversalGroup`. **This is a two-end wrap;
apply it as the single edit shown (replace the whole `body:` block) or replace the entire
file — do not hand-patch one end** (PREFLIGHT rule 6).

**Current (≈233–269):**
```dart
      body: !_restored
          ? const Center(child: CircularProgressIndicator())
          : LayoutBuilder(
              builder: (context, constraints) {
                final w = constraints.maxWidth;
                final h = constraints.maxHeight;
                final isLandscape = w >= h;

                if (widget.layout == MultiViewLayout.oneByTwo) {
                  // Portrait → stack vertically; landscape → side by side.
                  // Both use Expanded children so each cell fills its half.
                  return isLandscape
                      ? Row(children: _buildFlexCells())
                      : Column(children: _buildFlexCells());
                }

                // 2×2 — always 2 columns × 2 rows.
                // childAspectRatio = cellWidth / cellHeight
                //   cellWidth  = (w − 1 gap) / 2
                //   cellHeight = (h − 1 gap) / 2
                // This makes the grid fill the available body exactly in
                // both portrait and landscape without overflow or black bars.
                const gap = 2.0;
                final cellAspect =
                    ((w - gap) / 2) / ((h - gap) / 2);
                return GridView.count(
                  crossAxisCount: 2,
                  childAspectRatio: cellAspect.clamp(0.1, 10.0),
                  mainAxisSpacing: gap,
                  crossAxisSpacing: gap,
                  padding: EdgeInsets.zero,
                  physics: const NeverScrollableScrollPhysics(),
                  children: _buildGridCells(),
                );
              },
            ),
```

**Replacement:**
```dart
      // fix172: contain D-pad traversal to the cells so focus moves
      // cell→cell instead of escaping to the AppBar between presses.
      body: !_restored
          ? const Center(child: CircularProgressIndicator())
          : FocusTraversalGroup(
              child: LayoutBuilder(
                builder: (context, constraints) {
                  final w = constraints.maxWidth;
                  final h = constraints.maxHeight;
                  final isLandscape = w >= h;

                  if (widget.layout == MultiViewLayout.oneByTwo) {
                    // Portrait → stack vertically; landscape → side by side.
                    // Both use Expanded children so each cell fills its half.
                    return isLandscape
                        ? Row(children: _buildFlexCells())
                        : Column(children: _buildFlexCells());
                  }

                  // 2×2 — always 2 columns × 2 rows.
                  // childAspectRatio = cellWidth / cellHeight
                  //   cellWidth  = (w − 1 gap) / 2
                  //   cellHeight = (h − 1 gap) / 2
                  // This makes the grid fill the available body exactly in
                  // both portrait and landscape without overflow or black bars.
                  const gap = 2.0;
                  final cellAspect =
                      ((w - gap) / 2) / ((h - gap) / 2);
                  return GridView.count(
                    crossAxisCount: 2,
                    childAspectRatio: cellAspect.clamp(0.1, 10.0),
                    mainAxisSpacing: gap,
                    crossAxisSpacing: gap,
                    padding: EdgeInsets.zero,
                    physics: const NeverScrollableScrollPhysics(),
                    children: _buildGridCells(),
                  );
                },
              ),
            ),
```

**Key changes:** single `FocusTraversalGroup` wrapper around the `LayoutBuilder`; inner
block re-indented by two spaces. `FocusTraversalGroup` needs no import beyond the existing
`package:flutter/material.dart` (re-exports `widgets`). No logic change to either layout
branch.

## Notes

- **Process correction:** a present git tag / populated `version.json` is **not** proof
  that the tree compiles. fix170.2 shipped a tag but the source does not build. Going
  forward, the analyzer (or a real build) is the only compile authority — exactly the
  PREFLIGHT rule-7 spirit, now upgraded from brace-balance to a real
  `flutter analyze` (Flutter SDK installs cleanly in the new container from
  storage.googleapis.com). **Verify on the CI-pinned SDK** — read `flutter-version` in
  `.github/workflows` (currently `3.44.0` → Dart 3.12.0), not the pubspec floor
  (`^3.10.7`) or the newest public release; CI's analyzer fatal set is the one enforced at
  tag time.
- **Why a whole-method replace for 172.1:** this is the second off-by-one closer on this
  same method (fix170.2 twice). Atomic whole-block replacement is the only reliable way to
  land the brace count — re-confirmed by the analyzer here.
- **`autofocus` scope:** `cellIndex == 0` is evaluated per-cell inside the cell, so it
  needs no new constructor param and naturally no-ops while cell 0 is empty.
- **Fix numbering rule (new):** fix numbers are never reused — every fix is a unique,
  monotonically increasing number and bumps the patch version with a new immutable tag.
  fix172 → 1.23.9+172 accordingly.
