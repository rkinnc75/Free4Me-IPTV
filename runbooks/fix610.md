# fix610 — v2.2.21: Live TV remembers your place

User-approved (2026-06-29): returning to the Live TV tab reset the guide to the
top (Favorites / category list) every time, losing the category/channels view
the user was on. They asked for it to remember the last place.

## Change (tv_guide_view.dart `reloadGuide`)
reloadGuide is called by TvShell on every Live-tab re-select + return-from-
Settings (fix534), and it unconditionally re-ran `_init()` (rebuild rail + reset
to top). Its real purpose is only to pick up a CHANGED enabled-source set.

Now reloadGuide:
1. re-queries `getEnabledSourcesMinimal()` and compares (sorted) to the cached
   `_sourceIds`;
2. if not `_ready` OR the source set changed → `_init()` (full rebuild + reset,
   as before — preserves fix544);
3. if unchanged (the common tab-switch case) → keeps the kept-alive state
   (selected category, channels mode, grid) and restores focus in a post-frame
   to the focused channel (channels mode) or the selected category (categories),
   mirroring fix598/599.

Used a sorted `join(',')` compare (listEquals didn't resolve in this State).

## Review fix (same release)
HIGH regression caught by review: skipping _loadGuide on the unchanged path
dropped the fix541 NOW re-anchor, so a next-day return showed a stale EPG
window. Fixed: the unchanged path now calls `unawaited(_reloadGridProgrammes())`
(re-anchors _windowStart/_windowEnd + rebuilds _progByKey, leaving
channels/focus/mode untouched) before restoring focus. (Pre-existing gap noted:
no app-resume re-anchor without a tab switch — separate enhancement.)
