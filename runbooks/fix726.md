# fix726 — OLED-black background toggle (v4.1.30)

TV only; phone byte-identical. Mock-match (owner: build all approved). Mock §4.1.
Part of task #44 (crossfade is deferred to a separate careful fix — rewriting the
IndexedStack body risks tab keep-alive; not worth bundling with OLED).

## What
Settings ▸ Playback ▸ "OLED-black background" — swaps the neon
`tv_background.webp` for pure #000000 (mock §4.1). Persists; live swap.

## How (mirrors the fix719 accent-picker plumbing)
- `accent_scope.dart`: `final ValueNotifier<bool> appOledNotifier` (default false).
- `models/settings.dart`: `bool oledBlack` (default false).
- `backend/settings_service.dart`: `oledBlackProp` read/write.
- `main.dart`: startup `appOledNotifier.value = settings.oledBlack`.
- `tv_shell.dart`: the background Image+scrim is wrapped in
  `ValueListenableBuilder(appOledNotifier)` → `ColoredBox(0xFF000000)` when on,
  else the webp image + 0xBF scrim (live swap; keep-alive IndexedStack untouched).
- `settings_view.dart`: TV-gated (`tvRailPane`) `_switchTile` → sets
  `settings.oledBlack` + `appOledNotifier.value` + `updateSettings()`.
