# fix603 — v2.2.14: Live-guide behavioral fixes + EPG row alignment

From the user's v2.2.13 on-device test (real remote). Root causes confirmed
from the exported log + DBs (free4me-export-all-...-221351.zip):

| # | Report | Root cause | Fix |
|---|---|---|---|
| 2/11 | OK on a category opens the first channel FULL SCREEN | the channel's `node.onKeyEvent` fired `onTap` on a KeyUp with NO matching KeyDown on that node — the OK that selected the category released after focus moved to the first channel, landing the orphan KeyUp on it | `_selectDown` guard: only act on KeyUp if this node saw the KeyDown |
| 12/16 | Hold OK "flashes menu then plays" + play/pause cycling | the menu opened mid-hold (timer), moving focus to the modal while OK was still held → release/repeats leaked into it | open the menu on RELEASE: the timer only marks `_heldLong`; KeyUp decides menu-vs-play. Threshold 450→600ms |
| 10 | Browsing the category list changes the EPG grid ("no channels") | `_onCategoryFocused` debounced a `_loadGuide` follow on category focus | removed the follow + `_catFollowTimer`; grid changes only on OK-enter |
| 14 | Favorited channels show no star anywhere | favoriting persisted (14 channels flagged in db.sqlite) — the UI just had no indicator | amber `Icon(Icons.star,14)` on favorited channel tiles |
| 9 | Search Enter/empty escaped focus to the tab bar | `DpadTextField` submit always `node.nextFocus()` | `submitKeepsFocus` (search sets it); submit sets `_pendingFocusFirst`, results post-frame-focus the first card; empty stays in field |
| 15 | EPG rows don't line up with channels (rows "larger") | rail used variable-height tiles + 8px top padding while the grid used fixed `itemExtent:56`; AND the time header sat only above the grid, offsetting row-0 by the header height | rail → `controller:_railScroll` + `itemExtent:_rowHeight`, no padding; one-way scroll mirror rail→grid (`_syncGridToRail`); `_timeHeader` moved to a top row with a 210+1 spacer above the rail so rail & grid bodies start at the same Y; `jumpTo(0)` both on category-enter; removed `_scrollGridTo` |

## On-device note
The held-OK *timing* bugs (category auto-play, hold-leak) can't be reproduced via
adb — `input keyevent` is atomic (no separate down…up), so the orphan-KeyUp race
never occurs under adb (which is why the auto-play didn't show in my earlier adb
runs). Verified by root-cause + adversarial review; the user confirms with the
real remote. adb CAN verify: category-follow removal, favorite star, search
focus, and the row alignment (screenshots).
