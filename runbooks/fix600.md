# fix600 ‚Äî v2.2.11: EPG-current "Both" + search DOWN‚Üífirst-card + Live-guide held-OK menu

Next round after the v2.2.8‚Äì2.2.10 Live-guide redesign. Three user-approved
items from the v2.2.8 on-device test (decisions captured 2026-06-28):

| Ask | User decision | This fix |
|---|---|---|
| EPG empty / "On now" empty (stale forecast) | **Both**: foreground stale-check on launch **+** refresh-EPG-with-source-refresh | A |
| Focus after a search should reach the first card | **DOWN from the field ‚Üí first card** | B |
| Multi-view only openable from Search/Movies long-press, not the Live guide | **Add the held-OK menu to Live-guide channels** | C |

Root cause of the "EPG never loaded" report was confirmed from the exported
`epg.sqlite`: the forecast had lapsed (max programme start was before the test
window) because the onn's Workmanager background EPG task is unreliable
(Amlogic/onn kill background work). The data wasn't broken ‚Äî it was stale.

## A ‚Äî EPG stays current ("Both")

`lib/backend/epg_service.dart`
- `static final ValueNotifier<int> epgVersion` ‚Äî bumped at the end of every
  successful `refreshSource` (after `matchChannels`). Views listen and reload.
- `static Future<bool> isStale()` ‚Äî `SELECT count(*) FROM programmes WHERE
  start_utc <= now AND stop_utc > now` against `EpgDbFactory.db`; true when
  nothing is airing now (forecast lapsed). Returns false on any error (never
  triggers a refresh on a check failure).
- `static Future<void> refreshIfStale()` ‚Äî if any enabled source has an EPG URL
  AND `isStale()`, run `refreshAllSources(background: false)`. Cheap no-op when
  fresh or no EPG configured. Eligibility predicate matches `refreshAllSources`
  exactly (`s.enabled && resolveEpgUrl(s) != null`).

`lib/main.dart` ‚Äî after `scheduleBackgroundRefresh()`:
`unawaited(EpgService.refreshIfStale());` (non-blocking launch self-heal).

`lib/settings_view.dart` ‚Äî the source-refresh flow: after `Utils.refreshSource`,
also `EpgService.refreshSource(source, epgUrl: resolveEpgUrl(source), ‚Ä¶)` when
the source has an EPG URL. `onProgress` takes an `XmltvProgress` (use
`p.statusMessage`), NOT a String ‚Äî that mismatch was the first analyze error.

`lib/tv/tv_guide_view.dart` ‚Äî listen to `EpgService.epgVersion` in initState
(remove in dispose). On bump, `_reloadGridProgrammes()` re-fetches ONLY the
programmes for the channels already on screen and rebuilds the grid ‚Äî channels,
focus, and rail mode are left untouched (does NOT bump `_inv`, so a concurrent
`_loadGuide` always wins via the `inv != _inv` guard).

## B ‚Äî search DOWN ‚Üí first card

`lib/widgets/dpad_text_field.dart` ‚Äî new optional `bool Function()? onArrowDown`.
On D-pad DOWN, if provided AND it returns true, it OVERRIDES the default
`node.nextFocus()` (which is unreliable across the stacked result grids,
flutter/flutter#70364). Return false ‚Üí fall back to `nextFocus()`.

`lib/tv/tv_search_view.dart`
- `_orderedGroups()` ‚Äî single source of truth for the five shelves in display
  order (On now ¬∑ Coming up ¬∑ Channels ¬∑ Movies ¬∑ Series).
- `_firstResultNode()` ‚Äî the FocusNode of the first item of the first non-empty
  shelf (from `_nodeCache`).
- Field `onArrowDown:` ‚Üí `_firstResultNode()?.requestFocus()` (true if a card
  exists, else false).
- Tiles no longer autofocus (`autofocusFirst: false`): the field keeps focus
  after a search so the user can keep typing, and DOWN moves to the first card.
  (Autofocusing the first tile would yank focus out of the field mid-type.)
- UP from the first card still returns to the field (`_upTargetFor`, unchanged).

## C ‚Äî Live-guide held-OK menu

TV remotes can't fire `InkWell.onLongPress` (touch gesture) ‚Äî same constraint
fix586 solved for `channel_tile`. Mirror it in the guide's `_FocusTile`.

`lib/tv/tv_guide_view.dart`
- `_FocusTile` gains `Future<void> Function()? onLongPress`. When set (channels
  only ‚Äî categories pass null, keeping their key-down activate), the State binds
  `focusNode.onKeyEvent` (in initState) to a held-select detector: quick OK
  fires `onTap` on key-UP, a hold ‚â• 450ms fires `onLongPress`. Binding to the
  node's onKeyEvent pre-empts InkWell's ActivateIntent (which would otherwise
  play on key-DOWN before a hold registers) ‚Äî exact channel_tile/fix586 model.
  Non-select keys return `ignored` so LEFT still bubbles to the existing
  `onKeyEvent` observer. Cleared in dispose (channel nodes persist in
  `_channelNodes` across rail swaps, so a recycled node must not keep a disposed
  State's closure; the next `_FocusTile` re-installs).
- `_channelItem` passes `onLongPress: () => _showChannelMenu(ch)`.
- `_showChannelMenu(ch)` ‚Äî `showModalBottomSheet`: Play, Open in Multi-view
  (livestream + non-empty URL, via `MultiViewScreen.openWithChannel`), and
  favorite toggle (`Sql.favoriteChannel`, then `_loadGuide(_selectedGroupId)` to
  reflect). New import: `multi_view_screen.dart`.
