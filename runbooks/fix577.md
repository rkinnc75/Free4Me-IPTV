# fix577 — TV D-pad: channel surf from the guide + working bar reveal

**Version:** v2.1.4+577
**Date:** 2026-06-27
**Type:** TV/remote UX (player) — follow-up to fix576, fixing two on-device findings

## Context

fix576 (v2.1.3) added direct D-pad mapping in the player. On-device ADB testing
of the shipped v2.1.3 build on the onn 4K Plus found:
- ✅ OK → play/pause worked (proving keys reach the handler / focus is correct).
- ✅ fix575 stream-info latch emitted (`STREAMINFO emitted "1080p H.264"`).
- ❌ **Channel up/down did nothing** — `tv_guide_view.dart` launched `Player()` with
  NO `playlist`, so `_canSurf` was always false. Affected the D-pad arrows AND the
  dedicated CH+/CH− keys when a stream was started from the guide.
- ❌ **OK "reveal bars" did not show them** — fix576's keyed-remount + `visibleOnMount`
  trick failed (media_kit 2.0.1 preserves the controls state across the theme
  remount, so `visibleOnMount` was never re-read).

## Change

1. **Channel surf from the guide.** `tv_guide_view.dart` `_play()` now passes
   `playlist: PlaybackPlaylist(channels: _visibleChannels, index: indexOf(ch))`
   (null when not found). Browse + search already passed a playlist; the guide was
   the gap. Now ▲/▼ and CH+/CH− surf the guide's visible channel list.
2. **Working bar reveal.** `_revealControls()` now **synthesizes a tap** at the
   video centre via `GestureBinding.handlePointerEvent` (Down+Up) — media_kit
   toggles its bars on tap, so this shows them exactly like a screen tap, then they
   auto-hide via `controlsHoverDuration`. Removed fix576's dead remount machinery
   (`_controlsRevealTick`, `_revealControlsOnMount`, the key suffix, the
   `visibleOnMount` theme line).

## Scope / next

Still the **direct-map** scheme. The agreed full model's focusable-bar navigation
(D-pad moves between Cast/PiP/Subtitles/etc. once the bars are up) is the remaining
piece. Verify this build on-device first, then layer focus-nav.

## Verification

- `flutter analyze` clean; `flutter test` green (272; fix576 mapping test unchanged).
- **On-device (post-ship):** `adb install -r` the release APK on the onn (in-place,
  no data loss), tune from the guide, then `input keyevent DPAD_UP/DOWN` → confirm
  channel surf (logcat `Player: ... surf`), `DPAD_CENTER` → play/pause + bars appear
  (screencap), and the channel name shows the appended stream-info (fix575).
