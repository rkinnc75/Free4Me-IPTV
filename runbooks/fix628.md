# fix628 — Self-heal + harden the channels browse indexes (v2.2.39)

**Builder:** desktop coder+builder. **Live base:** v2.2.38. **Ships as:** v2.2.39+628.
(Restructure items #2/#3, reframed by the true root cause found on-device.)

## True root cause (found verifying fix627 on the onn)

Opening a Live TV category took 10–45 s AND fix627's `INDEXED BY
idx_browse_src_grp` was inert — because **the index does not exist on the
device**. The private log 2026-07-01 is conclusive:
- Midnight refresh: `dropped 19 index(es)` → `built 1/19 … 19/19` → complete. OK.
- **20:16 refresh: `dropped 19 index(es)` → FTS integrity-checks stalled
  (15–60 s each) → user hit Cancel at 21:57 → NO `built N/19` ever logged.** The
  refresh died between the DROP and the recreate; the recreate lives in a
  `finally`, which a process-kill/cancel skips → **all 19 browse indexes lost
  permanently.**
- Next refresh: `dropped 0 index(es) ()` — confirms they stayed gone.

With only `index_channel_source_id` surviving, EVERY browse (grouped AND
ungrouped) falls back to it and full-scans the ~1.2M-row catalog. This is a
**durability bug in `withDroppedBrowseIndexes`**, not a planner/index-design
problem.

## The fix (self-heal + harden — user-approved, all 19 indexes)

1. **Self-heal (`Sql.ensureBrowseIndexesPresent`, deferred from main):** a
   canonical map of all 19 channels indexes (`_canonicalChannelIndexes`, defs
   copied verbatim from the fix537 rebuilt set + migrations, 6-tier CASE shared
   via `_t6`). On startup it checks which are missing and rebuilds them. Cheap
   no-op when all present; unawaited/deferred so a multi-minute rebuild never
   blocks the splash. Heals the already-broken device AND any future
   interruption.
2. **Harden `withDroppedBrowseIndexes`:** persist the captured CREATE DDL to
   `app_meta('pending_browse_index_ddl')` BEFORE dropping; clear it only after
   ALL indexes rebuild. If the process dies mid-run, `ensureBrowseIndexesPresent`
   replays the exact persisted DDL next startup (covers custom/future indexes
   the canonical map might not know). Partial recreate keeps the record so the
   self-heal finishes the rest.

Together with fix627 (the hint), once `idx_browse_src_grp` exists the category
browse seeks straight to the group (sub-second); the self-heal also restores the
ungrouped/media-type indexes so the home "All" browse recovers too.
