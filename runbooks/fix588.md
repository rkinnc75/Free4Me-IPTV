# fix588 — #22: compact diagnostic-overlay variant for multi-view cells

**Version:** v2.2.4+589 (with fix589 #5)
**Date:** 2026-06-28
**Type:** Feature (debug UX)

## Problem

The full `DebugStatsOverlay` (8 rows, 11px) fits the single-cell full-screen
player but not a small multi-view / PiP cell. The full-screen player suppresses
the overlay in multi layouts, and multi-view cells never showed any stats.

## Implementation

- `lib/player/debug_stats_overlay.dart`: added a `compact` bool (default false).
  Compact mode renders a 2-line top-left panel — `WxH codec` and
  `<fps>fps  d <vo>/<dec>` at 8px — with the drops line turning amber on a
  positive delta. The 1 Hz `readPlaybackStats` poll + `STATS` log line are
  unchanged; only the rendered panel differs.
- `lib/multi_view_cell.dart`: imported the overlay and added it to the cell's
  Stack (last child, top-left so it clears the top-right volume badge and the
  bottom info bar) gated on `widget.settings.debugLogging && _engine is
  MpvEngine`, passing `compact: true`.

## Verification

- `flutter analyze` clean; `flutter test` green (272).
- **On-device (post-ship):** Settings → enable debug logging. Open multi-view
  2×2 with live channels → each cell shows the 2-line compact panel, no overflow
  / wrap, clear of the volume + info widgets. Open the single full-screen player
  → the full 8-row overlay is unchanged. Check logcat: STATS lines still emit
  (now up to 4/s in a 2×2 — watch for read stalls).
