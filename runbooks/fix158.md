# fix158.md — TV export via local HTTP server (port 9479) serving BOTH backup + debug log, with a QR code

> **Version:** Free4Me-IPTV 1.23.3 build. **TV-ONLY** changes.
> **Device:** onn 4K Plus (Google TV). **CRITICAL** — currently NO way to get
> settings backups OR debug logs off the device.
>
> **Problem:** both exports call `FilePicker.platform.saveFile(...)` →
> `ACTION_CREATE_DOCUMENT` intent. **Google TV has no Storage Access Framework
> UI / document-handler app**, so nothing receives the intent → "You don't have
> an app that can do this." Blocks settings backup and debug-log export.
>
> **Solution:** a single built-in HTTP server on **port 9479** that serves BOTH
> files at once — an index page links the **settings backup** and the **debug
> log**; the on-screen dialog shows the LAN URL **and a QR code** to scan. No
> auth, simple access. Download from any phone/PC on the same Wi-Fi.

---

## Scope & what's already in place

- **TV-only.** Phones/tablets keep the existing `FilePicker` export (works
  there). Server path is gated on `DeviceDetector.isTV()`.
- `android:usesCleartextTraffic="true"` and `INTERNET` permission are ALREADY in
  the manifest — **no manifest change**; LAN HTTP works.
- HTTP server + LAN IP use built-in `dart:io` (`HttpServer`,
  `NetworkInterface`) — **no dependency for the server**.
- **QR:** add `qr_flutter` (small, pure-Dart, no native code) for `QrImageView`.
  This is the only new dependency.

# Fix 158.1 — Multi-file local export server

**New file:** `lib/backend/export_server.dart`

Holds a SET of named payloads in memory and serves them. Index page at `/`
lists each file with a Download link; each file is served at
`/file/<key>`. Binds `0.0.0.0:9479`. Auto-stops after 10 minutes idle (does NOT
stop on first download, since two files may be fetched).

```dart
import 'dart:async';
import 'dart:io';
import 'package:open_tv/backend/app_logger.dart';

class ExportItem {
  final String key;          // url-safe id, e.g. 'backup' / 'log'
  final String filename;     // download name, e.g. 'free4me-backup.json'
  final String label;        // human label for the index page
  final List<int> bytes;
  final String contentType;  // e.g. 'application/json'
  const ExportItem({
    required this.key,
    required this.filename,
    required this.label,
    required this.bytes,
    required this.contentType,
  });
}

/// Serves one or more payloads (settings backup, debug log) over the LAN for
/// download from a phone/PC. Built for Google TV (no SAF / file picker).
/// Local network only, no auth — short-lived, user-initiated, the user's own
/// data on their own network.
class ExportServer {
  static const int port = 9479; // fix158: fixed, memorable port

  HttpServer? _server;
  final List<ExportItem> _items;
  Timer? _idleTimeout;

  ExportServer(this._items);

  /// Start and return the LAN URLs to display (one per non-loopback IPv4).
  Future<List<String>> start() async {
    _server = await HttpServer.bind(InternetAddress.anyIPv4, port, shared: true);
    AppLog.info('ExportServer: listening on :$port items=${_items.length}');
    _server!.listen(_handle);
    _resetIdle();

    final urls = <String>[];
    for (final ni in await NetworkInterface.list(
        type: InternetAddressType.IPv4, includeLoopback: false)) {
      for (final addr in ni.addresses) {
        urls.add('http://${addr.address}:$port/');
      }
    }
    return urls;
  }

  void _resetIdle() {
    _idleTimeout?.cancel();
    _idleTimeout = Timer(const Duration(minutes: 10), stop);
  }

  void _handle(HttpRequest req) async {
    _resetIdle();
    final path = req.uri.path;

    if (path == '/') {
      final buf = StringBuffer()
        ..write('<!doctype html><meta name="viewport" '
            'content="width=device-width,initial-scale=1">'
            '<body style="font-family:sans-serif;padding:2em;max-width:32em;'
            'margin:auto"><h2>Free4Me-IPTV export</h2>'
            '<p>Tap a file to download:</p>');
      for (final it in _items) {
        final kb = (it.bytes.length / 1024).toStringAsFixed(0);
        buf.write('<p><a href="/file/${it.key}" '
            'style="display:block;font-size:1.2em;padding:.7em 1em;margin:.4em 0;'
            'background:#4E9FE5;color:#fff;text-decoration:none;border-radius:8px">'
            '${it.label} <small>(${it.filename}, $kb KB)</small></a></p>');
      }
      buf.write('</body>');
      req.response
        ..statusCode = 200
        ..headers.contentType = ContentType.html
        ..write(buf.toString());
      await req.response.close();
      return;
    }

    if (path.startsWith('/file/')) {
      final key = path.substring('/file/'.length);
      final item = _items.where((i) => i.key == key).cast<ExportItem?>()
          .firstWhere((_) => true, orElse: () => null);
      if (item == null) {
        req.response.statusCode = 404;
        await req.response.close();
        return;
      }
      req.response
        ..statusCode = 200
        ..headers.contentType = ContentType.parse(item.contentType)
        ..headers.add('Content-Disposition',
            'attachment; filename="${item.filename}"')
        ..add(item.bytes);
      await req.response.close();
      AppLog.info('ExportServer: served ${item.filename}');
      return;
    }

    req.response.statusCode = 404;
    await req.response.close();
  }

  Future<void> stop() async {
    _idleTimeout?.cancel();
    await _server?.close(force: true);
    _server = null;
    AppLog.info('ExportServer: stopped');
  }
}
```

# Fix 158.2 — UI helper: start server, show URL + QR

**File:** `lib/settings_view.dart` (helper method on the state) — needs
`import 'package:qr_flutter/qr_flutter.dart';` and the export_server import.

```dart
  /// fix158: serve the given items over the LAN and show URL + QR (TV export).
  Future<void> _showExportServerDialog(List<ExportItem> items) async {
    final server = ExportServer(items);
    List<String> urls;
    try {
      urls = await server.start();
    } catch (e) {
      if (mounted) {
        ScaffoldMessenger.of(context).showSnackBar(
          SnackBar(content: Text('Could not start export server: $e')));
      }
      return;
    }
    if (urls.isEmpty) {
      await server.stop();
      if (mounted) {
        ScaffoldMessenger.of(context).showSnackBar(const SnackBar(
          content: Text('No network found. Connect to Wi-Fi and retry.')));
      }
      return;
    }
    if (!mounted) { await server.stop(); return; }

    final primary = urls.first; // Wi-Fi IP is almost always first
    await showDialog<void>(
      context: context,
      barrierDismissible: false,
      builder: (ctx) => AlertDialog(
        title: const Text('Download on your phone or PC'),
        content: Column(
          mainAxisSize: MainAxisSize.min,
          children: [
            const Text('On a device on the SAME Wi-Fi, scan this code '
                '(or type the address) and tap a file to download:'),
            const SizedBox(height: 16),
            // QR — white background so it scans on a dark TV theme.
            Container(
              padding: const EdgeInsets.all(12),
              color: Colors.white,
              child: QrImageView(
                data: primary,
                version: QrVersions.auto,
                size: 220,
                backgroundColor: Colors.white,
              ),
            ),
            const SizedBox(height: 16),
            for (final u in urls)
              SelectableText(u, textAlign: TextAlign.center,
                  style: const TextStyle(
                      fontSize: 20, fontWeight: FontWeight.bold)),
            const SizedBox(height: 8),
            const Text('Lists your settings backup and debug log. '
                'Server stops after 10 minutes.',
                style: TextStyle(fontSize: 12, color: Colors.grey)),
          ],
        ),
        actions: [
          TextButton(
            onPressed: () async {
              await server.stop();
              if (ctx.mounted) Navigator.pop(ctx);
            },
            child: const Text('Done'),
          ),
        ],
      ),
    );
    await server.stop(); // ensure stopped on any dismissal
  }
```

> QR sits on an explicit white `Container` so it scans reliably against the dark
> app theme. `version: QrVersions.auto` sizes for the short URL. If a TV reports
> multiple IPv4s, the QR encodes the first (Wi-Fi); all are listed as text.

# Fix 158.3 — Build BOTH payloads and serve them together (TV); phone unchanged

**File:** `lib/backend/settings_io.dart`

Factor the backup payload out of `exportToFile` so the bytes can be reused:
```dart

  // ... existing body of exportToFile that assembles `payload` ...
  return payload;
}
```
`exportToFile` (phone) calls `buildBackupPayload` then does the `FilePicker`
save as today.

**File:** `lib/settings_view.dart`

Add ONE TV export entry that bundles both files. Two ways to wire it — recommended:
a single new tile **"Export backup + log (TV)"** shown only on TV, plus making
the existing backup/log tiles route to the server on TV. Simplest coherent UX:
on TV, BOTH existing export tiles open the SAME combined server dialog (which
lists both files), so the user gets everything regardless of which they tapped.

Combined builder + launcher:
```dart
  Future<void> _exportEverythingViaServer({required bool includeCredentials}) async {
    final items = <ExportItem>[];

    // Backup
    final backup = await SettingsIo.buildBackupPayload(
        includeCredentials: includeCredentials);
    items.add(ExportItem(
      key: 'backup',
      filename: 'free4me-backup.json',
      label: 'Settings backup',
      bytes: utf8.encode(backup),
      contentType: 'application/json',
    ));

    // Debug log (only if present)
    if (settings.debugLogging) {
      final log = await AppLog.readLog();
      if (log.isNotEmpty) {
        items.add(ExportItem(
          key: 'log',
          filename: 'free4me_log_${DateTime.now().millisecondsSinceEpoch}.txt',
          label: 'Debug log',
          bytes: utf8.encode(log),
          contentType: 'text/plain; charset=utf-8',
        ));
      }
    }
    if (!mounted) return;
    await _showExportServerDialog(items);
  }
```
