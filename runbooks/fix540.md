**Base tag:** `v2.0.39`

## MANUAL STEP (required, before applying the patch)

Place the provided image file at `assets/tv_background.webp` in the repo root:

```bash
mkdir -p assets
cp /path/to/tv_background.webp assets/tv_background.webp
```

The image (`tv_background.webp`, the 4K neon art) is delivered alongside this runbook. `assets/` is already a registered asset directory in `pubspec.yaml` (`assets: - assets/`), so no pubspec asset-list change is needed — only the version bump in the patch.

After the file is in place, apply the patch below as usual.

---

## Change

Wrap the TV shell `Scaffold` in a `Stack`: a full-screen `Image(AssetImage('assets/tv_background.webp'), fit: BoxFit.cover)` (crops to fill any aspect), a flat **~90% black scrim** (`Color(0xE6000000)`) for readability, then the existing UI with `Scaffold(backgroundColor: Colors.transparent)` so the art shows through behind every tab. The dark theme's black surface previously hid anything behind it; transparency + scrim is what makes the ambient glow visible without harming legibility over grids/EPG.
