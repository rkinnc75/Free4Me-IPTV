# fix670 — Scheduled Recording creation actions + low-space guard (v3.0.14)

**Builder:** desktop. **Live base:** v3.0.13. **Ships as:** v3.0.14+670.

Fourth phase of the Scheduled Recording (SR) arc (667 schema → 668 capture →
669 list → **670 creation actions** → 671 optional transcode). Makes SR usable:
this is the first phase where a user can actually START a recording.

## What

- **Record from the TV guide** — a "Record" button in the programme detail
  dialog (`channel_schedule._showDetails`), shown for FUTURE or on-now
  programmes (`stopUtc > now`), the inverse of the existing catch-up "Watch
  from beginning" (past/on-now). Calls `scheduleForProgramme` with the pad
  defaults; a live over-run is covered by the after-pad.
- **Record now** on TWO surfaces (both live-only):
  - the channel long-press menu ("Record now"), and
  - the player top bar (record icon; hidden when playing a recording, i.e.
    `overrideUrl == null`).
  Both open a quick **duration picker (30 / 60 / 120 / Custom 1–720)** then
  `scheduleNow`.
- **Shared `recording_actions.dart`** — one entry point per surface
  (`recordProgramme`, `recordNow`) that centralises the duration picker,
  low-space handling, and the confirmation/failure snackbars, so each call site
  is one line.
- **Low-space guard (1 GB floor)** — `RecordingScheduler` now queries free
  bytes (new native `getFreeBytes` via StatFs on the external/recordings volume,
  exposed through `RecordingCapture.freeBytes`) and throws
  `LowDiskSpaceException` before creating any recording when free space is below
  1 GB. null free-space (query failed / non-Android) is treated as
  "unknown → allow". A long capture can't be sized ahead, so this is a
  "don't start on a nearly-full disk" guard, not a hard cap.

## Verified in-sandbox (real toolchain — Flutter 3.44.4 / Dart 3.12.2)

- `flutter analyze` → "No issues found!"
- `flutter test` → **All 325 passed** (321 + 4 new low-space boundary tests:

## VERIFY-FIRST — on-device (Samsung primary; onn for weak-hardware)

1. Guide: open a FUTURE programme → "Record" appears (not on a purely-past one)
   → tap → "Recording scheduled" snackbar → the row appears in Recordings
   (fix669) as scheduled → at the padded start it captures (fix668).
2. Record now — channel long-press → "Record now" (live only) → picker
   (30/60/120/Custom) → capture starts, appears in Recordings as recording.
3. Record now — player top bar record icon (live only; absent when playing a
   saved recording) → same picker → capture.
4. Custom duration accepts 1–720, rejects junk.
5. Low space: with <1 GB free, any of the above shows the low-space message and
   does NOT create a recording. With plenty free, proceeds normally.
6. Full end-to-end: schedule from guide → wait → file lands in Movies/Free4Me →
   plays from the Recordings list.
