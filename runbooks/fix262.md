> **Version:** patch hotfix on current `main` (**1.26.1+260**). Bump to **1.26.2+262**, tag `v1.26.2`.
> **Severity: CRITICAL** — every browse view (Live/Movies/Series/All/Favorites) throws `SqliteException(1): near "DESC": syntax error` and shows the red error dialog. The app is unusable for browsing until fixed.
> **Cause:** the favorites-first logic in BOTH ORDER BY sites was written as `CASE WHEN (...)='provider' THEN COALESCE(c.favorite,0) DESC ...`. `DESC` is an ORDER-BY direction keyword and is NOT valid inside a `CASE` expression. SQLite rejects it at prepare time. (This compiled clean — the SQL lives in a string literal — and only failed at runtime.)
> **Fix:** express "favorites first" as a sortable value inside the CASE: `CASE WHEN COALESCE(c.favorite,0)=1 THEN 0 ELSE 1 END` (sorted ASC). Same result, valid SQL. Applied at both ORDER BY sites in `lib/backend/sql.dart`.

## Root-cause note for future runbooks
`DESC`/`ASC` are ORDER BY *direction* keywords, not expression operators — they cannot appear inside a `CASE ... THEN ... END`. To sort a boolean "favorites first" inside an ASC CASE, map it to a rank: `CASE WHEN cond THEN 0 ELSE 1 END`. **`flutter analyze` cannot detect malformed SQL inside string literals — only running the query against SQLite catches it.** Any change to a SQL ORDER BY/WHERE must be executed against a real DB before shipping.
