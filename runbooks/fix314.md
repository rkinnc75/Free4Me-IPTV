# fix314 — Multi-view decode mode (fix NVIDIA Shield 2×2 colour corruption) + device diagnostic → v1.26.30+314

## Root cause
On NVIDIA Shield (Tegra), the 2×2 multi-view grid shows rainbow / swapped-chroma colour corruption. All preview cells use `hwdec=mediacodec-copy`; running multiple concurrent mediacodec-copy decode sessions on Tegra corrupts the copied-back colour planes. Confirmed in the log (`hwdec=mediacodec-copy isTV=true`) and the screenshots (some cells garbled, one loading).

## What this does (options 3 + 4)
1. **Decode-mode setting** `multiViewDecode` (Auto / Hardware (copy) / Software), default **Auto**, in the Multi-view settings group:
   - **Auto** → software decode (`hwdec=no`) on Tegra/Shield, `mediacodec-copy` everywhere else.
   - **Hardware (copy)** → force `mediacodec-copy` (pre-fix314 behaviour).
   - **Software** → force `hwdec=no` for all cells.
2. **On-device diagnostic** — at startup the app logs `fix314 device: <manufacturer/brand/model/device/board/hardware> | isTegra=<bool>`, so we can confirm Shield detection on real hardware from the debug log.

## How it's built
- New `MultiViewDecode` enum (json + label), mirroring `MultiViewLayout`.
- `DeviceDetector.isTegra()` (matches tegra/shield/nvidia across manufacturer/brand/board/hardware/model/device) + `boardInfo()` for the diagnostic.
- `Settings.multiViewDecode` field + ctor default; serialized in `settings_service.dart` (load/save) — also rides the backup payload automatically (same map).
- `mpv_engine.dart` preview branch honours the setting (auto = Tegra-aware) and logs the chosen mode.
- `main.dart` logs the device diagnostic at startup.
- Settings UI: `_multiViewDecodeTile` + `_showMultiViewDecodeDialog`, reusing the existing `SelectDialog`/`IdData` `action:` pattern.

## Note (fix108 history)
fix108 found pure `hwdec=no` stalled the single overlay preview on some streams. That was the single overlay engine; the multi-view cells here are the corruption case, and software decode is the standard fix for it. Auto only switches to software on Tegra/Shield, so non-Tegra devices (incl. the onn box where the overlay fix108 applies) keep mediacodec-copy. If a Shield user sees stalls instead of corruption on software, they can pick **Hardware (copy)** — the toggle exists for exactly this.

+///
+/// On some SoCs — notably NVIDIA Tegra / Shield — running several concurrent
+/// `mediacodec-copy` decode sessions for the 2×2 grid corrupts colour output
+/// (rainbow / swapped chroma planes). Software decode avoids the bug at the
+/// cost of CPU. This setting lets the user pick per device.
+///
+/// [auto]         — software decode on Tegra/Shield, mediacodec-copy elsewhere.
+/// [hardwareCopy] — force mediacodec-copy (the pre-fix314 behaviour).
+/// [software]     — force CPU decode (hwdec=no) for all cells.
+enum MultiViewDecode {
+  auto,
+  hardwareCopy,
+  software;

+        'software' => software,

+      };

+  String get label => switch (this) {
+        auto => 'Auto (recommended)',

+        software => 'Software',
+      };
