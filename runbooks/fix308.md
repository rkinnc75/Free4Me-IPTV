## What this does (two features)
1. **Favorite a category.** Long-press a category tile → "Favorite category (sort to top)". Sets `groups.favorite`; favorited categories sort to the top of the Categories list, existing order preserved below. Does NOT favorite the channels inside the category.
2. **Channel → category link.** In a channel's long-press menu, the category-name header (added in fix305) is now tappable: it opens the Categories list filtered to that category name (search box pre-filled), so the category surfaces at the top via the existing search + favorite-first sort. Reuses search instead of fragile scroll-to-index.

## How it's built
- Migration 25: `groups.favorite INTEGER DEFAULT 0` (additive, no backfill).
- `searchGroup`: `ORDER BY COALESCE(favorite,0) DESC, id ASC` (favorites top, prior rowid order below). `groupChannelToRow` maps `favorite` from column 6.
- `Sql.favoriteGroup(id, bool)` toggle.
- `ChannelTile`: group-tile long-press shows the favorite toggle (`onFavoriteGroup`); the channel menu's category header gets `onOpenCategory`.
- `home.dart`: wires both callbacks; new `_openCategoryByName` pushes a `ViewType.categories` Home with `query = name`; `searchController` now initializes from `filters.query` so the box shows the term.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Built against live main (1.26.23+307); patch applies cleanly to a fresh clone.
- Rule 8 (real SQLite): migration 25 adds the column with existing rows defaulting to 0 (no data loss); the favorite-first ORDER BY puts favorited categories on top and preserves prior order below.
