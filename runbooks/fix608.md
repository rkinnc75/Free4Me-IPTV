## #1 â€” defer the launch EPG refresh until warm-up finishes (main.dart)
`refreshIfStale` (fix600) fired immediately at launch, piling a 100k-programme
EPG download onto an already SQL-saturated startup. Now the heavy startup
warm-ups (`ChannelSearchCache.ensureBuilt`, `Sql.warmBrowseCache`,
`Sql.runPendingIndexMaintenance`) are collected into `warmups`, and
`Future.wait(warmups.map((f) => f.catchError((_){}))).whenComplete(...)` runs
`refreshIfStale` only after they all settle (error-swallowed so it always runs;
~2s on normal devices, deferred through the freeze on huge catalogs).

## #2 â€” bounded pragmas around the migration chain (db_factory.dart)
The import path (`Sql.withDroppedBrowseIndexes`) and fix542 deferred maintenance
already set `temp_store=FILE` + `cache_size=-32768`; the **versioned migrations
that build browse indexes (30, 33)** did not â€” so a user UPGRADING with an
already-populated huge `channels` table disk-spilled ~75s/index (the Shield's
case). Now the bounded pragmas are set on the writer right before
`migrations.migrate(db)` and restored to `-2000` after, covering every migration
index build at once.

## #4 â€” IF NOT EXISTS on all browse-index CREATEs (sql.dart + db_factory.dart)
The fix542 `rebuilt` map (sql.dart ~1287â€“1346) created indexes with no `IF NOT
EXISTS`; on a force-kill-mid-freeze relaunch the DROP/CREATE churned and a
`_safe` create hit "already exists". Added `IF NOT EXISTS` to all 7 map entries
and to `idx_browse_prov` / `idx_browse_cat` (db_factory migs). (db_factory's
`_safe` creates already had it.)
