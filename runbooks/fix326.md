# fix326 — Edit-source dialog cut off on TV (corrects fix324) + mask source password → v1.26.42+326

> ## WARNING BUILD MACHINE — APPLY THE PATCH, then RELEASE (EXACTLY). No questions, no DB to query.
> **FIRST:** `find . -name 'fix296-DIAGNOSE*.md' -delete 2>/dev/null || true`

## Why
1. **fix324 did not fix the scroll** — on TV (v1.26.41) the form still painted past the dialog bottom ("By category" spilling outside, info rows unreachable). `AlertDialog(scrollable: true)` wraps the content in a scroll view that hands it UNBOUNDED height, so the `ConstrainedBox(maxHeight: ...)` capped the Column and the Column overflowed instead of scrolling.
2. **Password shown in clear text** — provider credentials are assigned, not user-chosen, and cannot be rotated, so the Edit Source screen must not render the password in the clear.

## What this does
1. **Scroll (real fix):** drops `scrollable: true` + the `ConstrainedBox`/MediaQuery math; makes the dialog `content` a `SingleChildScrollView`. The `AlertDialog` bounds content to the screen automatically; the inner scroll view scrolls the FORM while title + **Save/Cancel** stay fixed and always visible. D-pad traversal auto-scrolls to the focused field.
2. **Password masking:** the Password field is now `obscureText` by default with a tap-to-reveal eye toggle (`_obscurePassword` state + suffix `IconButton`). Username and URL are unchanged.
3. No model/DB/logic change.
