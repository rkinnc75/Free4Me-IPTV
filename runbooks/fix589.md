# fix589 — #5: dwell auto-preview in the TV Movies/Series browse grid

**Version:** v2.2.4+589 (with fix588 #22)
**Date:** 2026-06-28
**Type:** Feature (TV UX, RAM-sensitive — verify last on the onn)

## Scope

Per the user's decision, #5 targets the Movies/Series **browse grid**
(`tv_browse_view.dart`), not the Live EPG guide (which already has the
0.7–1.1 s `TvHeroPreview`). Dwelling ~3 s on a card auto-starts a muted preview
of that title in a corner PiP.

## Implementation — REUSE the proven controller

Rather than a parallel new controller (the design's first cut), this reuses
`TvHeroPreview` verbatim — it already satisfies every adversarial-review
requirement: `_teardownEngine()` BEFORE allocating the next engine, generation
token re-checked around every await, `forceSoftwareDecode: true`, always muted,
error → fall back to art (never auto-restart), single engine at all times.

- `lib/tv/tv_hero_preview.dart`: added an optional `isLive` param to
  `onChannelFocused` / `_warm` (default true → guide unchanged) so browse VOD
  movies open non-live (seekable) while livestream tiles open live.
- `lib/channel_tile.dart`: added `onFocusGained(Channel)` emitted from the
  EXISTING focus listener (no second listener). Null everywhere except browse.
- `lib/tv/tv_browse_view.dart`: owns one `TvHeroPreview`. Each tile's
  `onFocusGained` arms `onChannelFocused(dwellMs: 3000, liveEnabled:
  settings.tvBrowseDwellPreview, isLive: mediaType==livestream)`. Teardown
  triggers: (a) the grid is wrapped in a non-focusable `Focus` whose
  `onFocusChange(false)` calls `_preview.stop()` when focus leaves the grid (to
  the rail / navbar / tab switch — IndexedStack keeps the view alive, so this
  focus-leave is what stops it); (b) `_loadItems` (category switch) stops it;
  (c) `dispose()` calls `disposeController()`. Preview renders as a 300×169
  corner PiP via `ListenableBuilder` on the controller; null view → nothing.
- New setting `tvBrowseDwellPreview` (default **false**) — settings.dart,
  settings_service read/write, settings_io backup, and a Settings toggle
  ("Preview on dwell in browse"). Default off protects the 2 GB onn.

## Risks (carried from the adversarial review)

- Engine leak / >1 engine on rapid D-pad: mitigated by TvHeroPreview's
  teardown-before-allocate + gen-token (proven on the guide).
- PSS growth on the 2 GB onn: the on-device check below is MANDATORY.
- Stale preview for up to one dwell after moving off a live tile (cosmetic;
  next tile's `_warm` tears it down). Acceptable.

## Verification (MANDATORY on the onn — verify LAST, RAM-focused)

- `flutter analyze` clean; `flutter test` green (272).
- **On-device (post-ship), tvBrowseDwellPreview = ON:**
  1. Movies tab: dwell 3 s on a card → muted corner PiP appears and plays.
  2. D-pad to the next card → PiP follows after the next 3 s dwell.
  3. D-pad LEFT to the category rail → PiP stops immediately.
  4. Switch tabs (Movies→Live→Movies) → PiP stops; no orphaned audio.
  5. `adb shell dumpsys meminfo me.free4me.iptv` before/after a 2-min browse
     session → PSS within ~20 MB of baseline; `pidof`/proc check → exactly ONE
     libmpv at any instant; logcat shows no concurrent-open / dispose errors.
  6. Toggle OFF → no preview on dwell; setting persists across restart.
