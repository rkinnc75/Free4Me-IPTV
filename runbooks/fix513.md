# fix513 — revert fix423's bottom-bar lift (+ restore the 0-info analyze gate)

**Version:** 2.0.12+512 → **2.0.13+513**
**Type:** Revert + analyze-gate restore.
**Tag:** v2.0.13
**Base:** v2.0.12 (2.0.12+512).

## Why
fix423 tried to make the auto-hidden control bar wakeable by lifting it above media_kit's 16px tap-dead ring + the device gesture inset, via `bottomButtonBarMargin.bottom: barLift` and `seekBarMargin: bottom 60+barLift`. That backfired: media_kit's `subtitleVerticalShiftOffset` (`material.dart:568-573`) is computed from `bottomButtonBarMargin.vertical`, and it's used to inset the tap-to-show `Listener` from the bottom (`material.dart:~891`). So the same value that lifted the bar ALSO enlarged the tap-dead strip by the same amount — net the bar floated up ~64px AND the dead zone roughly doubled (~64→~128px). Strictly worse on both axes.

This reverts to baseline edge-aligned margins (bar back at the screen edge, dead strip back to the pre-fix423 ~64px). The actual wake-on-tap is handled app-side in **fix514** (next).

## What
- `lib/player.dart` — `_mpvThemeData`: removed the `barLift` computation + `MediaQuery`/`systemGestureInsets` logic (no dangling `mq`); `seekBarMargin` back to `const EdgeInsets.only(bottom: 60)`; dropped the `bottomButtonBarMargin` override (back to media_kit's edge default).
- `lib/setup.dart` — **gate fix:** the 501–512 arc introduced a `use_build_context_synchronously` info at `setup.dart:368` (`context` passed into `importFromFile` after the `_ensureSetupLogging` await with no guard). Added `if (!mounted) return;` after that await (fix369 pattern). Restores `flutter analyze --no-fatal-infos` → **"No issues found!"**.

`flutter analyze` clean (0 infos), 171 tests pass.
