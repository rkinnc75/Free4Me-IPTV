# fix122.md — Build break: `autofocus` on `ExpansionTile` (undefined_named_parameter)

> **Version:** Free4Me-IPTV 1.22.8 (verified against actual source in the
> uploaded zip).
> **Evidence:** `flutter analyze --no-fatal-infos` output (CI pre-tag gate),
> exit code 1.
>
> **Bug:** the build fails to compile. `ExpansionTile` does not define an
> `autofocus` parameter, but the **"Playback"** `ExpansionTile` in the
> settings screen passes `autofocus: true`. This is a hard `error` (not an
> info/warning), so analyze exits 1 and CI never produces a build.
>
> **Root cause:** `autofocus` was added to the wrong widget. It is a valid
> parameter on the `TextField` lower in the same file (the XMLTV URL dialog)
> but `ExpansionTile` has no such parameter. The likely history is a
> copy/paste of `autofocus: true` onto the tile.

---

## What analyze proves

```
error • The named parameter 'autofocus' isn't defined. Try correcting the
name to an existing named parameter's name, or defining a named parameter
with the name 'autofocus' • lib/settings_view.dart:1357:21 •
undefined_named_parameter
   info • Don't use 'BuildContext's across async gaps… •
         lib/settings_view.dart:1861:66 • use_build_context_synchronously
   info • Don't use 'BuildContext's across async gaps… •
         lib/settings_view.dart:2421:48 • use_build_context_synchronously
3 issues found. (exit code 1)
```

Only the **first line is fatal**. The two `use_build_context_synchronously`
INFOs at ~1861 / 2421 are the long-standing tolerated INFOs (noted in the
project conventions) — they do not fail the gate and must be left alone.

There are **two** `autofocus: true` occurrences in the file:

```
1357:                    autofocus: true,   ← ExpansionTile  (THE ERROR — remove)
1990:                              autofocus: true,   ← TextField  (VALID — keep)
```

Only line 1357 is the problem. The line 1990 one is on the XMLTV URL
`TextField` (so D-pad/keyboard lands in the field) and is correct.

---

## The mechanism

`lib/settings_view.dart`, the top-level "Playback" tile (around line 1355):

```dart
                  ExpansionTile(
                    key: const PageStorageKey('playback'),
                    autofocus: true,                       // ← not a valid param
                    leading: const Icon(Icons.play_circle_outline),
```

`ExpansionTile`'s constructor has no `autofocus`. The analyzer flags it as
`undefined_named_parameter` (an error, not a warning), which fails the build.
`ExpansionTile` never had a focus-on-mount concept here, so the parameter is
not just invalid — it was doing nothing semantically. Removing it is a pure
no-op at runtime.

---

# Fix 122.1 — Remove `autofocus` from the Playback `ExpansionTile`

**File:** `lib/settings_view.dart`

**Current (around lines 1355–1358):**

```dart
                  ExpansionTile(
                    key: const PageStorageKey('playback'),
                    autofocus: true,
                    leading: const Icon(Icons.play_circle_outline),
```

**Replace with (delete only the `autofocus: true,` line):**

```dart
                  ExpansionTile(
                    key: const PageStorageKey('playback'),
                    leading: const Icon(Icons.play_circle_outline),
```

### Key changes

1. **One line deleted** — `autofocus: true,` from the `ExpansionTile` with
   `key: const PageStorageKey('playback')`. Nothing else on the tile changes.
2. **No runtime behaviour change** — the parameter was never honoured by
   `ExpansionTile`; the tile still starts collapsed (`initiallyExpanded:
   false` is untouched).
3. **The `TextField` `autofocus` at ~line 1990 is NOT touched** — it is valid
   and intentional.

### Verification

After the edit:

```
grep -n "autofocus" lib/settings_view.dart
```

**Expected:** exactly ONE line — the `TextField` autofocus, now at ~line 1989
(one earlier, since a line was deleted above it). If grep returns **zero**
lines you deleted the wrong occurrence — restore the `TextField` one.
