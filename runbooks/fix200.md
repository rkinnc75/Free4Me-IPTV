Log `free4me_log-20260602-092640.txt`: every "Home: switching view â†’ â€¦" line reports `sources=2` (filters already populated).
> **Bug:** The `_sourceColors` map is loaded inside the `if (widget.home.filters.sourceIds == null)` guard in `initializeAsync()`. Tab switches rebuild `Home` but carry `sourceIds` forward, so the guard is false and the color map never loads on those paths â€” leaving channel cards untinted.

The color load is gated on `sourceIds == null`, so it runs only on a fresh Home (e.g. returning from Settings, which builds Home with null filters). ## Fix 200.1 â€” load the color map on every Home init

**File:** `lib/home.dart`

One edit. Move the `_sourceColors` load out of the `sourceIds == null` guard so it runs unconditionally; leave `getEnabledSourcesMinimal()` + the `sourceIds` assignment inside the guard.

### Current code (verbatim â€” match exactly)
```dart
    if (widget.home.filters.sourceIds == null) {
      final sources = await Sql.getEnabledSourcesMinimal();
      // fix196: build sourceId -> tag color map for channel tiles (one call;
      // source count is tiny). Null entries simply mean no tint.
      final fullSources = await Sql.getSources();
      _sourceColors = {
        for (final s in fullSources)
          if (s.id != null) s.id!: s.color,
      };
      if (!mounted) return;
      widget.home.filters.sourceIds = sources.map((x) => x.id).toList();
    }
```

### Replacement code (verbatim)
```dart
    // fix200: load the source tag-color map on EVERY Home init, not only when
    // filters are fresh. Tab switches (Live/Favorites/History) rebuild Home but
    // carry sourceIds forward (see updateViewMode), so the old placement inside
    // the `sourceIds == null` guard ran only when arriving from Settings â€” which
    // is why colors appeared from Settings but vanished on Liveâ†’Favorites.
    final fullSources = await Sql.getSources();
    _sourceColors = {
      for (final s in fullSources)
        if (s.id != null) s.id!: s.color,
    };
    if (!mounted) return;

    if (widget.home.filters.sourceIds == null) {
      final sources = await Sql.getEnabledSourcesMinimal();
      if (!mounted) return;
      widget.home.filters.sourceIds = sources.map((x) => x.id).toList();
    }
```

### Key changes
- The `getSources()` call + `_sourceColors` map build now run unconditionally at the top, before the `sourceIds` guard.
- The guard retains only the `getEnabledSourcesMinimal()` call and the `sourceIds` assignment (its original purpose).
- Added a `mounted` check after the await (matches the existing pattern in this method).
