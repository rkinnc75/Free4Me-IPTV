## What this does
- Adds `channels.is_adult` (migration 24) — stored when the provider sends `is_adult`; providers without it contribute 0.
- At import, `is_adult = (provider is_adult == 1) OR (name/group matches the hardcoded safeModeBlocklist)`. So the blocklist becomes data in the column, not a runtime query.
- Every channel-path safe-mode filter simplifies to one indexed check `COALESCE(c.is_adult,0)=0` (was a per-term LIKE chain). The search cache reads the stored flag instead of re-scanning the blocklist.
- Categories (groups) safe-mode clause is unchanged — groups have no per-channel adult flag; a category is still hidden by name match.

## Verified before delivery
- `flutter analyze --no-fatal-infos` after all edits → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Patch applies cleanly to a fresh clone of main (`git apply --check` passed).
- Rule 8 (real SQLite): migration 24 on a seeded prior-schema DB adds the column with existing rows defaulting to 0 (no data loss); the `is_adult=0` filter hides adult rows and keeps the rest.
- Import logic tested standalone: provider flag=1 → adult; provider 0/clean → not; provider null + blocklist name/group → adult; M3U (no flag) honors blocklist.

## Effect timing
`is_adult` is populated at import, so the provider-flag and blocklist values land on the **next source refresh**. Existing rows are `is_adult=0` until then (migration default). Safe mode still works immediately for newly-imported data.
