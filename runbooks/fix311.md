## What this does
On TV, whenever the QR/LAN export HTTP server starts — from ANY entry point — it now always offers four downloads, all sharing one timestamp:
1. Source dump (`free4me-source-dump-<stamp>.txt`)
2. Debug log (`free4me_log-<stamp>.txt`)
3. Settings backup (`free4me-settings-<stamp>.json`)
4. Combined zip of all three (`free4me-export-<stamp>.zip`)

Previously the "export" entry point only served the source dump, and the other entry point served only backup + log. Both now route through one `_buildExportBundle` helper, so the set is identical regardless of how the server is started. Any file that can't be produced (e.g. no source dump captured yet) is omitted, and the zip contains whatever was produced.

## Verified before delivery
- `flutter pub get` resolves `archive ^3.6.1`; `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Built against live main (1.26.26+310); patch applies cleanly to a fresh clone.
- Zip logic tested standalone: 3 files encoded → decoded back to the same 3 names.
- The existing `ExportServer` already renders N items as individual download links + serves each by key; no server change needed — only the item list it's given.

## Note
- Non-TV (phone/PC with SAF) export paths are unchanged — they still use the native file picker per file. This fix is the TV LAN-server bundle only, per the request.
