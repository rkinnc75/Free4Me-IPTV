# fix307 — Disable a source's other actions when the source is disabled → v1.26.23+307

> ## ⚠️ BUILD MACHINE — APPLY THE PATCH, then RELEASE (EXACTLY). No questions, no DB to query.
> **FIRST:** `find . -name 'fix296-DIAGNOSE*.md' -delete 2>/dev/null || true`

## What this does
On the Sources list, when a source is disabled (`!source.enabled`), its other controls are now disabled too: the refresh button, edit button, delete button, and the leading color-picker tap all get `onPressed/onTap: null` (greyed, non-interactive). The enable/disable Switch stays active so the source can always be re-enabled. The row already dims to 50% opacity when disabled; this makes the buttons match that state functionally.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs (line numbers shifted to 2244/2839 from the added lines; same known baseline), no warnings/errors.
- Built against live main (1.26.22+306); patch applies cleanly to a fresh clone.
- The Switch's `onChanged` is untouched, so enable/disable always works. M3U sources still hide refresh/edit via the existing `Offstage` (unaffected).

## Note for Rich
- This disables **delete** for a disabled source too, per "all other functions." If you'd rather be able to delete a source while it's disabled, say so and I'll leave delete enabled in a follow-up.
