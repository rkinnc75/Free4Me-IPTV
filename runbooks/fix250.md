# fix250 — TV: D-pad access to the cell options menu + focusable "+" in multi-view (item 4)

> **Version:** build = highest fix number; patch = next available. Tag = next immutable.
> **Builds on:** current source **1.25.4+246**. Touches `lib/multi_view_cell.dart` only.
> **Bugs (from TV testing):**
>   1. Once all cells in a 2×2 / 1×2 are filled, there was NO way to open a cell's options (Replace channel / Full screen / Close) on a TV — the menu was bound to `GestureDetector.onLongPress`, which only fires for touch, not a D-pad long-press.
>   2. The empty-cell "+" button (a `FloatingActionButton`) had no autofocus and only a weak focused state, so it was hard to land on / see with the D-pad.
> **Fix:** map the D-pad select/center (and the dedicated menu/context key) to open the cell menu via Flutter `shortcuts`/`actions`; give the "+" a real focus ring and make it the initial D-pad target when a grid opens empty. Touch long-press still works on phones/tablets.

## Pre-verified facts (checked against 1.25.4, compiled)

- File: `lib/multi_view_cell.dart` only. Adds `package:flutter/services.dart` (for `LogicalKeyboardKey`), a small `_CellMenuIntent`, an `_addButtonFocused` state flag.
- Filled cells: `FocusableActionDetector` gains `shortcuts` (select / enter / gameButtonA / contextMenu → `_CellMenuIntent`) and `actions` (focus the cell, then `_showCellMenu()`).
- Empty cells: the FAB is replaced with a `FocusableActionDetector` + highlighted circular button; `autofocus` on cell 0; select/enter/gameButtonA → open the channel picker.
- **Compile:** `flutter analyze --no-fatal-infos` → 2 tolerated INFOs. Rule 6 clean.

### Replacement code (verbatim)
```dart
  Widget _buildEmptyCell() {
    // fix250: make the "+" reachable and clearly highlighted via D-pad.
    // FloatingActionButton had no autofocus and a weak focused state, so on
    // TV it was hard to land on. FocusableActionDetector gives it a visible
    // focus ring and lets cell 0's "+" be the initial D-pad target when the
    // grid opens empty.
    return Container(
      color: const Color(0xFF111111),
      child: Center(
        child: FocusableActionDetector(
          autofocus: widget.cellIndex == 0,
          shortcuts: const <ShortcutActivator, Intent>{
            SingleActivator(LogicalKeyboardKey.select): _CellMenuIntent(),
            SingleActivator(LogicalKeyboardKey.enter): _CellMenuIntent(),
            SingleActivator(LogicalKeyboardKey.gameButtonA): _CellMenuIntent(),
          },
          actions: <Type, Action<Intent>>{
            _CellMenuIntent:
                CallbackAction<_CellMenuIntent>(onInvoke: (_) {
              _pickChannel();
              return null;
            }),
          },
          onShowFocusHighlight: (f) => setState(() => _addButtonFocused = f),
          child: GestureDetector(
            onTap: _pickChannel,
            child: Container(
              width: 56,
              height: 56,
              decoration: BoxDecoration(
                color: _addButtonFocused
                    ? Theme.of(context).colorScheme.primary
                    : const Color(0xFF2A2A2A),
                shape: BoxShape.circle,
                border: Border.all(
                  color: _addButtonFocused
                      ? Colors.white
                      : Colors.white24,
                  width: _addButtonFocused ? 3 : 1,
                ),
              ),
              child: const Icon(Icons.add, color: Colors.white),
            ),
          ),
        ),
      ),
    );
  }
```

## Notes
- `LogicalKeyboardKey.select` is the standard Android-TV D-pad center; `enter`/`gameButtonA` cover remotes/controllers that report differently; `contextMenu` covers remotes with a dedicated menu key. This multi-key mapping is intentional for remote variety.
- Item 3 (focus starts on first tile generally + "up" enters the search bar, applied across all channel selectors) is a separate, broader focus-traversal change — see fix252.
- This does NOT change the audio-focus behavior (focused cell plays, others mute) — that path (`onShowFocusHighlight` → `onFocusTap`) is preserved.
