# fix503 — EPG guide grid (category rail + channel × time) for the Live tab

**Version:** 2.0.2+502 → **2.0.3+503**
**Tag:** v2.0.3
**Type:** Feature (large). Net-new TV guide. Includes the category rail (moved from fix501).
**Status:** Implemented + verified locally on Flutter 3.44.0 (analyze 0; 169 tests incl. the grid EXPLAIN test); shipped via CI.
**Base:** fix502.

## What
The Live tab is now an EPG guide (`lib/tv/tv_guide_view.dart`):
- **Category rail** (left) — live groups scope the grid by `groupId` (the rail moved here from fix501, since the guide is rail-scoped).
- **Channel × time grid** (right) — a fixed forward window (next 3 h, one shared time axis, no horizontal scroll), virtualized vertically (`ListView.builder`); programmes are proportional blocks with a "now" line. Select a program/channel to play.

## Backend
- `Sql.getGridPrograms(epgIdsBySource, windowStart, windowEnd)` — programmes for the realized channel set, driven by a **bounded `epg_channel_id IN`-list per source** (chunked < 900); grouped/sorted in Dart (no ORDER BY → no temp B-tree).
- **No new index / migration.** The perf test proved the existing `idx_programs_channel_time` already serves the IN-list seek (no source-wide or full scan), so the +36 % refresh write cost was avoided (measure-first). db.sqlite stays at 34; epg.sqlite stays at 2.
- Rail-scoped + a 200-channel cap so the grid never touches the full ~1.15M catalogue.

## Perf gate (passed)
`test/epg_guide_grid_query_test.dart`: 2 sources × 200 channels × 30 programmes seed + ANALYZE; EXPLAIN shows the IN-list query SEARCHes via `idx_programs_channel_time`, no full `SCAN programmes`. Absolute < 1000 ms cold on the cheapest box remains an on-device confirmation.

## Phone-mode safety
TV-only: `tv_guide_view` is mounted only by the shell's Live tab; `getGridPrograms` is a new method (no phone callers). Phone EPG views (`now_next_strip`, `channel_schedule`) unchanged.

## Known follow-up (on-device)
The grid's D-pad focus traversal + the absolute cold-timing on a cheap box benefit from on-device iteration (not reproducible on the headless authoring box). A horizontal-scroll / wider-window mode is a future enhancement.
