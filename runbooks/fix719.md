# fix719 — TV GUI Phase 5 unit 2: accent-color picker (v4.1.23)

**Builder:** desktop. **Ships as:** v4.1.23+719. TV only; phone/touch UI
byte-identical. Autonomous ("keep going, pick recommended option"). Owner chose
the palette (2026-07-12): **White + Sky Blue + Amber + Magenta + Green**
("White + 4 curated colors"). Plan: `docs/TV_GUI_REDESIGN.md` §4.10/§6 (Phase 5).

## What
The payoff of the accent-ring language built across Phases 1-4: a Settings picker
that swaps the focus-ring accent app-wide, live, persisted. Was white-only.

## How (5 files)
- **`accent_scope.dart`**: `class AccentPreset(id,label,color)` + `const
  kAccentPresets` (white/skyblue/amber/magenta/green) + `accentColorFromId(id)`
  (firstWhere id, orElse → first = White; safe for retired/unknown ids).
- **`models/settings.dart`**: `String accentName` (default `'white'`).
- **`backend/settings_service.dart`**: `accentNameProp`; `_readFromDb` reads it
  (guarded `!= null && isNotEmpty` → else keep default); `updateSettings` writes it.
- **`main.dart`**: startup `appAccentNotifier.value =
  accentColorFromId(settings.accentName)` before first frame; `build()` wraps
  `MaterialApp` in `ValueListenableBuilder<Color>(appAccentNotifier)` so the
  fix707 button-theme rings (read `appAccentNotifier.value` in `resolveWith`)
  re-resolve on change. AccentScope widgets already update via InheritedNotifier.
- **`settings_view.dart`**: `_accentColorTile()` (row of `_AccentSwatch` InkWell
  circles; selected = white ring + luminance-contrasted check) into
  `_playbackChildren` gated `if (widget.tvRailPane)`. Select → `settings.accentName
  = id; appAccentNotifier.value = color; updateSettings()`.

## P0 / design notes
- **Phone byte-identical**: picker row gated on `tvRailPane` (never in the phone
  ExpansionTile list); button themes gate on `!hasTouchScreen` so the accent value
  never affects a phone frame → a MaterialApp rebuild yields an identical phone UI.
- **Texture safety**: the `ValueListenableBuilder` REBUILDS (not remounts)
  MaterialApp — Element reuse preserves the Navigator + the `const
  OverlayPlayerWidget` mini-player Texture; the rebuild only fires on the rare
  accent pick. (accent_scope's "no MaterialApp rebuild" note was about avoiding
  CONTINUOUS churn; a one-off is fine.)
