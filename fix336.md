## Why
VOD on the ExoPlayer engine had NO transport controls — no play/pause, seek, or skip. The libmpv path gets full controls from media_kit's own UI, but the ExoPlayer path used a custom overlay (`_buildExoControls`) that only had a top bar (back/title/cast/PiP) and a live mini-player button. The `PlayerEngine` interface also had no pause/seek/duration, so there was nothing to wire a bar to.

## What this does
1. **Interface (`player_engine.dart`):** adds `duration`, `playingStream`, `isPlaying`, `pause()`, `play()`, `seek()`.
2. **ExoEngine:** implements them on the `VideoPlayerController` (real seek/pause), plus a `playingStream`.
3. **MpvEngine:** implements them for parity (`_player.pause/play/seek`, `state.duration`, `stream.playing`).
4. **Player UI (`player.dart`):** subscribes to position/playing; renders a VOD transport bar in the ExoPlayer overlay — back-10s, play/pause (autofocus for D-pad), forward-10s, a seek slider with scrub handling, and elapsed/total time. Shown only when `duration > 0` (VOD); hidden for live (duration 0). Seek clamps to [0, duration]; scrubbing pauses position updates until release.
