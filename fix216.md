## Notes — prevention (carry forward)
- Release-note TEXT in `_changelog` needs the same quote-safety as code. When adding a `'X.Y.Z': [...]` entry, any apostrophe in a single-quoted string MUST be `\'`, OR write the note as a double-quoted string `"..."` (escaping any inner `"`). Apostrophe-prone words: What's, doesn't, won't, can't, provider's, user's, it's.
- A green `flutter analyze` locally/in-sandbox before tagging would have caught this. See the companion CI runbook (fix218) to make the Release workflow gate on Analyze.
