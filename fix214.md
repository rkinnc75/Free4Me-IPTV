# fix214 — HOTFIX: unescaped apostrophe in 1.23.23 changelog string breaks the build

> **Version:** build = highest fix number in release; patch = next available (read live pubspec). Tag = next immutable `v1.23.X`.
> **Severity:** BUILD-BREAKING. `flutter analyze` fails with 10 errors; the app cannot compile until fixed.
> **Symptom (build machine email):** errors in `lib/whats_new_modal.dart` such as `Undefined name 's'`, `Undefined name 'New'`, `Unterminated string literal`, `Const variables must be initialized with a constant value`, `Expected to find ','` — all pointing at the same changelog line.
> **Root cause:** the 1.23.23 changelog entry is a SINGLE-quoted Dart string that contains an apostrophe in `What's`:
> ```dart
> 'Fix: "What's New" showed stale changelog entries from old patch versions. '
> ```
> The `'` in `What's` closes the string after `What`, so `s New"...` is parsed as broken Dart. This is a Rule-6-class bug (unescaped quote in a string literal) — note it was introduced in the human-readable changelog TEXT the build step authored for the 1.23.23 release, not in any fix's code.

## Pre-verified facts (checked against current `main` = `1.23.25+212`, compiled)

- **File:** `lib/whats_new_modal.dart`. The broken entry is the `'1.23.23'` changelog value (around line 20; line number drifts as entries are prepended — anchor on the text, not the line).
- Other `What's` occurrences in the file are SAFE and must not be touched: the `///` doc comments (lines ~4, ~1343, ~1375) and the double-quoted title `Text("What's new in $version")` (~line 1411). Only the single-quoted changelog string is broken.
- **Fix:** escape the apostrophe (`What\'s`). Chosen over switching to a double-quoted string because the text also contains literal double-quotes (`"What's New"`), which would then need escaping instead.
- **Compile status:** after the escape, `flutter analyze --no-fatal-infos` on the full current source (with fix208/210/212 all applied) → exactly the 2 tolerated `settings_view.dart` INFOs. Rule 6 grep clean (`\'` is a valid Dart escape; it is not a `\$` or `\u` violation).

## Fix 214.1 — escape the apostrophe in the 1.23.23 changelog entry

**File:** `lib/whats_new_modal.dart`

### Current code (verbatim — match exactly)
```dart
    'Fix: "What's New" showed stale changelog entries from old patch versions. '
```
### Replacement code (verbatim)
```dart
    'Fix: "What\'s New" showed stale changelog entries from old patch versions. '
```

### Key change
- A single backslash before the apostrophe in `What's` → `What\'s`. Nothing else changes. The surrounding entry (`'1.23.23': [ ... ]`) and all other lines are untouched.

## Notes — prevention
- **Changelog note TEXT needs the same quote-safety as code.** When the build step writes a release note as a single-quoted Dart string, any apostrophe (`What's`, `doesn't`, `provider's`, `can't`) must be escaped (`\'`) or the string must use double quotes. Future release-note generation should escape apostrophes by default, or wrap notes in double-quoted strings (escaping any inner `"`).
- This is independent of fix208/210/212 (all three are correctly applied and compile once this apostrophe is escaped). It is purely a fix to authored changelog text.
