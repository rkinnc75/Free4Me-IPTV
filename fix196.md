# fix196 — Per-source color tagging (pick a pastel per source; tint channel cards)

> **Version:** build `+196`; patch = next available at ship time (read live pubspec — patch and fix numbers drift; do NOT assume). Tag = next available `v1.23.X` (immutable; never reuse/delete/recreate).
> **Evidence:** With multiple sources enabled, channels from different sources are visually indistinguishable in the channel list. User wants to tell at a glance which source a channel came from.
> **Feature:** (1) Tapping the monitor icon on a source row opens a pastel color picker (8 pastels + None); the choice is saved per-source (optional — None = no color). (2) In the channel list, each channel card is tinted with its source's chosen color so the source is identifiable at a glance.
> **Migration:** adds `color` column to `sources` — **DB migration 17** (next integer after the shipped 16).

## Scope decisions (locked with the user)

- Tint **the whole channel card** (not just the name area).
- Pick color by **tapping the monitor (leading) icon** on the source row.
- **8 pastels + None**, no custom picker, **no new dependency**. Palette approved from a rendered sample.
- Optional: a source with no color chosen shows no tint (default for all existing sources).

## Pre-verified facts (checked against the uploaded `v1.23.17+192` source, compiled, migration-tested)

- **Migration site:** `lib/backend/db_factory.dart` — migrations are `..add(SqliteMigration(N, (tx) async {...}))`; highest shipped is **16** (`max_connections`). Next integer **17**.
- **Source DB mapping is positional `SELECT *`** in `lib/backend/sql.dart` `rowToSource`: id(0)…default_engine(8), max_connections(9). An `ALTER TABLE … ADD COLUMN` appends `color` at **index 10** (verified on a migrated DB; SQLite ADD COLUMN appends at end).
- **`Source` model** (`lib/models/source.dart`) has `int? maxConnections;` and a named-param constructor — `color` follows the identical pattern.
- **`updateSource`** (`lib/backend/sql.dart`) is the save path used by the existing source UI; adding `color = ?` to its SET list persists it. (The source **INSERT** path at add-time does NOT set max_connections and does NOT need to set color either — new sources start with NULL color, set later via the picker. No INSERT change.)
- **Channel→source:** `Channel.sourceId` exists (`lib/models/channel.dart:12`). The channel list is built in `lib/home.dart` (`ChannelTile(...)` at the SliverGrid builder), which already loads sources; a `Map<int,int?>` sourceId→color is built once from `Sql.getSources()` (source count is tiny) and passed per tile — no async lookup in tile build.
- **Channel tile background** is `Card.color` at `lib/channel_tile.dart` (was `Theme.of(context).colorScheme.surfaceContainer`). Tinting via `Color.lerp(base, tag, 0.35)` keeps the name legible and preserves the scan border + focus elevation.
- **Color API:** uses `Color.lerp` and `Color(int)` — NOT the deprecated `.red/.green/.blue` accessors (this Flutter bundles Dart 3.12.1 where those are deprecated; confirmed by analyze).
- **Rule 8 migration test (in-sandbox sqlite3 3.45.1):** seeded a sources table at the prior (migration-16) schema with 2 rows, ran `ALTER TABLE sources ADD COLUMN color INTEGER`. Result: `color` at index 10, existing rows NULL, `SELECT *` returns max_connections at 9 and color at 10, and the new `updateSource` UPDATE (with `color = ?`) writes/reads back `0xFFB3E5FC` correctly while leaving the other source's color NULL.
- **Compile status:** all edits applied to the uploaded `1.23.17+192` tree; `flutter analyze --no-fatal-infos` → **exactly the 2 pre-existing tolerated INFOs** in `lib/settings_view.dart` (shifted to ~2178/2766 by the settings edit). Rule 6 grep (`\$`, `\u`) across all 7 touched files: clean.

## Fix 196.1 — NEW FILE: `lib/source_color_picker.dart`

**Create this file** (it does not exist). Full contents:

```dart
import 'package:flutter/material.dart';

/// fix196: the approved pastel palette for per-source color tagging.
/// Values are full ARGB ints (stored in sources.color; null = None/no tint).
class SourcePalette {
  static const List<({String name, int? value})> swatches = [
    (name: 'None', value: null),
    (name: 'Rose', value: 0xFFF8BBD0),
    (name: 'Peach', value: 0xFFFFCCBC),
    (name: 'Butter', value: 0xFFFFF9C4),
    (name: 'Mint', value: 0xFFC8E6C9),
    (name: 'Sky', value: 0xFFB3E5FC),
    (name: 'Periwinkle', value: 0xFFC5CAE9),
    (name: 'Lavender', value: 0xFFE1BEE7),
    (name: 'Aqua', value: 0xFFB2DFDB),
  ];

  /// Blend a stored source color (~35%) over a base surface color so channel
  /// text stays legible. Returns base unchanged when color is null.
  static Color tintOver(int? color, Color base) {
    if (color == null) return base;
    // Color.lerp(base, color, 0.35): 35% toward the tag color, opaque result.
    return Color.lerp(base, Color(color), 0.35) ?? base;
  }
}

/// Shows a pastel picker. Returns a record indicating whether the user made a
/// choice and the chosen color (null = None). Returns null if dismissed.
Future<({bool chose, int? color})?> showSourceColorPicker(
  BuildContext context, {
  int? current,
}) {
  return showDialog<({bool chose, int? color})>(
    context: context,
    builder: (ctx) {
      return AlertDialog(
        title: const Text('Source color'),
        content: SizedBox(
          width: double.maxFinite,
          child: GridView.count(
            shrinkWrap: true,
            crossAxisCount: 3,
            mainAxisSpacing: 12,
            crossAxisSpacing: 12,
            children: [
              for (final s in SourcePalette.swatches)
                _Swatch(
                  name: s.name,
                  value: s.value,
                  selected: s.value == current,
                  onTap: () =>
                      Navigator.of(ctx).pop((chose: true, color: s.value)),
                ),
            ],
          ),
        ),
        actions: [
          TextButton(
            onPressed: () => Navigator.of(ctx).pop(null),
            child: const Text('Cancel'),
          ),
        ],
      );
    },
  );
}

class _Swatch extends StatelessWidget {
  final String name;
  final int? value;
  final bool selected;
  final VoidCallback onTap;
  const _Swatch({
    required this.name,
    required this.value,
    required this.selected,
    required this.onTap,
  });

  @override
  Widget build(BuildContext context) {
    final isNone = value == null;
    return InkWell(
      autofocus: selected,
      borderRadius: BorderRadius.circular(10),
      onTap: onTap,
      child: Container(
        decoration: BoxDecoration(
          color: isNone ? Colors.transparent : Color(value!),
          borderRadius: BorderRadius.circular(10),
          border: Border.all(
            color: selected ? Colors.greenAccent : Colors.grey,
            width: selected ? 3 : 1,
          ),
        ),
        child: Center(
          child: Text(
            name,
            textAlign: TextAlign.center,
            style: TextStyle(
              color: isNone ? Colors.grey : Colors.black87,
              fontWeight: FontWeight.w600,
              fontSize: 13,
            ),
          ),
        ),
      ),
    );
  }
}
```

> This is the ONE new `.dart` file (preflight Rule 11 exception: a new standalone widget; its whole-file balance matters, so it is provided complete rather than as fragments). All remaining changes below are `str_replace` edits to existing files.
