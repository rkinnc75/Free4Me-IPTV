# fix537 — Eliminate enable/disable-source delay (cat_enabled index write-amplification)

**Base tag:** `v2.0.36`
**Version:** `2.0.36+536` → `2.0.37+537`
**Scope:** new DB migration 39 (rebuild browse indexes without `cat_enabled`, drop 5 unused, one-time VACUUM) + test updates. No query-path change (VisibilityClause already emits the residual `cat_enabled = 1`). Full suite green.

---

## Problem (root-caused on the real field DB)

Enabling/disabling a source — or Select-All/Unselect-All on categories — took **10–72 s** on the onn 4K box. The 1.43 GB field `db.sqlite` (1.2 M channels, ~1 GB of indexes) revealed the cause: **8 browse indexes were PARTIAL on `cat_enabled = 1`.** Toggling `cat_enabled` on hundreds of thousands of rows forces SQLite to insert/remove every affected row in every one of those partial indexes — massive write amplification.

## Fix (measured on the field DB)

Remove `cat_enabled` from the 7 browse indexes' partial predicate so toggling it touches **zero** browse indexes; the browse queries keep these indexes and apply `cat_enabled = 1` as a cheap residual filter on the already-narrowed page (VisibilityClause already emits it — no query change). Also drop 5 never-selected indexes and VACUUM once to reclaim freed pages.

Measured, real DB:

| Metric | Before | After | Win |
|---|---|---|---|
| Enable all-source live toggle | 11002 ms | **602 ms** | **18×** |
| DB size | 1426 MB | **1135 MB** | −291 MB |
| Browse (alpha/provider/category × safe/non-safe × mt-only/src-scoped) | index-served | index-served, no temp B-tree | no regression |
| Worst case (fully-disabled source) | — | 0 rows in 37 ms (index-served) | no full scan |

Empirically verified (EXPLAIN QUERY PLAN on the field DB) that **only 5 indexes are genuinely unused**: `idx_browse_cat`, `idx_browse_cat_safe` (planner always prefers `idx_channels_browse_mt_safe`), `index_channel_name` (covered by `index_channel_name_source`), `idx_channel_adult`, `idx_channel_divider`. The provider/alpha/source-led browse indexes are all kept (just cat_enabled-free).

`Sql.withDroppedBrowseIndexes` (refresh) recaptures index DDL live from `sqlite_master`, so it automatically recreates the new cat_enabled-free DDL after migration 39 — no change needed there.

### One-time cost
Migration 39 rebuilds 7 indexes (~17 s on the field DB) and the post-migrate VACUUM (~21 s here; minutes on the TV box). Gated by a `app_meta` marker so VACUUM runs once. The changelog warns the user the first launch may be slow.
