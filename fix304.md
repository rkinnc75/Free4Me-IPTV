# fix304 — Pin search bar + radar/multi-view icons; scanner starts at first visible tile → v1.26.20+304

> **Note:** the home.dart diff is large because the search-bar Row was moved out of the scrolling sliver list (lines re-indent). Apply the patch as-is; do not hand-merge.

## What this does (two features)
1. **Pinned search panel.** The search bar and the radar + multi-view buttons were inside the first sliver of the `CustomScrollView`, so they scrolled away. They're now the fixed first child of a `Column`, with the grid in an `Expanded(CustomScrollView(...))` below — so they stay pinned at the top while the grid scrolls.
2. **Scanner starts at the first visible tile.** `_startScan` previously scanned `channels` from index 0. It now computes the first on-screen tile from the scroll offset (tiles are `mainAxisExtent=100` in a `crossAxisCount` grid; the pinned bar means offset maps directly to rows), slices `channels` from there, and scans `streamScanMaxCount` downward. Scroll 2-3 pages, hit radar → it scans the settings count starting at the first visible channel.

## Verified before delivery
- `flutter analyze --no-fatal-infos` → 2 tolerated `settings_view.dart` INFOs, no warnings/errors.
- Patch applies cleanly to a fresh clone and compiles with 0 errors there.
- Layout: search Row is the first `Column` child (pinned); grid `CustomScrollView` is in `Expanded` below; `crossAxisCount`/grid delegate unchanged; empty-state `SliverFillRemaining` still inside the scroll view.
- Scanner: `startIndex = (offset/100).floor() * cols`, clamped to `[0, channels.length-1]`; `channels.sublist(startIndex)` passed to `StreamScanner.scan` with the unchanged `maxChannels`. Guards `_scrollController.hasClients`.

## Note
- The visible-tile math assumes the grid tile height (100) and column rule (`width/350`, clamped 1–3) — both unchanged in this fix. If either is changed later, update `_startScan` to match.
