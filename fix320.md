# fix320 — Fix Dino refresh crash on a null category; group such channels as "Uncategorized" → v1.26.36+320

## Root cause (traced from log stack + Dino dump)
Log: `Settings: refresh "Dino" — ERROR: type 'Null' is not a subtype of type 'String' in type cast`, stack `Sql.wipeSource ... sql.dart:985`.

In the Dino dump, a series record (`DE - NCIS: New Orleans (2014)`) and a live record (`FR - GOLF+`) have `category_id: null`. That produced a group with a NULL `name`; when that group was disabled, `wipeSource` cast the null name `as String` and threw, aborting the refresh **mid-wipe** so series never committed (`series=0`). Root of the recurring Dino failure and a cause of partial-catalogue commits.

## What this does
Null categories now become **"Uncategorized"** instead of nameless groups (so the channels stay browsable), and the wipe path is null-safe:
- `xtreamToChannel`: `group: categoryName ?? 'Uncategorized'`.
- `m3u` parse: missing/empty `group-title` → `'Uncategorized'`.
- `Sql.wipeSource`: disabled-group-name collection maps a null name to `'Uncategorized'` (no more `as String` crash), matching the synthetic name above.

No schema change.
