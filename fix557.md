# fix557 — TV search: consistent grid layout + remove the 36-result cap

**Version:** 2.0.57 (`2.0.57+557`)
**Base:** v2.0.56 (live CI tag). Single ```diff block, byte-exact. Verified
clean on a fresh v2.0.56 clone.
**Branch from:** v2.0.56.

---

## Three bugs, one root model (Netflix-shelf layout + a silent shared page cap)

1. **Inconsistent card sizes.** Search used its own card sizing (poster 168×270,
   landscape 320×104) instead of the Categories/Movies/Series tile (120/0.773,
   fix553).
2. **Wrong pagination model.** Each section was a single horizontal-scrolling
   row (`ListView.builder(scrollDirection: horizontal)`), not the wrapping
   multi-row grid every other TV screen uses.
3. **Hard 36-result cap.** `Sql.search`'s `LIMIT` used the global `pageSize`
   constant (36) unconditionally. TV search never paginated past page 1, so any
   query silently capped at 36 — "fox" showed 34 results while the real catalog
   has **4,000+** matching channels (confirmed against the field seed).

## Fix

- **`Filters.limit`** (new, nullable): per-call override for the page size.
  **Defaults to `null` → behaves exactly as `pageSize` for every existing
  caller** (phone browse, TV browse/guide, favorites, history) — zero risk to
  anything else. Threaded through `Sql.search`'s FTS branch, `_searchInMemory`,
  and `_searchLike`.
- TV search passes **`limit: 1000`** — a safe ceiling so a real query returns
  effectively everything, without a 1–2 char query pulling the whole catalog.
- **Layout:** `tv_search_view.dart`'s `_shelf` (horizontal, mixed sizes) is
  replaced by `_section` — a titled, **wrapping `GridView`** at the unified
  120/0.773/10 tile size (shrink-wrapped, non-scrolling; the outer `ListView`
  still owns the page scroll). All five sections (On now / Coming up / Channels
  / Movies / Series) now use the same poster tile — On now/Coming up drop the
  inline now/next text strip (poster mode doesn't render it) in exchange for
  visual consistency with the rest of the TV UI.

## Verification

- `flutter analyze` → **No issues found!**
- `flutter test` → **259 passed**
- `git apply --check` → clean on a fresh v2.0.56 clone
- Confirmed against the field DB seed: "fox" matches **4,946** enabled channels
  (was capped to 34/36).

## On-device check after CI

1. Search "fox" — results now show as a wrapping multi-row grid, same card size
   as Movies/Series/Categories, far more than 36 results (capped at 1000).
2. On now/Coming up sections (if any EPG matches) use the same grid tile.
3. Normal phone/TV browse, favorites, history — unaffected (no `limit` set,
   still uses `pageSize`).
