# fix407 — Restore working playback controls (revert fix406's lowered-button hack; keep EPG)

**Target version:** `1.35.6+407` (from `1.35.5+406`)
**Branch:** `main` · **Base:** `d1cb749` (v1.35.5+406)
**Toolchain (verified):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.

## What & why

fix406 lowered the single-cell centre control row by wrapping each `primaryButtonBar` button in a `Transform.translate`. That shipped two regressions:
1. **Buttons stopped responding to taps.** media_kit hard-centres `primaryButtonBar` and lays a gesture surface over the control area. `Transform.translate` moves the buttons' *paint* down but leaves them outside the region media_kit routes taps to, so taps land on the background gesture layer instead of the buttons.
2. **Wrong position when entering from portrait.** The offset was snapshotted from `MediaQuery.height` at the first build, which (because the player forces landscape) happens while still portrait — so it used the tall portrait height and never recomputed for landscape.

Both stem from fighting media_kit's centred bar with a `Transform`. The correct way to place these controls low is to use media_kit's **bottom** bar (which is correctly positioned and tappable) — a deliberate follow-up, not another blind hack.

**This fix reverts the lowering** so the controls return to media_kit's default centre, where they work reliably. **The EPG top-bar feature from fix406 is kept** (it was unaffected).
