## Why (S24 21:38 log on 1.26.57 + external runbook review)
All four Exo cells fired `startup watchdog fired after 30s — open succeeded but no frame` right after the stability buffer released, looping retries; the libmpv segment of the same log was clean (fix341 quick re-open: 21 re-opens, 0 failures). Root cause — a liveness/arming RACE: the cell's buffering listener is attached BEFORE `engine.open()`, but the watchdog was armed AFTER. ExoPlayer auto-plays during open(), so its ONE-SHOT fix335 liveness signal could arrive (and did, every time with the buffer) before the watchdog existed — consumed. Exo never toggles isBuffering on raw `.ts`, so nothing later cancelled the watchdog. The external runbook's quoted sequence (`signalling liveness` -> `open() succeeded` -> `watchdog armed`) shows the same race latent on ALL paths; unbuffered cells were just winning it by timing.

## External-runbook review (what was adopted / rejected)
- **ADOPTED — Change #1, arm the watchdog BEFORE `engine.open()`:** structurally removes the race on every path; any liveness/buffering event now lands after arm and cancels. The generation guard keeps a fire during a slow open() safe.
- **ADOPTED — Change #2, `_cancelStartupWatchdog()` helper:** both cancel sites routed through it.
- **REJECTED — Change #3, cancel the watchdog right after open() succeeds:** defeats the watchdog's purpose. "open succeeded but no frame" is a REAL failure mode (the Shield texture-attach race, fix337). Not shipped.
- **REJECTED — Change #4, cancel only on buffering == false:** regression risk. The any-event semantic is deliberate (fix94): the FIRST buffering signal — true or false — proves the engine is alive; cancelling only on false would let the watchdog kill a genuinely slow-buffering stream. Unchanged.

## What this does
1. **Arm-before-open (`multi_view_cell.dart`):** the watchdog (with its fix341 buffer-window extension) is created BEFORE `engine.open()`; `bufferSecs` hoisted above the try so both the arm and the post-open buffer block share it.
2. **Helper:** `_cancelStartupWatchdog()`; both cancel sites use it.
3. **Belt 1 (kept):** when the stability buffer releases (`engine.play()`), the cell cancels the watchdog explicitly and logs `(buffer released)`.
4. **Belt 2 (kept, `exo_engine.dart`):** `play()` emits a liveness `buffering=false` so resuming always signals.
