# fix144.md — Build break from fix140: missing import for MediaType in multi_view_screen.dart

> **Version:** Free4Me-IPTV 1.22.12 (the fix136/138/140/142 build).
> **Evidence:** `flutter analyze --no-fatal-infos`, exit code 1:
> ```
> error • Undefined name 'MediaType'. Try correcting the name to one that is
> defined, or defining the name • lib/multi_view_screen.dart:157:41 •
> undefined_identifier
> ```
>
> **Cause:** fix140 added `ch.mediaType == MediaType.livestream` to
> `_restoreChannels()` in `multi_view_screen.dart`, but that file does not import
> `MediaType`. `MediaType` is defined in its OWN file (`lib/models/media_type.dart`),
> NOT in `channel.dart`. (fix140 incorrectly assumed it came with the already-
> present `channel.dart` import.) The two `use_build_context_synchronously` INFOs
> (~1860/2420) are the pre-existing tolerated ones — leave them.

---

## fix144.1 — Add the MediaType import

**File:** `lib/multi_view_screen.dart`

The file currently imports (lines 8–11):
```dart
import 'package:open_tv/models/channel.dart';
import 'package:open_tv/models/multi_view_layout.dart';
import 'package:open_tv/models/settings.dart';
import 'package:open_tv/models/source.dart';
```

**Add** the `media_type` import (alphabetical order puts it right after
`channel.dart`):

```dart
import 'package:open_tv/models/channel.dart';
import 'package:open_tv/models/media_type.dart';   // fix144: MediaType for fix140's livestream guard
import 'package:open_tv/models/multi_view_layout.dart';
import 'package:open_tv/models/settings.dart';
import 'package:open_tv/models/source.dart';
```

(This matches how `channel_picker_screen.dart` imports it — its own line
`import 'package:open_tv/models/media_type.dart';`.)

### Verify

```
flutter analyze --no-fatal-infos
```

`undefined_identifier` for `MediaType` must be GONE. Expected remaining: only
the two tolerated `use_build_context_synchronously` INFOs at settings_view.dart
~1860 / ~2420. Gate passes.
