# fix400 — Channel search waits for ≥2 characters (skip the 1-char full-catalogue scan)

**Target version:** `1.34.28+400` (from `1.34.27+399`)
**Branch:** `main` · **Base:** `b9c0e1e` (v1.34.27+399)
**Toolchain (verified):** Flutter 3.44.0 / Dart 3.12.0 (CI pin), `libsqlite3.so` symlinked.

## What & why

A device log (S24, v1.34.25+397) showed the channel search firing a substring query on the **first typed character**: a 1-char `name LIKE '%y%'` matches almost everything and scans the whole catalogue (~1.8–2.0 s on a large multi-source library) — only to be immediately superseded by the next keystroke. The result was correctly discarded by the debounce/supersede logic (so the user never waited on it), but it was real wasted work on every search.

Fix: gate the debounce so a single character doesn't run the query. Behaviour, per the agreed UX:
- **≥2 non-whitespace characters →** search (unchanged).
- **exactly 1 character →** skip; the current list is left untouched (no scan, no flicker).
- **empty / whitespace-only →** unchanged; still restores the full browse.

The decision is a one-line pure helper (`searchQueryShouldLoad`) so it's unit-testable in isolation; `home.dart`'s debounce callback returns early when it's false. No change to the search SQL, the cache, or the ≥2-char path.
