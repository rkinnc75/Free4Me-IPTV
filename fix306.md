## What this does
When a stream scan finishes, the persisted `streamValidated` results are now copied onto the in-memory channel objects in the current list, then `setState` refreshes the outlines. Previously the post-scan UI relied only on the transient `StreamScanner.results` map; once that map is cleared (next scan), validated tiles could lose their outline until a reload. This writes the flag through so it stays correct.

Done **in place** — no `load()`/re-query, so the scroll position is preserved (a full reload would reset to page 1 and jump to the top).
