# fix421 — diagnostic: time the cold connection setup (DNS / TCP / TLS) per channel start

**Version:** 1.35.19+420 → **1.35.20+421**
**Type:** Diagnostic only (no behaviour change to playback). New file + small hook.
**Tag:** v1.35.20
**Base:** fix420 (1.35.19+420). Apply fix419 → fix420 → fix421 in order.

## Why
Before optimising cold channel-start latency (the DNS pre-resolve / token pre-fetch ideas), we need to know where the pre-first-byte time actually goes. This logs the breakdown so the data — not an estimate — decides whether any of it is worth building. If DNS turns out to be 30 ms, there's nothing to win there.

## What it does (gated behind debug logging)
- New `lib/backend/conn_timing.dart`: `ConnTiming.probe(url)` times **DNS resolve → TCP connect → TLS handshake** for the stream's host and logs one line:
  `CONNPROBE host=<…> dns=Xms tcp=Yms tls=Zms total=Tms`.
- Hooked in `player.dart` `_startPlayback`, fired **once per user-initiated channel open** (a `_connProbeDone` flag skips it on reconnects), launched *before* mpv connects so it captures the cold DNS state, **fire-and-forget** (never blocks or fails playback).

## Safety properties (deliberate)
- **Sends no HTTP request** — a bare TCP+TLS handshake carries no credentials, so it does **not** consume a provider "max connections" slot the way an authenticated GET would.
- **Host is redacted** by the logger (fix415) like every other line; the raw exception is never logged (it can contain the resolved IP) — only the phase values + the exception *type*.
- Every phase has a 5 s timeout; the socket is always destroyed in `finally`.
- Off entirely unless debug logging is on.

`flutter analyze` clean, 166 tests pass.

## How to capture (Rich)
Apply it → debug logging ON → **cold start** (force-stop, ideally after the device has idled a while) → start a channel → start a couple more. Send the log. The `CONNPROBE` lines tell us if cold start-up time is in DNS (→ pre-resolve is worth ~15 lines), TLS, or neither (→ the time is in mpv/server and pre-resolve buys nothing).

+import 'dart:io';

+import 'package:open_tv/backend/app_logger.dart';

+/// fix421: one-shot connection-setup timing diagnostic. Measures the cold
+/// pre-first-byte phases — DNS resolve, TCP connect, TLS handshake — for a
+/// stream's host, so a log shows whether channel-start latency lives in
+/// connection setup (where DNS pre-resolution could help) or somewhere else.
+///
+/// Deliberately sends NO HTTP request: a bare TCP+TLS handshake carries no
+/// credentials, so it does not consume a provider "max connections" slot the
+/// way an authenticated GET would. Gated behind debug logging, fire-and-forget,
+/// and never blocks or fails playback. The host is redacted by the logger
+/// (fix415) like every other line.
+class ConnTiming {
+  static Future<void> probe(String url) async {
+    if (!AppLog.enabled) return;
+    final Uri uri;
+    try {
+      uri = Uri.parse(url);
+    } catch (_) {
+      return;
+    }
+    if (uri.scheme != 'http' && uri.scheme != 'https') return;
+    final host = uri.host;
+    if (host.isEmpty) return;
+    final isTls = uri.scheme == 'https';
+    final port = uri.hasPort ? uri.port : (isTls ? 443 : 80);
+    const limit = Duration(seconds: 5);
+    final sw = Stopwatch()..start();
+    int dns = -1, tcp = -1, tls = -1;
+    Socket? sock;
+    try {
+      final addrs = await InternetAddress.lookup(host).timeout(limit);
+      dns = sw.elapsedMilliseconds;
+      if (addrs.isEmpty) {
+        AppLog.info('CONNPROBE host=$host dns=${dns}ms (no address)');
+        return;
+      }
+      sw.reset();
+      sock = await Socket.connect(addrs.first, port).timeout(limit);
+      tcp = sw.elapsedMilliseconds;
+      if (isTls) {
+        sw.reset();
+        sock = await SecureSocket.secure(sock, host: host).timeout(limit);
+        tls = sw.elapsedMilliseconds;
+      }
+      final total = dns + tcp + (tls < 0 ? 0 : tls);
+      AppLog.info('CONNPROBE host=$host dns=${dns}ms tcp=${tcp}ms'
+          '${isTls ? ' tls=${tls}ms' : ''} total=${total}ms');
+    } catch (e) {
+      // Never log the raw exception — it can contain the resolved IP. The phase
+      // values already show how far the handshake got before failing.
+      AppLog.info('CONNPROBE host=$host dns=${dns}ms tcp=${tcp}ms'
+          '${isTls ? ' tls=${tls}ms' : ''} aborted=${e.runtimeType}');
+    } finally {
+      try {
+        sock?.destroy();
+      } catch (_) {}
+    }
+  }
+}
