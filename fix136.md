## Background

The player currently has **no orientation listener** â€” it only SETS preferred
orientations (forces landscape on fullscreen entry, restores on exit). Rotation
events are therefore invisible in the log. The connection lifecycle is driven
solely by channel open/close/swap/reconnect; nothing is wired to orientation,
which is correct. This fix records rotation so it can be confirmed and so any
future rotation-specific RENDER glitch (distinct from connection issues) has a
timeline.

`_PlayerState` is `class _PlayerState extends State<Player>` (player.dart:74)
with `initState` (149) and `dispose` (777). It does NOT currently mix in
`WidgetsBindingObserver`.

**(c) Implement `didChangeMetrics`** (place near other lifecycle overrides in

```dart
  // fix136: DIAGNOSTIC ONLY. Logs device rotation. Does NOT touch the engine,
  // connection, demuxer, or decode pipeline â€” rotation is a pure layout event;
  // the mpv Texture re-lays-out on its own. Recycling the connection here would
  // cause a needless rebuffer/reconnect on every rotate (and risk an instant
  // "Failed to open" on connection-limited providers), so we deliberately do
  // nothing but log + dump the surface state to confirm the Texture resizes
  // WITHOUT a reconnect.
  @override
  void didChangeMetrics() {
    super.didChangeMetrics();
    if (!mounted) return;
    final view = WidgetsBinding.instance.platformDispatcher.views.first;
    final size = view.physicalSize;
    final orientation =
        size.width >= size.height ? Orientation.landscape : Orientation.portrait;
    if (orientation == _lastOrientation) return; // not a rotation
    final prev = _lastOrientation;
    _lastOrientation = orientation;
    AppLog.info(
      'Player: ROTATE ${prev?.name ?? 'init'} â†’ ${orientation.name}'
      ' channel="${widget.channel.name}"'
      ' (no reconnect â€” layout only)',
    );
    // Confirm the video surface stays alive across the rotate (no connection
    // recycle): dump texture id / rect via fix130.4's logSurface.
    if (_engine is MpvEngine) {
      (_engine as MpvEngine).logSurface('rotate-${orientation.name}');
    }
  }
```

**(d) Unregister the observer in `dispose`.** In `dispose()` (line 777), add as
the FIRST line (before `unregisterMain`):
```dart
    WidgetsBinding.instance.removeObserver(this); // fix136
```

## What you'll see in the log

On each rotation while a full-screen channel plays:
```
Player: ROTATE portrait â†’ landscape channel="..." (no reconnect â€” layout only)
MpvEngine: SURFACE[rotate-landscape] eid=... textureId=<same id> rect=WxH playerWH=WxH ...
```
Key confirmations:
- **`textureId` is the SAME before and after** â†’ the surface was not recreated.
- **NO `MpvEngine: dispose()`, NO `open()`, NO `onDisconnect`/reconnect** around
  the ROTATE line â†’ the connection was not recycled.
- `rect`/`playerWH` may update to reflect the new layout â€” that's the Texture
  resizing, which is correct and harmless.

If a future rotate ever DOES show a dispose/open near a ROTATE line, that's a
regression to investigate â€” but with this build, rotation is layout-only.
