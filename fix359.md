## What this does

### MED-1 — migration registration out of order (`db_factory.dart`)
`SqliteMigration(12)` was registered before `SqliteMigration(11)`. sqlite_async 0.13.1 asserts migrations are added in ascending `toVersion` order, so any asserts-enabled build (debug/profile/widget tests) crashed at DB init. Release builds (asserts stripped) were never affected — which is why production never saw it.
**Fix:** swap the two registration blocks to 11-then-12; drop the now-obsolete fix170 "registered out of order" comment. **Schema-neutral:** migration 13 already re-creates `playback_metrics` + `idx_pm_session` with `IF NOT EXISTS`, and the runtime applies migrations by ascending `toVersion` regardless of registration order, so no upgraded or fresh device sees any schema difference. **Rule 8 executed:** modelled the v10→11→12→13 DDL — the chain yields the table+index exactly once; 13 is idempotent over 12 and self-sufficient alone; reorder is provably schema-neutral. Migration ceiling unchanged at **28**.

### MED-2 — import dialog claimed "replace all sources" but merges (`settings_io.dart`)
The confirm dialog said it would "overwrite your current settings and replace all sources," but the import path merges by source name (existing sources are reused/updated, not deleted). **Fix:** wording now states it updates settings and adds/updates the backup's sources while keeping existing ones — matching the actual (and intended) merge semantics. No behaviour change.

### MED-3 — log rotation never fired mid-session (`app_logger.dart`)
The 20 MB cap was only evaluated in `_ensureOpen()` (enable/clear), so a long logging session grew unbounded. **Fix:** `log()` now tracks bytes written since open and triggers an async `_rotate()` (close → rename current to `.old`, replacing any prior `.old` → reopen fresh) when the cap is crossed; a re-entrancy guard prevents overlap, and the byte counter resets on every `_ensureOpen()`.

### LOW-1 — leaked FocusNode per rebuild (`settings_view.dart`)
`_DpadSliderState.build()` allocated a `FocusNode(skipTraversal:true)` every rebuild (one per drag frame). **Fix:** hoisted to a `late final _innerInertNode`, disposed with the state.

### LOW-2 — empty DVR cache dir left on low-disk abort (`mpv_engine.dart`, fix357 follow-up)
`_computeDvrWindowMB()` created `free4me_dvr` and set `_dvrDir` before the under-5-minutes check; a low-disk `return 0` then left an empty dir behind (cleanup only runs when `_dvrActive`). **Fix:** size first against the temp path; create the dir and set `_dvrDir` only once a non-zero window is committed.
