# fix284 — Hotfix to fix282: self-heal aborts on "local changes would be overwritten"

> ## ⚠️ BUILD MACHINE — EXECUTE EXACTLY
> One small edit to `.github/workflows/release.yml`. Apply the `Replacement` block verbatim, commit, push. Then re-tag the release. No app code changes.

## What happened (fix282 worked — this is a git-mechanics bug in it)
fix282's self-heal fired correctly: it detected the stale version.json and began repairing. It then failed with:
```
error: Your local changes to the following files would be overwritten by checkout:
 version.json
Aborting
```
Cause: the step runs `python3 scripts/update_version_json.py` (which modifies version.json in the working tree) and then `git checkout -B main origin/main`. Git refuses to switch branches because the uncommitted version.json change would be lost. The diagnosis behind fix282 is correct — this is just a missing stash before the branch switch.

## Verified
Reproduced the exact failure and the fix in a real git repo with a real origin: with a `git stash --include-untracked` before the checkout (and `git stash drop` after), the self-heal completes — commits the synced version.json to main, moves the tag, and the guard passes on the re-tagged commit.

## Fix 284.1 — `.github/workflows/release.yml`: stash before the branch switch
### Current code (verbatim)
```yaml
          echo "version.json was stale — regenerating, committing to main, and moving $GH_TAG."
          git --no-pager diff version.json || true
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          # Commit the regenerated version.json onto main, then point the tag here.
          git fetch origin main
          git checkout -B main origin/main
          python3 scripts/update_version_json.py
```
### Replacement code (verbatim)
```yaml
          echo "version.json was stale — regenerating, committing to main, and moving $GH_TAG."
          git --no-pager diff version.json || true
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          # fix284: park the just-regenerated version.json so the branch switch
          # below cannot be blocked by "local changes would be overwritten".
          git stash --include-untracked || true
          # Commit the regenerated version.json onto main, then point the tag here.
          git fetch origin main
          git checkout -B main origin/main
          git stash drop || true
          python3 scripts/update_version_json.py
```
> Why drop the stash: after switching to main we regenerate version.json fresh (the next line), so the stashed copy is not needed. Dropping it avoids leaving stash entries around on the runner. `|| true` keeps the step from failing if the stash was empty.

## Step 3 — re-release 1.26.9 (delete the bad tag/release; CI self-heals on re-tag)
```bash
GITHUB_TOKEN=$(security find-internet-password -s "api.github.com" -a "rkinnc75" -w 2>/dev/null)
git tag -d v1.26.9 2>/dev/null || true
git push origin :refs/tags/v1.26.9 2>/dev/null || true
REL_ID=$(curl -sf -H "Authorization: token $GITHUB_TOKEN" \
  "https://api.github.com/repos/rkinnc75/Free4Me-IPTV/releases/tags/v1.26.9" \
  | python3 -c "import sys,json;print(json.load(sys.stdin).get('id',''))" 2>/dev/null || true)
if [ -n "$REL_ID" ]; then
  curl -sf -X DELETE -H "Authorization: token $GITHUB_TOKEN" \
    "https://api.github.com/repos/rkinnc75/Free4Me-IPTV/releases/$REL_ID"
fi
git fetch origin && git checkout main && git reset --hard origin/main
grep '^version:' pubspec.yaml          # confirm 1.26.9+282
git tag v1.26.9
git push origin v1.26.9
```
CI fires on the tag push. The self-heal now completes: it stashes the regenerated version.json, switches to main, regenerates+commits version.json there, pushes main, moves v1.26.9 to that commit, and the re-triggered run builds + publishes. version.json staleness can no longer fail or stall a release.
