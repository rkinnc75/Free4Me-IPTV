# fix613 — Search Perf Test: same query set for cold vs warm + sanitized probes

**Target:** v2.2.24+613 (diff against live tag **v2.2.23**, verified `git ls-remote --tags`)
**Apply check:** `git apply --check` PASSED on a fresh v2.2.23 clone.
**Analyze:** run on the build machine — no Flutter toolchain in the authoring sandbox, so the 0-issue gate is NOT asserted here. (Manual checks: brace/paren balance, no stale `_Probes` refs, raw-string regex intact after apply.)

## Why (from the live result + report `2026-06-29T18-15-59Z-onn4kplusstreami`)
The first version reported **cold faster than warm**, which looked wrong. The log shows it wasn't a timing error — it was a **design flaw**:
- **Cold and warm measured different workloads.** Cold timed only the *first* probe (`oneWord.first`); warm timed all 15 across one-word + multi-word + prefix. A cheap first query made cold look faster than a median that included slower multi-word queries. Not apples-to-apples.
- **The probes were garbage.** Sampling `words.first` verbatim produced `"#Li"`, `"'Al"`, `"(AU"` — names beginning with punctuation. One of those cold FTS queries even logged 8473 ms. Those aren't how anyone searches.

## Fix
- **Same probe set for both.** Build one flat, ordered list of 15 queries (5 one-word, 5 multi-word, 5 three-char prefix). **Cold** = median of pass 1 with SQLite's page cache dropped first; **warm** = median over 3 further passes of the *identical* list. Cold vs warm now differ only by cache state.
- **Sanitized probes.** `_clean()` keeps only alphanumerics (Unicode-aware), stripping leading/trailing symbols, with de-duplication — no more `"#Li"` / `"(AU"`.
- **inMemory** still rebuilds its cache before pass 1 (so its queries time against a built cache); the one-time build cost is no longer smeared into the per-query median. Degraded flag unchanged.
- **Caveat text updated** to say cold and warm run the same searches and differ only by cache (the OS-file-cache caveat across methods still stands).

## Files
- `lib/backend/search_perf_test.dart` — pass-based cold/warm over a shared sanitized probe list; `_Probes` class removed; `_clean()` added; `_warmPasses` const.
- `lib/widgets/search_perf_dialog.dart` — updated caveat text.
- `pubspec.yaml`, `lib/whats_new_modal.dart` — version bump + changelog.
