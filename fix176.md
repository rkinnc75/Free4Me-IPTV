# fix176 — fix174's escaped `$` interpolations broke analyze; and harden the preflight (the runbook author CAN now compile)

> **Version:** 1.23.10+174 → **1.23.11+176** (phone/even build; patch bump from shipped 1.23.10+174).
> **Evidence:** `Free4Me-IPTV-main` source zip (post-fix174 tree, `version: 1.23.10+174`) + the CI `flutter analyze --no-fatal-infos` output that failed with 6 issues.
> **Bugs:**
> 1. **fix174's code blocks contained backslash-escaped `$` (and `\u2022`) that were pasted verbatim into the source, producing 6 analyzer warnings → CI fail.** All six trace to one authoring defect: escaped string interpolation inside markdown code fences. `VALUES \$values` put the literal text `$values` into the SQL and left the Dart `values` unused; `\${_etaSuffix(...)}` turned the helper call into dead string text, orphaning `_etaSuffix` and its `m`/`r` locals.
> 2. **The preflight doc's founding premise — "the runbook author has no Flutter SDK and cannot compile" — is false.** The Dart SDK installs from `storage.googleapis.com` and Flutter from `github.com`, both on the container network allowlist. `flutter pub get` + `flutter analyze` run in the container. The entire "verify by inspection only" posture that allowed bug 1 through was built on an untested assumption.
> **Root cause (bug 1):** runbook code fences are not syntax-checked; the brace/quote-balance preflight treats string-literal contents as opaque, so an escaped `$` inside a string is invisible to it. **Root cause (bug 2):** an assumption ("no SDK / can't compile") was never tested; it was tested for the first time during fix174 triage and proved wrong.

Six sites, two files, all the same escape defect (grep `'\\\$'` / `'\\u2022'`):

# Fix 176.1 — remove the escaped `$` / `\u2022` from the six sites

These are one-character (drop-the-backslash) corrections. `\u2022` is also replaced with a literal `•` to remove the escape entirely (both compile, but a literal bullet can't be re-escaped by a future copy-paste). **Two attached pre-corrected files** (`sql.dart`, `sources_refresh_dialog.dart`) are the verified-clean versions — the build machine may drop them in directly *or* apply the six edits below.

## Part A — `lib/backend/sql.dart`

Current verbatim (line 133, inside `insertChannelsBulk`'s `tx.execute('''…''')`):

```dart
        VALUES \$values
```

Replacement:

```dart
        VALUES $values
```

That single change makes `$values` interpolate (so the SQL gets the row placeholders) and makes the `values` local used (clears the :116 warning). No other edit in this file.

## Part B — `lib/widgets/sources_refresh_dialog.dart` (five sites)

**B1 — line 35**, current:

```dart
  if (s < 60) return '  \u2022  ~\${s}s left';
```

Replacement:

```dart
  if (s < 60) return '  • ~${s}s left';
```

**B2 — line 38**, current:

```dart
  return '  \u2022  ~\${m}m \${r}s left';
```

Replacement:

```dart
  return '  • ~${m}m ${r}s left';
```

**B3 — the Text progress line (lines 87–89)**, current:

```dart
                      '\${((rowsDone / rowsTotal) * 100).clamp(0, 100).toStringAsFixed(0)}%'
                      '  \u2022  \${rowsDone.clamp(0, rowsTotal)} / \$rowsTotal'
                      '\${_etaSuffix(saveStartedAt, rowsDone, rowsTotal)}',
```

Replacement:

```dart
                      '${((rowsDone / rowsTotal) * 100).clamp(0, 100).toStringAsFixed(0)}%'
                      '  • ${rowsDone.clamp(0, rowsTotal)} / $rowsTotal'
                      '${_etaSuffix(saveStartedAt, rowsDone, rowsTotal)}',
```

B3 makes `_etaSuffix` actually called (clears `unused_element` at :27), and once it's called its `m`/`r` locals are reachable through real interpolation (clears :36/:37). All five B-edits are applied as one block per file to avoid partial application.
