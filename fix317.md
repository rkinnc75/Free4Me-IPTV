## What this does
Adds an **Import sources** form to the TV LAN export web portal (the QR-code page, port 9479). Upload a settings file from a phone/PC browser and the device imports **only the sources** — all other settings are skipped (device-specific: buffering, decode, multi-view, etc. shouldn't cross devices). Imported sources are **merged by name** into the existing list (existing reused, new added), and the device **auto-refreshes** the sources on success.

## How it's built
- `SettingsIo.importSourcesOnly(bytes)` — parses the backup JSON, validates schema, inserts each source via `Sql.getOrCreateSourceByName` (merge), stages channel preserves (favorites/history/EPG ids) for the post-refresh re-apply, and returns the imported count (-1 on parse/schema error). Touches nothing else.
- `ExportServer` gains an optional `onImportSources` callback, an upload form on the index page, and a `POST /import-sources` handler with a minimal multipart/form-data parser (pure Dart — extracts the file part bytes).
- `settings_view._showExportServerDialog` wires the callback: import → `reloadSources()` → `showSourcesRefreshDialog` (post-frame, so the HTTP response returns first). The server runs on the app isolate, so DB writes + refresh happen directly.
