## Why (from the 17:42 log, Galaxy S24 Ultra / sms938u, on 1.26.38)
fix322's diagnostics gave the answer: `notification permission = granted` **but** `startService not successful (result=Instance of 'ServiceRequestFailure')`. Cause: `flutter_foreground_task` **requires a `<service>` declaration in `AndroidManifest.xml`** — ours was never added (fix318 added only the permissions). On Android 14+ a foreground service cannot start without a declared `android:foregroundServiceType`, so `startService` fails. Confirmed against the plugin docs for the pinned `^8.17.0`.

## What this does
- Adds the plugin's required service block inside `<application>`:
  `com.pravera.flutter_foreground_task.service.ForegroundService`, `foregroundServiceType="dataSync"` (matches the existing `FOREGROUND_SERVICE_DATA_SYNC` permission), `stopWithTask="true"`, `exported="false"`.
- Service name is fixed by the plugin — **do not rename**.
- No Dart changes, no new fields, no SQL/migration.
