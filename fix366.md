## Root cause (on-device 1.32.4+364, 2026-06-13 20:04 log + screenshot)
fix364 correctly added the `dvrActive` getter and the transport bar, yet the controls still didn't show. The log proves DVR fully activates (`MpvEngine: DVR enabled — window=30min …`), but the engine sets `_dvrActive = true` **inside `open()`** — i.e. AFTER the player's controls theme (`_mpvThemeData`, which contains `primaryButtonBar`) was first built with `dvrActive == false`. On a stream that then plays cleanly with no further `setState`, the bar is never re-evaluated, so it stays on the play/pause-only branch. (The theme IS rebuilt on `setState`, but nothing fired one specifically when `_dvrActive` flipped.)

## Fix
One line in `player.dart`: immediately after `open()` succeeds, if `_engine.dvrActive`, call `setState(() {})`. That rebuilds the controls theme, `primaryButtonBar` re-evaluates `_engine.dvrActive` (now true), and the rewind / play-pause / forward / back-to-live row renders. No-op when DVR is off (condition guards it) and for VOD.
