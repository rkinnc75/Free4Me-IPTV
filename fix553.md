## Changes

### #550 correction — report-dialog focus (`settings_view.dart`)
On-device, focus should **start on Subject**, not Submit; after submitting, focus
should land on **OK**.
- Subject `TextField`: `autofocus: true` (dialog opens ready to type).
- Submit `FilledButton`: removed `autofocus` (no longer steals initial focus).
- Result dialog OK `TextButton`: `autofocus: true`.
The fix550 DpadFocusEscape traversal (D-pad can leave the Details field) is
unchanged — this only moves where focus starts/ends.

### #551 correction — Categories left-rail + unified grids
The fix547 centered pill row (Live TV / Movies / Series) is replaced by a **left
vertical type-rail**, matching the Movies/Series browse layout, so all three TV
screens share one structure.
- `tv_categories_view.dart`: `build()` is now `Row [210px _typeRail | divider |
  body]`. New `_typeRail()` / `_typeRailItem()` use a local `_FocusTile`
  (mirrors `TvBrowseView._FocusTile`, with `autofocus`). The old `_TypeButton`
  widget is removed. Selecting a type switches the whole screen's category grid.
  The rail owns initial focus, so the grid's first tile no longer autofocuses
  (removes a focus conflict).
- **Unified grid tile across all three grids** (Categories + Movies + Series):
  `maxCrossAxisExtent: 120`, `childAspectRatio: 0.773`, spacing `10`.
  - Cards are **wider** (Movies/Series extent 90→120; Categories 150→120) with
    **tighter spacing** (16/14 → 10).
  - **Card height is held constant** at the Movies/Series ~155px: AR 0.773 =
    120/155. (Categories was 150-wide/203-tall → now 120/155, matching.)
- `tv_browse_view.dart`: Movies/Series grid updated to the same 120/0.773/10.

### #552 correction — icon corner crop (`assets/icon.png`)
The square neon PNG showed **white corners** behind the rounded TV-header frame.
Replaced with the same art but **rounded corners baked as transparency**
(RGBA, ~18% corner radius), so it renders cleanly at any clip. Matches the
launcher icon shape.

### Scrim → 75% (`tv_shell.dart`)
`Color(0xCC000000)` (80%) → **`Color(0xBF000000)`** (75%) per on-device review.
