# fix333 — Category enable checkbox unreachable by D-pad on TV → v1.26.49+333

## Why
On TV the per-category enable checkbox (Categories view) cannot be reached with the remote. Cause in `channel_tile.dart`: the whole tile is one `InkWell` with a single focus node, and the checkbox was a plain `Checkbox` child with no focus node — so a D-pad had exactly one focusable target per tile (select = open the category) and could never land on the checkbox. (Touch works because you tap it directly.)

## What this does
- Wraps the checkbox in its own focusable `InkWell` (`canRequestFocus: true`), with `IgnorePointer` on the inner `Checkbox` so activation has a single source of truth (the InkWell's `onTap` toggles enabled). 
- The tile's existing focus node already forwards arrow-right via `focusInDirection(right)`, so right-arrow from a category tile now lands on the checkbox (with a visible focus highlight) and select toggles it; it still falls through to the navbar when there's nothing to the right.
- Categories-view only (guarded by `mediaType == group && onToggleEnabled != null`); non-category tiles unchanged.
