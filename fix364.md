## Root cause (confirmed by on-device screenshot + log, 1.32.3+363)
fix360 was supposed to add: (a) a `dvrActive` getter on the engine, (b) the rewind/forward/back-to-live transport bar in `player.dart`. **On live main only the `_dvrActive` field landed** — the `dvrActive` getter is absent from `mpv_engine.dart`, the interface getter is absent from `player_engine.dart`, and the transport bar is absent from `player.dart` (still the fix348 "DVR deferred" code). The engine half of DVR works perfectly (log: `DVR enabled — window=30min … e401667071`, guard running, per-engine dir from fix363), but the UI read the **interface default `false`**, so the transport row never rendered — exactly the screenshot (DVR live + paused, only play/pause shown). fix360's player/engine-interface hunks evidently failed to apply at release time and were committed half-done.

## What this does (re-applies the missing fix360 pieces on current main)
- `lib/player/player_engine.dart` — adds `bool get dvrActive => false;` to the interface.
- `lib/player/mpv_engine.dart` — adds the real `@override bool get dvrActive => _dvrActive;` (THE missing line that caused the UI to see false).
- `lib/player.dart` — adds `_dvrSeekBy` / `_dvrGoLive` / `_dvrButton` helpers and the live transport bar: when `mediaType == livestream && _engine.dvrActive` → **rewind 10s · play/pause · forward 10s · back-to-live** (real IconButtons, D-pad LEFT/RIGHT traversal, CENTER activates). DVR off → unchanged play/pause-only live bar. VOD unchanged.
