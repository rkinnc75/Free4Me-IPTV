# fix539 — TV: Favorites built into Live/Movies/Series; remove "All" + standalone Favorites tab

**Base tag:** `v2.0.38`
**Version:** `2.0.38+538` → `2.0.39+539`
**Scope:** TV shell + guide + browse view. No schema/query change (reuses `ViewType.favorites`). Full suite green.

---

## Changes (item 5)

1. **Movies/Series (`tv_browse_view.dart`):** the rail's top item "All" is replaced by **"Favorites"** (a `_favGroupId = -1` sentinel). Selecting it loads `ViewType.favorites` scoped to that tab's single media type; real category ids still scope into their category. The view now **defaults to Favorites** on open.
2. **Live (`tv_guide_view.dart`):** the **"All" pill is removed**; "Favorites" becomes a sticky toggle (tap to switch favorites-only ⇄ all). Live now **defaults to favorites-only** (`_favOnly = true`), consistent with Movies/Series.
3. **Shell (`tv_shell.dart`):** the standalone **Favorites tab is removed** (redundant now). `_landingIndex` updated — a saved "favorites" default view now lands on Live TV (which itself defaults to Favorites); History shifts from index 5 → 4. The History long-press handler resolves by `viewType`, so the index shift needs no further change.

Favorites correctness: `ViewType.favorites` appends `AND favorite = 1` to the normal media-type-filtered query, so favorites + a single `mediaType` yields media-type-scoped favorites.
